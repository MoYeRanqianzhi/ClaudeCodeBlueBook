# 蓝皮书索引

## 这本书写什么

这不是一份 API 速查表，也不是一份产品宣传材料。`CCBlueBook` 关注的是 Claude Code 的三层结构：

1. 表层：用户能看到什么能力，应该如何高效使用。
2. 中层：Agent loop、工具系统、权限系统、扩展系统如何配合。
3. 深层：它为什么强，强在 prompt 之外的哪些工程化护栏。

## 当前结论

- Claude Code 的核心不是“模型一次答得好”，而是“模型被放进了一套长期可运行的操作系统里”。
- 真正让它变强的是 harness：启动优化、权限系统、上下文压缩、任务系统、扩展能力、多 Agent 隔离、远程执行与产品化开关。
- 使用层最佳实践和源码设计哲学高度一致：任务外化、上下文最小化、并行化、精确授权、隔离执行、结果回收。
