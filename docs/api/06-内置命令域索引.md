# 内置命令域索引

本章不是列出每个命令的全部细节，而是先把内置命令面按功能域排清楚，给后续补全“全部功能支持”打地基。

## 1. 命令系统的三层边界

Claude Code 的命令支持不能只看 `src/commands/` 目录，还要同时看：

1. `COMMANDS()` 内置主表
2. `INTERNAL_ONLY_COMMANDS`
3. `getCommands()` 最终装配结果

证据：

- `claude-code-source-code/src/commands.ts:225-254`
- `claude-code-source-code/src/commands.ts:258-346`
- `claude-code-source-code/src/commands.ts:449-517`

## 3. 功能域划分

- `clear`

- `resume`

- `security-review`
- `branch`
- `files`
- `commit`

这类命令主要面向代码修改、审查和提交工作流。

## 4. 有哪些命令是“结构上值得注意”的

### 4.1 `context` 同时有交互版和非交互版

这说明同一功能域会按运行模式拆出不同命令实现。

### 4.2 `review` 同时覆盖 ultrareview 相关逻辑

这说明命令层已经吸收产品分层，而不只是简单地“触发一个 prompt”。

### 4.3 `plugin` 是一个完整 UI 子系统

它不是单一命令文件，而是由多组件拼成的本地管理面。

## 5. 为什么命令域索引重要

从第一性原理看，Claude Code 不是只靠自然语言驱动。  
用户也通过命令在做 orchestration。

如果蓝皮书不把命令面系统化，就会错误低估：

- Claude Code 的控制平面
- Claude Code 的产品表面
- Claude Code 的“可操作性”
