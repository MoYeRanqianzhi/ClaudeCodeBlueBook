## 本轮继续深入的核心判断

这一轮最值得收的，

不是继续给 `03-参考索引/README.md` 强化 section-level hub 语气，

而是把它压回：

- 索引结构总览
- 误判回查定位

## 为什么这轮值得单独提交

### 判断一：问题不在它列目录，而在它把自己写成“入口角色”

当前 `03-参考索引/README.md` 一开头仍写成：

- 承担两个入口角色
- 先给出 section-level hub，方便按目录结构进入

这会让“参考索引”这一层继续像：

- 前门分诊页

而不是：

- deep inventory / 回查定位页

但从它实际内容看，

这一层真正擅长的是：

- 组织各类索引面
- 让读者带着误判回来定位

而不是再当一次高层 front-door。

## 这轮具体怎么拆

### 判断二：保留目录总览，但去掉“入口角色”话语

这轮只改三处：

1. 把开头从：
   - `承担两个入口角色`
   改成：
   - `主要承担两件事`
2. 把：
   - `section-level hub`
   改成：
   - `索引结构总览`
3. 把：
   - `如果你已经知道自己要找哪类叶子问题`
   改成：
   - `如果你已经带着具体误判或叶子问题回来回查`

这样：

- 这页仍保留索引目录
- 但不再自我定义成 front-door router
