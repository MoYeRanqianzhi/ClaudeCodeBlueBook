# memory-root subfamily exposure narrowing 拆分记忆

## 本轮继续深入的核心判断

在当前 worktree 已经有：

- `100-149` family hub
- `122-149` subfamily hub
- `128-138 / 133-138 / 139-149 / 144-149` 更深一层 subfamily hub

之后，

这一轮最值钱的目录优化，

不是继续新增目录，

也不是让根 `README` 把所有深层 subfamily 都挂出来，

而是把：

- `docs/userbook-memory/05-结构演化与迭代记录/README.md`

压回：

- 根 family hub
- 一个高流量的中层 subfamily hub

这两级 first-stop 入口。

换句话说，

根索引该回答的是：

- “我第一次该停在哪”

不是：

- “这棵树还能往下裂几层”

## 为什么这轮值得单独提交

### 判断一：当前问题不在“有没有入口”，而在“根入口暴露得太深”

根 `README` 现在同时挂着：

- `122-149`
- `128-138`
- `133-138`
- `139-149`
- `144-149`

这会让根 quick index 逐渐长成：

- 一份更短的全树目录

而不是：

- 一份 first-stop 路由

但从阅读路径看，

当读者已经知道自己落进：

- `122-149`

就已经完成了一次足够稳定的对象收窄。

此时更深的第二次分流，

应该在 `122-149` hub 内完成，

不该再被根索引重复暴露。

### 判断二：目录优化不是持续加目录，而是控制暴露层级

这轮如果继续新增更细目录，

或者继续把更深 subfamily 回挂到根 README，

都会把：

- family hub
- subfamily hub
- root quick index

这三层的职责重新写平。

更稳的最小动作其实是：

1. 根 `README` 只保留：
   - `168-190`
   - `100-149`
   - `191-206`
   - `122-149`
2. 把更深的：
   - `128-138`
   - `133-138`
   - `139-149`
   - `144-149`
   退回 `122-149` hub 内部

这样目录结构变得更窄，

但读者的 first-stop 成本反而更低。

### 判断三：这轮补的不是路径数量，而是根入口的角色边界

这一刀真正保护的是：

- 根索引只签 first-stop
- `122-149` 这种中层 hub 才负责第二次分流
- 更深 subfamily 只在父 hub 内继续挂载

这和前面多轮结构页 hardening 的核心判断是一致的：

- 上层只签骨架
- 下层才继续展开

## 这轮具体怎么拆

这轮只做三件事：

1. 修改 `memory/05 README`
   - 删除根 quick index 上更深的 `128-138 / 133-138 / 139-149 / 144-149` 直挂入口
   - 改成一条明确说明：进入 `122-149` 后，再在该 hub 内继续分流
2. 修改 `122-149` hub
   - 明说它就是 root 之后的第二次分流层
   - 明说不必再回根索引追更深子家族
3. 新增这条 `399` 记忆
   - 记录为什么目录优化在这一轮表现为“暴露层级收窄”，而不是“继续加层”

这样：

- 根索引不再复制整棵树的中下层
- `122-149` 真正承担起 subfamily hub 的职责
- 目录结构优化也和第一性原理对齐：先停对层级，再往下分

## 苏格拉底式自审

### 问：我是不是删掉了有价值的路径，导致读者更难找到 `139-149` 或 `144-149`？

答：不是。那些入口没有消失，只是退回了 `122-149` hub 内部；根索引失去的是重复暴露，不是可达性。

### 问：为什么不把 `128-138`、`139-149` 继续和 `122-149` 并列留在根索引？

答：因为一旦 `122-149` 已经存在，这些更深 subfamily 再和它并列，就会把“第一停”和“第二停”混成一层。

### 问：为什么这轮不继续抽一个更细的新 hub？

答：因为当前问题不是缺少新入口，而是根入口已经暴露过深。继续抽新 hub 只会掩盖真正的层级失真。

### 问：为什么这也算“加强对使用功能的剖析”？

答：因为 userbook 的使用剖析本质上依赖 first-answer order。根入口如果复制过多深层名词，读者就会重新退回“按页碰运气”，而不是按能力/控制面逐层收窄。
