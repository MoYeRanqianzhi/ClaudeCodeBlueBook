# context intake topic-layer route split 拆分记忆

## 本轮继续深入的核心判断

在完成：

- 顶层 README 的“把上下文送准”目标入口
- `00-阅读路径` 的路径 5
- `08-上下文接入、附件与提示编译专题`

之后，

这条线当前最小却仍缺的一刀是：

- `04-专题深潜/README`
  里的 topic-layer route

## 为什么这轮该补专题层 route

### 判断一：顶层和阅读路径都能把人带到这条线，但专题层还没把它抬成同级 first-hop

当前用户已经可以在：

- 顶层 README
- 路径 5

看到这条线，

但进入 `04-专题深潜`

之后，

前面的 first-hop 还没有把“把上下文送准”抬到和：

- settings/status/budget 分流
- continuity
- session operations

同级。

所以这轮缺的是：

- topic-layer discoverability

不是：

- 再写正文

## 为什么这条 route 要先给路径 5，再给专题 08

### 判断二：专题层 first-hop 回答的是“先去哪”，不是“正文内部怎么展开”

`08`

正文已经能回答：

- 提示词、附件、图片、IDE 选区
- `/files`、`/context`
- `/add-dir`
- hooks 与本地命令回流

这些对象的边界。

但如果读者此时还没分清：

- 该先把事实挂成附件/选区/path refs
- 还是该正式扩工作面

直接只送专题 08，

仍会跳过路径 5 已经给出的阅读顺序价值。

所以这轮 route 更稳的顺序应当是：

- 先给路径 5
- 再给专题 08
- 最后把“正式扩工作面”分流到 `05/07 Add-dir`

## 为什么这轮不先改任务矩阵

### 判断三：矩阵层当然也有价值，但读者一旦已经进入专题层，眼前更近的断点是 first-hop 缺失

并行只读分析提示：

- 任务矩阵也缺“补上下文/送附件”这一跳

但对已经进入 `04-专题深潜`

的读者来说，

当前更直接的物理断点仍然是：

- 看见专题列表，却没有看到这条线被抬出来

所以这轮先补：

- `04-专题深潜/README`

下一轮再考虑：

- `03-参考索引/05`

## 为什么 route 文案必须保护 `/add-dir` 的主语

### 判断四：`/add-dir` 回答的是工作面制度承认，不是一般意义上的“补上下文”

如果文案只写成：

- “上下文不够就去 `/add-dir`”

会重新把：

- 附件
- 图片
- IDE 选区
- path refs
- `/files`
- `/context`

和：

- 正式扩工作面

混成一件事。

所以这轮 route 必须显式写出：

- 如果卡在“新目录到底算不算已经正式进入当前工作面”，再继续看 `Add-dir`

而不是把它和一般材料接入写成同层。

## 苏格拉底式自审

### 问：我现在缺的是再写一遍上下文接入正文，还是缺把这条高频工作线在专题层抬出来？

答：如果顶层、路径和专题正文都已存在，先缺的是 topic-layer route。

### 问：我是不是又想把 `/add-dir` 当成“所有上下文问题的下一步”，从而抹掉附件、选区和 path refs 的主语？

答：如果 route 不显式保护“正式扩工作面”这层主语，这个误判会立刻复发。

### 问：我是不是应该直接去补任务矩阵，而不先修专题层里更近的断点？

答：对已经进入专题层的读者来说，first-hop 缺口更近，所以应先补 route，再补矩阵。
