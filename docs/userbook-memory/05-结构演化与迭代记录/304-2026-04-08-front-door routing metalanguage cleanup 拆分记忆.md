## 为什么这轮仍值得单独提交

### 判断一：问题不在“建议先看后看”，而在“把建议读法写成栏目本体属性”

当前根页、导读、专题 README、控制面 README 已经基本形成稳定分工：

- 根页负责按目标进入
- 导读负责解释栏目职责
- 专题与控制面 README 负责按主题继续下钻

但这些前门页里还残留着一组旧话语：

- `first-hop`
- `second-hop`
- `第一跳`

它们会把：

- 这次建议先看什么

写成：

- 这个栏目天然属于第几跳

这会继续混淆：

- 阅读建议
- 栏目职责
- 物理目录结构
