## 苏格拉底式自审

### 问：为什么这轮不是去做 `116-127`？

答：因为 `116-127` 会和 `122-149` 发生显著范围重叠；`100-115` 则是更干净、边界更清楚的一段。

### 问：为什么这轮不是继续补 `208/209/210` 的页首或分层？

答：因为这三页已经有结构页和 scope guard。当前缺的是目录层怎样把人直接送到这一簇，而不是再加一层正文说明。

### 问：为什么这也算“加强对使用功能的剖析”？

答：因为用户对 summary / wire / callback / UI 这半段的理解，本质上依赖先走对入口。没有 `100-115` 这层子家族，读者仍会在 `100-149` 家族里按范围碰运气，而不是沿着对象层级下钻。
