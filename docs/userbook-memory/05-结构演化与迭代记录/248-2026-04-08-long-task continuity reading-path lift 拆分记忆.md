## 为什么这条路径要先去主线 04，再去专题 02，最后才去专题 12

### 判断二：读者先要知道“什么时候用哪条手段”，再要知道“这些手段各自延续什么对象”，最后才需要“过去对象怎么被找到”

`01-主线使用/04`

先给：

- `/memory`
- `/compact`
- `/resume`

的动作判断。

`04-专题深潜/02`

再给：

- continuity object model

也就是：

- 当前工作对象如何继续

`04-专题深潜/12`

最后才接：

- 以前的工作在哪里
- 该恢复哪个会话

所以这条路径不应一上来就去：

- 会话发现

否则 continuity 会再次被误压成：

- 历史会话功能

## 为什么这轮要用“当前/过去”“恢复前/恢复后”来收口，而不碰正文合同

### 判断三：当前问题是索引措辞边界，不是能力合同本身出了错

并行边界分析提示：

- `/resume` 天然横跨 continuity 与 session discovery
- `/rename` / custom title 也是跨边界能力

因此这轮不该去重写：

- `02-连续性与记忆专题` 的稳定面 / 条件面
- `12-会话发现、历史检索与恢复选择专题` 的稳定主线 / 条件边界

真正需要补的是：

- 索引层把“当前工作对象如何继续”
- 与“过去工作对象如何被找到”

拆开说清。

所以本轮阅读路径文案只应强化：

- 当前 / 过去
- 恢复前 / 恢复后

而不去动正文里的能力分层合同。
