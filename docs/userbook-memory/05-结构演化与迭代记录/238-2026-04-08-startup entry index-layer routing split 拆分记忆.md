## 为什么 `03-参考索引/01-命令工具/README` 也要补

### 判断三：用户进到命令工具目录时，很多时候并不是来查单个命令，而是来分层

`03-参考索引/01-命令工具/README`

当前只是六个索引页的裸列表。

但进入这个目录的用户，常见真实问题并不是：

- “我只要找一个命令名字”

而是：

- root flags / root commands / slash commands 到底怎么分
- 为什么同名 root/slash 入口不在一层

这说明这个 README 也应该先给出：

- 路径 14
- 路径 15

再让用户决定是否进入具体索引页。

## 为什么这轮不改顶层 README

### 判断四：顶层发现性已经补过，当前缺的只是中层索引接力

顶层 README 现在已经能把用户送到：

- 启动入口分流

所以当前问题不在最外层。

问题在于用户一旦进入：

- 专题目录
- 命令工具目录

就又会掉回：

- 文件列表

而不是：

- 路由对象

所以这轮只补中层索引接力，不重复改顶层。
