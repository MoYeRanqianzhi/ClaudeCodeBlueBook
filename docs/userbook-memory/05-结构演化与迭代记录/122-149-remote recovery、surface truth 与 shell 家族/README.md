- `122-127` remote recovery / ownership / terminality / compaction

- `139-149`

## 第一性原理

这组记忆真正想回答的不是：

- “`122-149` 该按编号怎么顺着翻”

而是：

- 哪些记忆负责把 `122-127` 收成 recovery 双主干与 zoom
- 哪些记忆负责把 `128-138` 收成 surface presence 到 interaction shell 的接力线
- 哪些记忆负责把 `139-149` 收成 remote truth、shell pair 与 env-driven persistence 的后继结构

所以这个目录的作用只是：

- 把现有结构页、scope guard 与子区间接力关系收成一个更窄的家族入口

不是：

- 再制造一张新的 umbrella map
