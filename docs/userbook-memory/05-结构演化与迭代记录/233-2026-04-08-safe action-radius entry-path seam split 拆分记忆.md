# safe action-radius entry-path seam split 拆分记忆

## 本轮继续深入的核心判断

`安全放大工作面`

这条线现在已经有：

- 顶层目标入口
- 物理前门
- 独立阅读路径

但这三层之间还缺一个很轻的接缝：

- 顶层入口没有把首跳正式交给“路径 14”
- 物理前门也没有提醒“如果你已经确认问题落在这条线里，去路径 14”

所以这轮更稳的做法不是继续补正文，

而是：

- 把 entry -> front door -> reading path 串成连续导航线

## 为什么这轮该补“入口到路径”的接缝

### 判断一：顶层目标入口回答“先去哪”，阅读路径回答“接下来怎么连续推进”

`bluebook/userbook/README.md`

里的“按目标进入”更像：

- 立即命中问题

`00-阅读路径.md`

里的路径则回答：

- 如果我要按稳定顺序继续深入，该怎么往下读

如果顶层入口仍然先把用户送到物理前门，

却没有先把首跳交给：

- 已存在的独立路径

用户就还是会在：

- 做完第一跳分诊后
- 自己回头去找路径

这说明当前缺的不是说明文字，

而是：

- 导航 seam

不是：

- 新分析页

## 为什么物理前门本身也需要提示阅读路径

### 判断二：前门和阅读路径不是重复，而是“分诊”与“连续推进”的两层

物理前门负责：

- 把 `/permissions`
- `/plan`
- EnterWorktree
- `/add-dir`

先分清。

阅读路径负责：

- 按稳定顺序把这条线一次读通

如果前门页不提示路径层，

读者仍然会把前门误当成：

- 这条线唯一的导航页

然后在深页之间自己重新拼顺序。

所以更稳的收口是：

- 顶层目标入口先交给阅读路径
- 物理前门保留一个很短的 handoff，承接只想先做第一跳分诊的读者

## 为什么这轮只改两处导航，不继续扩别的页

### 判断三：当前缺的是“你已经到入口了，下一步去哪”，不是“你还不知道这个入口存在”

顶层 README 已经有：

- `安全放大工作面，而不是一上来把行动半径开到最大`

前门页也已经存在。

阅读路径 14 也已经存在。

这说明对象层和目录层都已经齐。

还缺的只是：

- 顶层目标入口是否把首跳正式交给路径层
- 前门页是否把连续推进交给路径层

所以这轮不该再去改：

- `00-阅读路径` 的顺序
- `04-专题深潜/README`
- `05-控制面深挖/README`
- 更多正文页

只做最短的 handoff 提示就够了。

## 为什么这轮要把判断沉到记忆

### 判断四：以后每补一个物理前门，都要追问它有没有接到阅读路径层

前几轮已经说明：

- 物理前门解决第一跳
- 独立路径解决连续推进

这轮进一步说明第三件事：

- 顶层入口和前门页还要明确告诉读者“这条线已经有独立路径”

否则用户仍会在第一跳之后掉回手工搜索。

所以后续继续深入时，

每新增一个高频前门，都应补一个固定自问：

- 顶层入口有没有指向它
- 前门有没有 handoff 到独立阅读路径

## 苏格拉底式自审

### 问：我现在缺的是更多正文，还是缺一个“读完前门后下一步去哪”的提醒？

答：如果入口、前门、路径都已经存在，缺的就是接缝。

### 问：我是不是又想开新页，来解决一个本质上只是导航连续性的问题？

答：如果对象层已经齐，再开新页只会制造更多入口，不会减少摩擦。

### 问：我是不是只把入口做到了“看得见”，却没做到“继续读得下去”？

答：如果用户还要自己回头翻阅读路径，那就是。
