# front-door link target repair 拆分记忆

## 本轮继续深入的核心判断

这一轮最值得收的，

不是继续改 taxonomy，

而是修掉前门导航层里已经真实断开的链接目标。

## 为什么这轮值得单独提交

### 判断一：这是导航可用性问题，不是文风问题

并行只读审查指出：

- `00-阅读路径.md`
- 根 `README.md`
- `05-控制面深挖/README.md`
- `03-参考索引/01-命令工具/README.md`

这几层仍残留一批旧 href：

- 有些还指向旧标题版本
- 有些相对路径层级写浅了一层

这些问题会直接让前门继续阅读链断掉，

因此优先级高于继续润色术语。

## 这轮具体怎么拆

### 判断二：只修真实断链，不顺手改正文叙事

这轮分两类修复：

1. 对 `00-阅读路径.md`、根 `README.md`、`05-控制面深挖/README.md`
   使用受限规则修复：
   - 只有当链接文字本身就是现存 `.md` 路径
   - 且当前 href 解码后落不到文件
   才把 href 改成现有文件路径
2. 对 `03-参考索引/01-命令工具/README.md`
   手工把：
   - `../00-阅读路径.md`
   改成：
   - `../../00-阅读路径.md`

这样能把：

- 旧标题残留
- section README 回跳层级错误

一起收掉，

但不会把这轮扩大成全书通扫。

## 为什么这轮不把所有坏链一次扫完

### 判断三：当前只修前门导航层，避免把正文层海量长链一起卷进来

更深层页面也可能还有旧 href，

但这一轮只处理：

- 用户最先进入的前门层
- section-level hub 回跳

先把高频导航面恢复可用，

再决定是否需要单独做正文层链接整修。
