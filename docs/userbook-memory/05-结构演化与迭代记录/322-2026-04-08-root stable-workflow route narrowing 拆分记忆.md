# root stable-workflow route narrowing 拆分记忆

## 本轮继续深入的核心判断

这一轮最值得收的，

不是继续压缩根 `README` 整个“按目标进入”区，

而是把：

- `建立稳定工作流`

这一条

从“具体入口后又回到两个大 hub”

收回成：

- 具体入口
- 已存在的精确阅读路径

这两跳。

## 为什么这轮值得单独提交

### 判断一：前门如果承诺“按目标进入”，就不该把读者重新送回高层分诊

根 `README` 当前这条写法是：

- `01-主线使用/05-开工前自检...`
- `04-专题深潜/README.md`
- `05-控制面深挖/README.md`

问题不在于这些页不重要，

而在于：

- 第一跳已经是具体目标页
- 第二、三跳却重新退回两个 section hub

这会让“按目标进入”变成：

- 先给一个具体入口
- 再把你送回大门口重新选路

### 判断二：这里已经有更准确的现成路径，不需要再发明新路由

`00-阅读路径.md` 已经存在：

- `路径 3：我想建立稳定工作流`

而且它给的是一条按稳定工作流展开的宽路径，

比直接把读者丢进 `04` 和 `05` 的两个总 README 更准确。

所以这一刀最小、最稳的做法不是重写稳定工作流条目，

而是把根前门和现成路径对齐。

## 这轮具体怎么拆

### 判断三：保留最小起步页，再把扩展阅读交给“路径 3”

这轮只做一件事：

1. 保留第一跳：
   - `01-主线使用/05-开工前自检：状态、诊断、额度与工作边界.md`
2. 把后面的：
   - `04-专题深潜/README.md -> 05-控制面深挖/README.md`
   收成：
   - `00-阅读路径.md` 里的“路径 3：我想建立稳定工作流”

这样：

- 想先落地的人，仍能直接从具体起步页开始
- 想继续展开完整稳定工作流的人，会被送到已经写好的精确路线
- 根前门不再自己承担二次分诊

## 为什么这轮不顺手调整其它“按目标进入”条目

### 判断四：当前最明确的问题是“目标入口后又回大 hub”，而不是整块前门都失效

根 `README` 的“按目标进入”区仍有其它可继续收紧之处，

但这轮最清楚、最独立的一刀，

是：

- 修掉一个已经有现成替代路径、且误导最明显的条目

如果把其它目标入口一起改，

这轮就会从：

- 单条路由纠偏

膨胀成：

- 前门整体重写

不符合当前“小切片、低扰动、可提交”的纪律。
