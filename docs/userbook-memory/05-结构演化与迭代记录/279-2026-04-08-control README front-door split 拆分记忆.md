## 为什么这轮落点选 `05-控制面深挖/README.md`

### 判断一：控制面 README 也需要让读者分清“高价值主线”与“完整目录”

这页当前已经明确：

- 它按控制面组织
- 不承担命令速查

但差异说明之后立刻进入长列表，

读者看不出：

- 哪些是优先控制面
- 哪些开始转入完整目录

这会让 control README 在 first-hop 和 full inventory 之间继续混层。

## 为什么这轮要补范围说明

### 判断二：控制面页不该承担工作目标 first-hop

`05`

比 `04`

更进一步：

- 它不是工作目标分诊
- 它也不是命令速查

所以这轮要显式写出：

- 还没分清工作目标时先回根 README
- 主要缺工作主题 first-hop 时先回 `04-专题深潜/README`

不把这层范围说清，

读者就会继续把：

- 控制面页

当作：

- 万能首跳页

## 为什么这轮只加两个标题和一句范围说明

### 判断三：当前缺的是导航角色信号，不是重写控制面索引

这轮的问题不在于：

- 具体控制面页有误

而在于：

- README 没有把“优先读哪些”与“完整目录”分开

所以最小且最稳的一刀仍然是：

- 补范围说明
- 补 `优先控制面`
- 补 `完整控制面目录`

## 为什么把 `16/18/19` 那组导读保留在两层之间

### 判断四：它更像从优先控制面过渡到完整目录的桥，而不是纯目录条目

`16/18/19`

这组说明本来就在做：

- 局部阅读链引导

它既不是纯 first-hop，

也不是完整目录本身。

所以更合理的位置是：

- 放在 `优先控制面` 之后
- `完整控制面目录` 之前

让它承担桥段，而不是吞掉任一层角色。

## 苏格拉底式自审

### 问：我现在缺的是再补控制面叶子，还是先让控制面 README 像一个分层目录？

答：如果叶子本身已经很多，当前更缺的是分层目录信号。

### 问：我是不是又把控制面 README 当成工作目标 first-hop？

答：如果读者还没分清工作主题，应该先回根 README 或 `04`，而不是直接压进 `05`。

### 问：我是不是把 `16/18/19` 这组阅读链说明当成普通目录条目？

答：这组更像桥段，应该被夹在优先控制面和完整目录之间，而不是和任一层混成一页平表。
