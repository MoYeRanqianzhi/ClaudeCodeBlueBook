## 为什么 `03` 和 `04` 需要互导

### 判断一：一张回答启动形态，一张回答同名入口错位

`03-CLI 旗标与根命令索引`

回答的是：

- 进程启动层有哪些 root flags / root commands / fast-path

`04-根命令与斜杠命令对照索引`

回答的是：

- 名字相近的 root / slash 入口为什么不在同一层

如果没有互导，

读者会在两种误判间来回：

- 来 `03` 找同名入口错位
- 来 `04` 找启动形态分流

所以这轮补的不是新内容，

而是：

- 问题类型到索引页的映射句

## 为什么这轮不再改专题 18/19

### 判断二：专题层已经闭环，当前缺的是索引层的对象分流

专题 18 已经会提示：

- 如果你真正卡住的是同名入口不在一层，去 19

专题 19 也已经把 18 放进交叉阅读。

这说明专题层已经能互相送达。

而索引层原先还没有这层对象区分句，

所以更值钱的是先收：

- index-to-index routing

而不是继续给专题页加解释。

## 为什么这轮仍然是低风险

### 判断三：只加顶部路由句，不碰索引表本体

这轮没有改：

- 任何表格项

只改：

- 两张索引页开头的路由句

所以它属于：

- cross-routing repair

不是：

- 索引重构

## 苏格拉底式自审

### 问：我现在缺的是新索引，还是缺一句“你问的是另一层问题，请去另一页”？

答：如果两张索引都已存在，缺的通常只是互导。

### 问：我是不是把“启动形态分流”和“同名入口错位”误当成一张索引能同时讲完的对象？

答：如果对象不同，就该用两张索引加互导，而不是一张页里继续堆。

### 问：我是不是又想改表格明细，而当前真正缺的只是读者进入索引页后的第一句分流？

答：如果表格已经对，先补第一句分流。
