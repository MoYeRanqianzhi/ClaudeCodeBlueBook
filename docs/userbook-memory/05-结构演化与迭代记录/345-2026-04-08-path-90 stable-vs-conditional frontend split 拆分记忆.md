# path-90 stable-vs-conditional frontend split 拆分记忆

## 本轮继续深入的核心判断

这一轮不是再扩写多前端接续的能力面，

而是先把：

- `路径 90`

里的稳定接续路线

和：

- `06-前端与远程专题.md`

所承载的更宽口径条件前端/灰度远端面

重新拆开。

## 多 Agent 交叉结论

这一刀直接吸收了三条并行只读分析：

1. `Hegel` 指出当前最需要继续显式化的结构，
   不是再堆更多叶子页，
   而是让：
   - 输入 / 控制 / 执行 / 反馈 / 恢复 / 治理
   这几层更能被一次复述；
   其中“反馈面”最散，
   “控制 vs 治理”仍容易重叠，
   “恢复对象”也还需要继续分型。
2. `Gibbs` 指出当前 userbook 最大的问题不是目录名不对，
   而是：
   - first-hop owner 不唯一
   - 根前门、导读、路径页、专题 README、控制面 README 都还在争第一跳
3. `Boole` 则从稳定/灰度边界上指出，
   `路径 90`
   默认把：
   - `06-前端与远程专题.md`
   放进顺序，
   会让 `--remote`、`/voice`、`/chrome`、受账户/平台前提限制的 `/desktop`
   这类条件面
   混进“继续同一工作”的稳定路线。

这三条结论合起来，

把当前最小可落地修正压缩成一句话：

- 稳定 continuation route 不该默认吞下条件前端总览

## 为什么这轮值得单独提交

### 判断一：`路径 90` 的标题承诺的是“继续同一工作”，不是“前端与远端能力大全”

`路径 90`

真正应该先帮用户做的是：

- 区分 IDE 连接、Desktop handoff、移动端下载和 remote session viewer 的对象差异
- 避免把 `/session`、`/remote-control`、`/mobile` 混写成同一种“远程入口”

这是一条：

- 稳定 continuation / host-object 分型路线

而不是：

- 条件前端 + 远端能力总表

### 判断二：`06-前端与远程专题` 宽口径过大，默认放进路径 90 会稀释稳定功能与灰度功能的边界

`Boole`

指出，

`06`

这一页会同时覆盖：

- `--remote`
- `/voice`
- `/chrome`
- `/desktop`
- 更宽的前端/远端总览

其中多项不是：

- 默认稳定 continuation 面

而是：

- 隐藏 / Beta / 条件公开 / 账户或平台前提受限能力

如果把它继续放在 `路径 90` 默认顺序里，

就会让读者误以为：

- 这些条件面也是“继续同一工作”时默认该看的稳定对象

### 判断三：把 `06` 降成条件分支，比继续在正文里补免责声明更稳

当前最小有效修正不是继续在 `06`

里加更多“注意这是灰度面”的句子，

而是先从路径编排上承认：

- 这不是默认第 3 步

只有当读者确实想看：

- 更宽口径的前端/远端总览

时，

再把它作为条件化二跳放出来。

## 这轮具体怎么拆

这轮只做两件事：

1. 把 `路径 90` 默认顺序改成：
   - `多前端接续与条件远端/README`
   - `16-IDE、Desktop、Mobile 与会话接续专题`
   - `01-主线使用/01-进入系统：启动、会话与工作目录`
2. 把：
   - `06-前端与远程专题`
   改成路径末尾的条件化补充说明，
   并明确它会混入更多条件开放或灰度能力，
   不再属于“继续同一工作”的稳定默认路线

## 为什么这轮不直接去做更大的“反馈面 / 控制 vs 治理 / 恢复对象”结构重写

因为多 Agent 的 broader analysis

已经清楚指出方向，

但当前最小可提交的真实边界仍然是：

- 一条具体阅读路径里对稳定功能与灰度功能的默认混排

先把这条混排拆开，

后面再继续把：

- 反馈面升格
- 控制与治理 crosswalk
- 恢复对象分型

压成更系统的目录演化提交，

粒度会更稳。

## 苏格拉底式自审

### 问：为什么 `06-前端与远程专题` 不能继续留在默认顺序里，只靠正文提醒它有条件面？

答：因为路径顺序本身就在传递“默认应该先看什么”。只要它还在默认步骤里，免责声明再多，也挡不住读者把它听成稳定 continuation 路线的一部分。

### 问：把 `06` 降成条件分支，会不会损失总览能力？

答：不会。总览仍然保留，只是退出默认前门，改成按需展开。

### 问：这一刀和前两刀的根入口收窄是什么关系？

答：前两刀是在根前门层恢复“先走路径”；这一刀是在路径内部恢复“先看稳定接续对象，再按需扩到条件前端总览”。
