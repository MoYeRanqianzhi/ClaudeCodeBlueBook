# startup entry top-level lift 拆分记忆

## 本轮继续深入的核心判断

在修完 `00-阅读路径.md` 的编号冲突以后，

“启动入口分流”这条线当前真正还缺的不是：

- 新物理前门
- 新独立阅读路径
- 新专题页

而是：

- 顶层 `README` 的目标入口

因为这条线其实已经有：

- `路径 14：我想分清 root flags、root commands 和 slash commands`
- `路径 15：我想分清会外 root commands 和会内面板`
- `04-专题深潜/18`
- `04-专题深潜/19`
- 两张参考索引页

所以这轮更稳的动作不是继续加层，

而是：

- 让用户在顶层就看见这条线存在

## 为什么这轮不再造新前门

### 判断一：这条线和 safe action-radius 不同，它缺的不是对象分诊页

`safe action-radius` 当时的问题是：

- `/permissions`
- `/plan`
- EnterWorktree
- `/add-dir`

虽然都很高频，

但用户没有一个能先分清四者对象差异的前门。

而启动入口这条线不同。

这里已经有：

- `03-参考索引/01/03`
- `03-参考索引/01/04`
- `04-专题深潜/18`
- `04-专题深潜/19`
- `00-阅读路径` 的路径 14 / 15

也就是说，

对象分诊页和连续阅读顺序其实都已经存在。

还没被补上的，只是：

- 顶层“按目标进入”看不见这条线

## 为什么顶层入口应该先交给路径 14，而不是直接交给专题 18

### 判断二：用户在顶层问的是“我该先从哪一类对象分起”，不是“我先看哪一篇专题”

`04-专题深潜/18`

本身已经很深，

它更适合：

- 你已经知道自己在看启动层

但顶层入口要回答的是：

- 启动层和会话层怎么先分
- root flags / root commands / slash commands 怎么先分

这更像阅读路径的职责。

所以顶层入口应先落到：

- `路径 14`

而不是直接丢给专题页。

## 为什么还要顺带提 `路径 15`

### 判断三：不少用户真正卡住的是 root command / slash command 对照，而不是启动 flags 本身

从用户症状看，

最常见的启动线误判并不只有：

- `--resume` vs `/resume`

还有：

- `claude auth login` vs `/login`
- `claude mcp ...` vs `/mcp`
- `claude plugin ...` vs `/plugin`
- `claude doctor` vs `/doctor` `/status`

这些更直接落在：

- 会外 root command
- 会内面板

的错位上。

所以顶层入口在指向 `路径 14` 的同时，

补一句：

- 如果你主要卡在同名入口不在一层，继续看 `路径 15`

收益很高，但改动很小。

## 为什么这轮仍然是低 churn

### 判断四：只加一个顶层 goal-entry，不动现有阅读路径和专题页

这轮没有改：

- `00-阅读路径.md`
- `04-专题深潜/18`
- `04-专题深潜/19`
- 任何新的目录结构

只补：

- 顶层 README 的一条目标入口

所以这轮是：

- top-level discoverability lift

不是：

- 新的结构工程

## 苏格拉底式自审

### 问：我现在缺的是再开一个前门，还是只缺让用户在顶层看见现有路径？

答：如果路径和专题都已有，缺的是顶层发现性。

### 问：我是不是又想用新目录去解决一个本质上只是顶层 README 少一条 bullet 的问题？

答：如果只差第一跳，新目录只会制造重复。

### 问：我是不是把“启动 flags 分流”和“同名 root/slash 对照”混成一层了？

答：顶层入口可以先给路径 14，再顺手把路径 15 作为次级分流提示，但不应把两条线压成一页。
