# 第一性原理：Claude Code 到底是什么

如果把所有 UI、品牌词和灰度功能都剥掉，Claude Code 的内核可以压成五句话。

## 1. 它是一个输入编译器

用户输入、附件、命令、技能、hooks、IDE 选区、记忆，并不是原样送给模型，而是先被整理成请求对象。

## 2. 它是一个工具调度器

模型面对的是一个被权限、模式和 feature gate 过滤后的工具池，并通过主循环反复调用工具推进任务。

## 3. 它是一个治理系统

权限模式、计划模式、deny 规则、沙箱、托管策略与远程开关决定了模型能在多大边界内行动。

## 4. 它是一个连续性系统

压缩、恢复、记忆、标签、导出、任务输出与后台任务把一次对话升级成可持续工作现场。

## 5. 它是一个可扩展宿主

技能、插件、MCP、hooks、worktree、子代理、团队、远程执行把 Claude Code 变成一个宿主运行时，而不是单机聊天器。

## 由此得到的使用结论

### 不要把它当成纯聊天工具

因为那会错过它最重要的执行面和治理面。

### 不要把它当成单模型自动驾驶

因为它的价值不只来自模型强度，更来自正确入口、正确工具和正确边界。

### 最佳使用姿势是“任务编译”

把任务送进最合适的入口，让运行时替你维护：

- 上下文。
- 工具。
- 安全边界。
- 连续性。
- 并行与隔离。
