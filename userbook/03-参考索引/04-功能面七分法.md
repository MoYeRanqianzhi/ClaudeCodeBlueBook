# 功能面七分法

基于当前 `v2.1.88` 反编译源码，Claude Code 的功能面更稳妥的收口方式是七类，而不是单纯“命令 + 工具”。

## 1. 命令面

这一层回答“用户从哪里进入系统”。包括：

- 会话与导航
- 上下文与连续性
- 运行时配置
- 扩展管理
- 代码工作流
- 设备与前端
- 账户与观测
- 远程与桥接
- 杂项与灰度入口

权威注册点：

- `src/commands.ts`
- `src/types/command.ts`

## 2. 工具面

这一层回答“模型到底能做什么动作”。包括：

- 文件与搜索
- shell 执行
- Web
- Notebook / LSP / StructuredOutput
- 计划模式 / worktree / 任务 / 团队 / 协作
- MCP 资源与 Skill tool

权威注册点：

- `src/tools.ts`
- `src/Tool.ts`

## 3. 技能面

这一层回答“哪些工作流已经被做成正式可复用对象”。包括：

- bundled skills
- 项目技能目录
- 插件技能
- 动态技能
- MCP skills

权威注册点：

- `src/skills/bundled/index.ts`
- `src/skills/bundledSkills.ts`
- `src/skills/loadSkillsDir.ts`

## 4. 设备 / 远程面

这一层回答“Claude Code 可以在哪些前端和宿主环境中继续工作”。包括：

- IDE
- Desktop
- Mobile
- session / QR / URL
- remote-control / bridge
- remote env
- voice
- chrome

代表文件：

- `src/commands/ide/ide.tsx`
- `src/commands/session/session.tsx`
- `src/commands/voice/voice.ts`
- `src/bridge/bridgeMain.ts`

## 5. 代理 / 任务 / 调度面

这一层回答“如何把行动放大成长期执行和并行执行”。包括：

- AgentTool
- Task*
- `/tasks`
- Team*
- SendMessage
- Cron*
- RemoteTrigger
- coordinator/swarm
- worktree 隔离执行

代表文件：

- `src/tools/AgentTool/AgentTool.tsx`
- `src/utils/tasks.ts`
- `src/coordinator/coordinatorMode.ts`
- `src/utils/worktree.ts`

## 6. 扩展面

这一层回答“如何把 Claude Code 接到外部能力与外部流程”。包括：

- skills
- plugins
- MCP
- hooks

代表文件：

- `src/plugins/builtinPlugins.ts`
- `src/commands/plugin/plugin.tsx`
- `src/commands/mcp/mcp.tsx`
- `src/services/mcp/client.ts`
- `src/utils/hooks/AsyncHookRegistry.ts`

## 7. 灰度 / 内部面

这一层回答“哪些能力虽然可见，但不该直接写进普通用户稳定主线”。包括：

- feature-gated 命令和工具
- ant-only 能力
- undercover mode
- remote managed settings / killswitch
- 被 DCE 消除后只剩门控影子的内部模块

代表文件：

- `src/commands.ts`
- `src/tools.ts`
- `src/utils/undercover.ts`
- `claude-code-source-code/README_CN.md`

## 使用方式

### 先问自己当前在看哪一面

是用户入口、模型行动、工作流复用、设备接续、并行执行、外部扩展，还是灰度边界。

### 不要把七面混成一张表

这样写出来的 userbook 才不会把：

- 命令和工具混写。
- 稳定面和灰度面混写。
- 用户入口和内部执行对象混写。
