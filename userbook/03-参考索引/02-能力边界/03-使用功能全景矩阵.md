# 使用功能全景矩阵

下表不是按源码模块排，而是按用户任务排。目的是帮助你在真实工作中快速选入口。

| 用户目标 | 首选入口 | 次选入口 | 稳定性 |
| --- | --- | --- | --- |
| 探索代码结构 | 普通提示词 + `Read/Grep/Glob` | `LSP` | 稳定，LSP 条件 |
| 修改文件 | 普通提示词 + `Edit/Write` | `Bash` | 稳定 |
| 看当前变更 | `/diff` | `git diff` via `Bash` | 稳定 |
| 审查 PR | `/review` | `ultrareview` | `/review` 稳定，`ultrareview` 灰度 |
| 生成提交 | `/commit` | 手工 git | 稳定 |
| 管理权限 | `/permissions` | settings / config | 稳定 |
| 先设计后实施 | `/plan` / `EnterPlanMode` | 普通提示词 | 稳定，部分会话上下文有限制 |
| 隔离改动面 | `EnterWorktree` | 手工 git worktree | 稳定，依赖 Git/worktree 上下文 |
| 长任务后台跑 | `Agent` + `Task*` + `/tasks` | 主线程单跑 | 部分稳定，部分条件 |
| 并行代理协作 | `Agent` + `Team*` + `SendMessage` | 多会话手工协作 | 灰度更重 |
| 管理记忆 | `/memory` | 直接编辑 `CLAUDE*.md` | 稳定 |
| 压缩上下文 | `/compact` | 新开会话 | 稳定 |
| 恢复旧会话 | `/resume` | 手工找 session 文件 | 稳定 |
| 扩展工作流 | skills | plugin | 稳定 |
| 接入外部系统 | MCP | plugin + hooks | 稳定但依赖配置 |
| 在关键事件插钩子 | hooks | 外部 wrapper | 稳定 |
| 连接 IDE | `/ide` | CLI flag / auto-connect | 稳定 |
| 远程接续 | `/session` / `/remote-control` | web 侧入口 | 部分稳定、部分灰度 |
| 语音输入 | `/voice` | 无 | 灰度/条件 |
| 浏览器侧集成 | `/chrome` | MCP/browser tool | 灰度/条件 |

## 这张矩阵的用法

### 看主线

如果目标属于稳定列，就优先走稳定面，不要先上灰度面。

### 看边界

如果某项功能落在“条件公开但已成形”或“灰度更重”，就应在文档、团队约定和自动化中单独标注。

### 看替代路径

Claude Code 很多能力并不是“有或没有”，而是“有推荐路径和退化路径”。

## 用户层结论

Claude Code 的真正强项不是单个命令有多炫，而是：

- 几乎每种工作目标都存在一个更低协调成本的正式入口。
- 当正式入口不可用时，通常还能退化到 prompt、shell、手工工作流。
