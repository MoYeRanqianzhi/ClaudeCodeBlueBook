# 三张控制面总图：世界进入模型、扩张定价与防过去写坏现在

Claude Code 最难被抄走的不是功能全集，而是这三条设计母线。

## 0. 三条母线

如果把 Claude Code 继续压缩到最少，它真正持续回答的只有三件事：

1. 世界如何进入模型
2. 扩张如何被定价
3. 当前如何不被过去写坏

这三件事分别对应三张控制面图：

1. 请求装配控制面
   - 对象镜像见 `architecture/82`
2. 反扩张治理控制面
   - 对象镜像见 `architecture/83`
3. 当前真相与反僵尸控制面
   - 对象镜像见 `architecture/84`

也就是说，Claude Code 真正的高级感不是：

- 它会更多功能

而是：

- 它先把这三个问题制度化了

`09` 也只保留这三条母线的宪法级 first answer、最短动作链与 cross-line mirror；它不再在总图页补机制 inventory、治理尾链模板或源码质量 ladder。

如果再往上压一层，这三张图现在也只各自保留一条宪法级判断：

1. 请求装配线
   - `same-world test` 是否仍成立：later consumer 仍不必重答谁在定义世界、边界内哪些动作和工具仍合法，也不必把已排除路径重新拉回候选集
2. 治理线
   - 最早 `unpaid expansion` 是否仍被同一条 pricing order 单源定价；更细 `typed-state checkpoint law / verb ceiling` 统一回 `10`
3. 当前真相线
   - 当前是否仍先被正式保护；更细 `promotion / downgrade / unresolved-authority note` 统一回 `guides/102` 与 `architecture/84`

这里也只再保留一条 continuity crosswalk：

- `continuity` 不是第四张控制面图；它只是三张图在时间维度上的共同验收面。
- 第一张图回答 witness chain 是否仍成立。
- 第二张图回答 continuation pricing 是否仍愿意为这次继续收费。
- 第三张图回答恢复资产、旧写入者与 replay 路径是否仍没篡位 current truth。

如果继续把这三张图压成最短动作链，它们还可以再写成：

1. 请求装配线
   - `compile -> verify -> enforce -> continue -> explain`
2. 治理线
   - `price -> externalize -> arbitrate -> continue-or-stop -> cleanup`
3. 当前真相线
   - `surface -> writeback -> freshness -> evict`

一旦高阶总论页不能再压成这三条动作链，说明解释还停在对象层总结，没有真正回到设计宪法。

如果再往目录入口压一层，这三张图也只该继续暴露三条 first-answer hooks：

1. 请求装配线
   - 只先问 later consumer 是否仍不必重谈世界、重搜动作、重判继续资格；why 与 verdict box 回 `84`，builder-facing object chain 再回 `81`
2. 治理线
   - 只先问扩张是否仍被同一条 pricing order 单源定价；更细治理链统一回 `10`
3. 当前真相线
   - 只先问当前是否先被正式保护；evidence grading / why / object seam 统一先回 `guides/102`

如果 later maintainer 第一次进入 `09` 时还看不见这三条，解释就还停在对象层，总图还没有真正变成入口宪法。

如果 later maintainer 已经开始寻找每条母线的具体 reject path、failure order 或排查顺序，本页就已经不该继续下沉；那类分流统一回 `navigation/README` 与各归属页，本页只保留首答 hook 与 cross-line mirror。

## 0.1 这三张图不是高级观察，而是设计宪法

这三张图不是读后总结，而是设计宪法。

1. 它们不是专题并列关系，而是后续一切功能与目录都要回来的宪法层。
2. 少了第一张图，Prompt 会退回更像真相的文案。
3. 少了第二张图，安全与省 token 会退回彼此分裂的局部优化。
4. 少了第三张图，源码质量会退回更漂亮的分层截图与目录体感。

所以这三张图真正值钱的地方，不是解释功能，而是规定：

- 什么问题配先成为先验约束
- 什么扩张配先进入当前世界
- 什么旧状态、旧入口与旧写入者永远不配重新篡位当前世界

这三张图共享的制度敌人也只剩一个：重判资格回流。凡是展示层、消费面、恢复资产或局部文本又重新拿回改判世界、免费续租扩张或篡位当前的资格，系统就会从制度能力退回协商。

## 1. 第一张图：世界如何进入模型

Claude Code 第一张图真正强的不是：

- prompt 更会说话

而是：

- `world entry / request assembly / six-stage assembly chain` 先把世界合法编译进模型

Anthropic 官方文档常把 Claude Code 写成 `gather context -> take action -> verify results` 的 agentic loop；但在总图页里更稳的结论只有一条：

- 第一张图只先问 later consumer 是否仍不必重谈世界、重搜动作、重判继续资格；why 与 verdict box 去 `84`，builder-facing object chain 去 `81`

如果 `handoff / compact / resume` 之后，later consumer 还要重谈世界、重搜动作，或把已退役判断重新拉回候选集，说明你其实还没离开 `84`。

离开这一步，Prompt 效力就只剩“会开场”，不再是可闭环的 same-world runtime。

如果继续把第一张图压得更硬，也只保留一句：Prompt 的本体不是更长文案，而是让世界在进入、压缩、分叉与交接后仍保持同一现场；why 留在 `84`，builder-facing 机制链再交给 `81`。

如果这条流水线不存在，系统再会写 Prompt，也只是在堆更像真相的说明书。

## 2. 第二张图：扩张如何被定价

Claude Code 真正成熟的安全设计，不是更会拦截，而是先把未定价扩张挡在当前世界之外。

在总图页，这里只保留三条治理宪法句：

- 安全与省 token 共享同一条 pricing order；前者在问什么动作不配免费进入，后者在问什么字节、上下文与续租不配免费留在当前世界。
- `approval / modal / status / usage / compact` 这类 surface 没有新增 `decision delta` 时，一律只按 `zero-delta ask / weak readback / receipt-grade evidence` 读取。
- 若你已经开始追 `pricing-right / repricing proof / cleanup authority / human fallback`，说明问题已离开总图，应回 `10 / philosophy/61 / security`。

这也意味着第一张图和省 token 并不冲突；`stable prefix / dynamic boundary` 本来就是把世界准入写进同一价格秩序。

也就是说，Claude Code 真正在做的不是“限制模型”，而是：

- 让模型只能在已授权、已外化、已定价的当前世界里行动

## 3. 第三张图：当前如何不被过去写坏

Claude Code 的源码先进性，不是目录更整齐，而是先把“谁配宣布现在、过去为何不配篡位现在”写成结构约束。

在总图页，这里也只保留三句 current-truth 宪法句：

- 当前必须先被正式保护；过去写入者、恢复资产与 replay 路径都不配越权写回 `present`。
- future maintainer 必须继续拥有正式 reject 能力，所以“文件更小 / 目录更整齐”都不自动构成 advancedness 证据。
- 若你已经开始追 `claim-state / landing card / fail-closed seam`，说明问题已离开总图；统一回 `guides/102 / philosophy/86 / philosophy/87 / architecture/README`。

## 4. 为什么这三张图必须先于功能解释

如果把它们降成后置总结，Claude Code 就容易继续被误读成：

1. Prompt 更强
2. 安全更严
3. 代码更漂亮

更成熟的解释顺序应该先声明：

1. 它首先在编译世界
2. 它首先在为扩张定价
3. 它首先在防止过去写坏现在
4. 它还分别给出了这三件事各自的判断标准，而不是只给出三组实现细节

只有把这三件事前置，后面的架构、API、哲学、指南、案例和演练才不会重新写散。

这也是为什么三张控制面之后还仍可能需要 `navigation/README`：

1. 只有当问题已经变成 route dispute，且主语 / claim-state 已知，才再进入导航层。
2. `navigation` 只指出下一种 artifact 或最近的 fail-closed seam，不在总图页预排固定导航顺序。

如果少了这三步，三张图本身也会重新退回漂亮总结。

## 5. 如果从第一性原理重新设计 Agent Runtime

如果今天从零开始设计一台 Agent runtime，最先问的应当不是：

- 有哪些工具
- 能不能多 Agent
- 支不支持远程

而应当先问：

1. 现在的世界是怎样被合法编译进模型的
2. 哪些扩张要立刻收费、哪些扩张必须延后、哪些扩张必须迁出主工作集
3. 哪些旧写入者、旧状态、旧入口绝不能重新写坏现在

如果这三件事没有先成立，后续功能再多，也只是更厚的外观。

## 6. 苏格拉底式追问

在你准备宣布“我已经抓住了 Claude Code 的设计内涵”前，先问自己：

1. 我描述的是功能，还是世界进入模型的方式。
2. 我描述的是更多限制，还是扩张如何被正式定价。
3. 我描述的是分层更漂亮，还是过去怎样被正式阻止写坏现在。
4. 这些问题能不能直接拿去审一个新 runtime，而不是只拿来赞美 Claude Code。
5. 如果把功能都先删掉一半，这三张控制面还在不在。
6. 我复制的是这三张图里的约束链与判断顺序，还是只复制了措辞、面板与目录外观。

## 7. 一句话总结

Claude Code 最难被抄走的部分，不是更多功能，而是三张控制面图：它先规定世界怎样进入模型，再规定扩张怎样被定价，最后规定过去怎样不得写坏现在。
