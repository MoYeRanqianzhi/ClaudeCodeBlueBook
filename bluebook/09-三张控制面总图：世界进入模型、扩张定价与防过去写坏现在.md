# 三张控制面总图：世界进入模型、扩张定价与防过去写坏现在

这一章回答五个问题：

1. 为什么 Claude Code 最难被抄走的部分，不是功能全集，而是三张控制面图。
2. 为什么 Prompt 魔力、安全/省 Token 统一设计、源码先进性最后会收敛成三个更不可约的问题。
3. 为什么这三张图应当前置到主线，而不是只留在深层专题里。
4. 如果我要设计一个新的 Agent runtime，最先该回答哪三件事。
5. 怎样用苏格拉底式追问判断自己是在复制制度，还是在复制外观。

## 0. 先说结论

如果把 Claude Code 继续压缩到最少，它真正持续回答的是三件事：

1. 世界如何进入模型
2. 扩张如何被定价
3. 当前如何不被过去写坏

这三件事分别对应三张控制面图：

1. 请求装配控制面
   - 见 `architecture/82`
2. 反扩张治理控制面
   - 见 `architecture/83`
3. 权威面与反僵尸控制面
   - 见 `architecture/84`

也就是说，Claude Code 真正的高级感不是：

- 它会更多功能

而是：

- 它先把这三个问题制度化了

## 1. 第一张图：世界如何进入模型

Claude Code 的 Prompt 魔力，真正强的不是：

- 写得更会说话

而是：

- 世界进入模型之前，已经先被编译过一轮

这里至少包含六个正式对象：

1. `authority chain`
2. `section registry`
3. `dynamic boundary`
4. `protocol transcript`
5. `lawful forgetting`
6. `cache-safe forks`

这意味着 Prompt 的本体不是：

- 一段文案

而是：

- 一条请求装配流水线

如果这条流水线不存在，系统再会写 Prompt，也只是在堆更像真相的说明书。

## 2. 第二张图：扩张如何被定价

Claude Code 真正成熟的安全设计，也不是：

- 更会拦截

它更接近：

- 更会给扩张收费

这里被统一定价的至少有四个空间：

1. 动作空间
2. 权威空间
3. 上下文空间
4. 时间空间

所以安全与省 Token 并不是两套系统。

它们共同反对的是：

- 免费扩张

也就是说，Claude Code 真正在做的不是“限制模型”，而是：

- 让模型只在已定价世界里行动

## 3. 第三张图：当前如何不被过去写坏

Claude Code 的源码先进性，也不是：

- 目录更整齐
- 文件更碎

它真正先进的地方在于，它会先把这些问题结构化：

1. 当前哪一个 surface 有资格宣布真相
2. 哪些恢复资产只能帮助恢复，不能篡位
3. 哪些陈旧 finally、陈旧 snapshot、陈旧 pointer 不得写回 present state
4. 哪些编译期 / 运行期边界根本不该进入公开图谱

所以这里真正被保护的不是：

- 当前分层截图

而是：

- 未来维护者继续拒绝错误实现的正式能力

## 4. 为什么这三张图必须进入主线

如果它们只留在深层专题里，读者就容易继续把 Claude Code 误读成：

1. Prompt 更强
2. 安全更严
3. 代码更漂亮

但更成熟的主线入口应该先告诉读者：

1. 它首先在编译世界
2. 它首先在为扩张定价
3. 它首先在防止过去写坏现在

只有把这三件事前置，后面的架构、API、哲学、指南、案例和演练才不会重新写散。

## 5. 如果从第一性原理重新设计 Agent Runtime

如果今天从零开始设计一台 Agent runtime，最先问的应当不是：

- 有哪些工具
- 能不能多 Agent
- 支不支持远程

而应当先问：

1. 现在的世界是怎样被合法编译进模型的
2. 哪些扩张要立刻收费、哪些扩张必须延后、哪些扩张必须迁出主工作集
3. 哪些旧对象、旧状态、旧入口绝不能重新写坏现在

如果这三件事没有先成立，后续功能再多，也只是更厚的外观。

## 6. 苏格拉底式追问

在你准备宣布“我已经抓住了 Claude Code 的设计内涵”前，先问自己：

1. 我描述的是功能，还是世界进入模型的方式。
2. 我描述的是更多限制，还是扩张如何被正式定价。
3. 我描述的是分层更漂亮，还是过去怎样被正式阻止写坏现在。
4. 这些问题能不能直接拿去审一个新 runtime，而不是只拿来赞美 Claude Code。
5. 如果把功能都先删掉一半，这三张控制面还在不在。

Claude Code 最难被抄走的部分，不是更多功能，而是三张控制面图：它先规定世界怎样进入模型，再规定扩张怎样被定价，最后规定过去怎样不得写坏现在。
