# 事故标签体系与交叉索引：阶段、资产、症状、根因与恢复动作

这一章把样本层继续压成索引层。

它主要回答五个问题：

1. 为什么案例库最终必须有统一标签体系，而不能只靠篇名检索。
2. 怎样把 Prompt、治理、源码演化三条线放进同一套标签坐标。
3. 怎样让一次案例既能按阶段查、按资产查，也能按根因和恢复动作查。
4. 怎样让后续新增样本不破坏现有索引结构。
5. 怎样用苏格拉底式追问避免把标签体系做成目录洁癖。

## 0. 第一性原理

强样本库的关键不在：

- 收集更多故事

而在：

- 让不同故事能够落回同一组制度维度

所以统一标签体系至少要回答五个问题：

1. 事故发生在哪个阶段。
2. 它伤到了哪个制度资产。
3. 它表现成什么症状。
4. 它的根因属于哪一类失效。
5. 它最终用什么动作被恢复。

## 1. 阶段标签

```text
- design
- assembly
- runtime
- recovery
- evolution
```

解释：

- `design`：修宪、改规则、改结构时就已经埋下问题
- `assembly`：组装路径或边界编译出错
- `runtime`：运行时决策、竞速、可见性或状态机失效
- `recovery`：compact、resume、reconnect、rebuild 失败
- `evolution`：兼容层、影子层、registry 在长期演化中失真

## 2. 资产标签

```text
- section
- boundary
- sovereignty
- tool visibility
- stable bytes
- build surface
- shadow / stub
- transport shell
- recovery asset
- object state
```

资产标签回答的是：

- 系统里哪一块正式制度对象被伤到了

补充元字段：

- authority source
- assembly path
- evidence schema

它们不直接描述受损资产，但决定这个样本以后能否被稳定检索和复盘。

## 3. 症状标签

```text
- cost spike
- latency spike
- cache break
- split truth
- replay mismatch
- stale state
- wrong allow
- wrong deny
- drift
- zombie risk
```

症状标签回答的是：

- 用户或系统首先看见了什么


根因标签回答的是：

## 5. 恢复动作标签

```text
- rollback

- rebuild
- replay
- merge-against-fresh
- drop-stale
- retire-compat
- tighten-order
- restore-guard
- add-regression
```

恢复动作标签回答的是：

- 团队最终用什么动作把系统拉回合法状态

### 6.1 Prompt 样本

- `casebooks/01`
  - 根因：`section drift / boundary drift / path parity split / lawful-forgetting failure / invalidation drift`
  - 资产：`section / boundary / sovereignty / recovery asset`
  - 阶段：`design / assembly / recovery`

### 6.2 治理样本

  - 根因：`order violation / hard-guard bypass / approval-race degradation / stable-bytes drift / stop-logic failure`

## 7. 使用协议

新增一个样本时，至少回答下面五项：

```text
[ ] 阶段标签
[ ] 资产标签
[ ] 症状标签
[ ] 根因标签
[ ] 恢复动作标签
[ ] authority source / assembly path
[ ] evidence schema
```

只要这五项答不清，就说明这个案例还没有被制度化。

## 8. 苏格拉底式追问

在你准备给一个事故贴标签前，先问自己：

1. 我贴的是现象标签，还是制度根因标签。
2. 这个案例能否跨 prompt / 治理 / 结构三线被复用。
3. 我是在帮助未来检索，还是只是在给当前事故起个好听名字。
4. 这次恢复动作能否被归为可复用的制度动作。
5. 标签体系是在提高可解释性，还是在制造新的分类噪音。
