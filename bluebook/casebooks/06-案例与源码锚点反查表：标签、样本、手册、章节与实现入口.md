这一章把参考层进一步做成可反查表。

它主要回答四个问题：

1. 已经知道标签后，应该回到哪篇样本、哪份手册、哪条底盘章节。
2. 已经看到症状后，怎样最快定位到源码入口。
3. 为什么 crosswalk 比继续全文搜索更有制度价值。
4. 怎样用苏格拉底式追问避免把反查表做成死表。

## 4. 症状优先反查

如果先看到的是：

- `cache break / cost spike`：优先查 Prompt 与治理两线
- `wrong allow / wrong deny / ask 太多`：优先查治理线
- `split truth / replay mismatch / stale state / zombie risk`：优先查结构线

最稳的顺序是：

1. 先给症状贴 `casebooks/04` 标签。
2. 再回到本页找对应样本、手册和源码。
3. 最后再决定是否需要新增样本或更新标签字典。

## 5. 苏格拉底式追问

在你准备靠搜索直接跳源码前，先问自己：

1. 我已经知道这是哪一类标签了吗。
2. 我是在查一个案例，还是在查一类制度失效。
3. 我需要的是样本、手册、底盘章节还是源码入口。
4. 这个 crosswalk 是否已经足够让我少走一轮全文搜索。
