# 结构宿主修复稳态纠偏再纠偏执行反例：pointer 主语、假单源与 reconnect 责任幻觉

这一章不再回答“结构宿主修复稳态纠偏再纠偏执行该怎样运行”，而是回答：

- 为什么团队明明已经写了 recorrection card、authority-first reject order、single-source seam audit、anti-zombie evidence rebinding 与 reopen liability drill，仍会重新退回 pointer 主语、假 single-source 与 reconnect 责任幻觉。

## 0. 第一性原理

结构宿主修复稳态纠偏再纠偏执行真正最容易失真的地方，不在 `recorrection card` 有没有写出来，而在：

- current-truth surface、writer chokepoint、freshness gate 与 later reject path 是否仍围绕同一个结构真相面运作

## 1. pointer 主语：authority by pointer

### 坏解法

- 宿主、CI 与评审只要看到 pointer 还在、telemetry 还绿、`reject_verdict=steady_state_restituted`，就默认 authority 已成立。

### 为什么坏

- authority 不是“入口还活着”，而是“同一个 current-truth surface 已重新夺回主权”。
- 一旦 authority 退回 pointer 健康感，later 团队就会重新依赖作者解释补齐边界。

### Claude Code 式正解

- `reject verdict` 应先绑定同一个 `authority_object_id + current_write_path + writer_chokepoint`，再宣布 restored。

## 2. 假 single-source：truth by tidy tree

### 坏解法

- 团队虽然写了 `single-source seam audit`，但真正执行时只要目录更整洁、依赖更集中、最终结果没坏，就默认唯一真相入口已经成立。

### 为什么坏

- single-source 保护的是“谁是唯一写回入口”，不是“目录看起来更像单点”。
- 一旦 single-source 退回目录审美，结构先进性就会从单一真相制度退回目录美感。

### Claude Code 式正解

- single-source 必须围绕 `single_source_ref + writeback_primary_path + append_chain_ref + writer_chokepoint`，而不是围绕 tidy tree。

## 3. 假 lineage：恢复结果被幸运 reconnect 替代

### 坏解法

- 只要 reconnect 成功、最终结果没坏，就默认 lineage 已被复证。

### 为什么坏

- lineage reproof 保护的是恢复顺序仍合法闭合，不是“反正又连上了”。

### Claude Code 式正解

- `resume lineage + generation guard + stale finally suppression` 必须先成立，再谈 restored。

## 4. 假 anti-zombie：证据被 archive prose 替代

### 坏解法

- 团队虽然写了 anti-zombie restitution，但真正归档时只要产出一份复盘文字、几条日志截图与 archive note，就默认 zombie 风险已被复证。

### 为什么坏

- anti-zombie 保护的是“过去不会再写回现在”，不是“我们已经写了复盘”。

### Claude Code 式正解

- anti-zombie evidence 应首先是 `stale writer / duplicate / orphan` 的可审计证据，其次才是说明文本。

## 5. 假 reopen liability：liability by reconnect hint

### 坏解法

- 工单里写“以后有问题再 reconnect”“原入口还在”，却没有正式绑定 `reopen_boundary`、`return_authority_object` 与 `return_writeback_path`。

### 为什么坏

- reopen liability 不是“以后再试一次”，而是 later 团队未来沿哪条正式边界推翻当前状态。

### Claude Code 式正解

- reopen liability 应围绕 authority、writeback 与 boundary，而不是 reconnect 提示与旧入口。

## 6. 苏格拉底式追问

在你读完这些反例后，先问自己：

1. 我现在修回的是唯一 authority object，还是只把入口感重新包装了一遍。
2. 我现在保住的是 `single-source + lineage reproof`，还是更漂亮的目录与几次幸运 reconnect。
3. 我现在归还的是 anti-zombie 证据，还是一段更会解释的 archive prose。
4. 我现在保留的是 future reopen 的正式 liability，还是一句“以后再试一次”。
5. later 团队明天如果必须 reopen，依赖的是正式 boundary，还是作者口述与旧入口循环。

## 7. 一句话总结

真正危险的结构宿主修复稳态纠偏再纠偏执行失败，不是没跑 recorrection card，而是跑了它却仍在围绕 pointer 主语、假 single-source 与 reconnect 责任幻觉消费结构世界。
