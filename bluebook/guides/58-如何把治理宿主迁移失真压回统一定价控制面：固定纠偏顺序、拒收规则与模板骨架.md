# 如何把治理宿主迁移失真压回 governance key：固定纠偏顺序、拒收规则与模板骨架

治理宿主迁移纠偏真正要救回的，不是更顺眼的 mode 面板、弹窗与 token 图表，而是同一条统一定价控制面重新回到固定顺序：

1. `governance key`
2. `externalized truth chain`
3. `typed ask`
4. `decision window`
5. `continuation pricing`
6. `rollback / durable-transient cleanup`

`authority source`、`permission ledger`、`continuation gate` 与 `rollback object` 在这里只能保留为 source slot、事务证据面、pricing verdict 与 cleanup carrier。

## 1. 第一性原理

治理纠偏真正要救回的不是：

- 当前 mode 名字
- 一组弹窗截图
- 一条 token 进度条

而是：

- 围绕同一个 `governance key` 成立的外化真相、事务 ask、决策窗口、继续定价与 cleanup 边界

所以更稳的纠偏目标不是先把 UI 调到“看起来正常”，而是：

1. 先把 `governance key` 从 mode 投影里救出来。
2. 先把 `externalized truth chain` 从 UI 记忆里救出来。
3. 先把 `typed ask` 从弹窗事件里救出来。
4. 先把 `decision window` 从仪表盘里救出来。
5. 先把 `continuation pricing` 从默认继续里救出来。
6. 先把 cleanup 语义从文件动作里救出来。

## 2. 固定纠偏顺序

### 2.1 先拆掉 mode 崇拜

第一步不是补弹窗，而是：

1. 把 `permission_mode` 降回 `governance key` 的投影。
2. 恢复 `sources / effective / applied / externalized` 的统一权威视图。
3. 明确 internal mode 与 external mode 的边界。

### 2.2 再恢复 `externalized truth chain`

第二步要救回：

1. `settings.sources`
2. `settings.effective`
3. `settings.applied`
4. `pending_action`
5. `worker_status`
6. `context_usage_snapshot`

这一步不恢复，交接就仍然只剩“刚刚应该已经切好了吧”。

### 2.3 再恢复 `typed ask`

第三步要修的是：

1. `typed_decision`
2. `request_id + tool_use_id`
3. `pending_permission_ledger`
4. duplicate / orphan / cancel 的仲裁语义

不要继续做的事：

1. 不要把审批弹窗出现过当 ask 成立。
2. 不要把 `permission ledger` 当独立根对象。

### 2.4 再恢复 `decision window`

第四步要修的是：

1. `Context Usage + pending_action + state`
2. `context parity snapshot`
3. `why_now / diminishing_return_reason`

不要继续做的事：

1. 不要把 Context Usage 当漂亮仪表盘。
2. 不要把 token 百分比当治理对象。

### 2.5 再恢复 `continuation pricing`

第五步要把继续从默认行为拉回：

1. `continuation_pricing_ref`
2. `currentTurnTokenBudget`
3. `continuationCount`
4. `completionEvent`
5. `diminishing_return_reason`

这一步不恢复，时间与输出扩张就仍然是免费的。

### 2.6 最后恢复 `rollback / durable-transient cleanup`

最后才修：

1. `rollback_object`
2. `rewind_files` 与对象的映射
3. `durable_assets_after`
4. `transient_authority_cleared`
5. `re_entry_condition`

不要反过来：

1. 不要先恢复 mode 面板。
2. 不要先恢复 token 图表。
3. 不要先恢复文件回退。

## 3. 硬拒收规则

出现下面情况时，应直接拒收当前治理宿主实现：

1. `permission_mode_replaces_governance_key`
2. `externalized_truth_chain_missing`
3. `typed_ask_collapsed_to_modal`
4. `permission_ledger_missing`
5. `context_usage_detached_from_window`
6. `continuation_pricing_defaulted`
7. `rollback_not_object`
8. `transient_authority_resumed`

## 5. 苏格拉底式检查清单

在你准备宣布“治理失真已经纠偏完成”前，先问自己：

1. 我救回的是治理对象，还是 mode 面板。
2. 我的权限请求现在还是事务 ask，还是 UI 记忆。
3. `Context Usage` 在我的系统里是 decision window，还是仪表盘。
4. continuation 在我的系统里是正式 pricing，还是默认继续。
5. rollback 在我的系统里还是对象，还是文件回退。

## 6. 一句话总结

真正成熟的治理纠偏，不是把 mode、弹窗和图表修得更漂亮，而是把权限账本缺席、假窗口对齐与免费继续幻觉重新压回同一个 `governance key`。
