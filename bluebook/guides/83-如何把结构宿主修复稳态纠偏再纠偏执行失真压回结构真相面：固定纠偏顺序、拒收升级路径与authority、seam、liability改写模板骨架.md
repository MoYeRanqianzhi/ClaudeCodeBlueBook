# 如何把结构宿主修复稳态纠偏再纠偏执行失真压回结构真相面：writer chokepoint、shared reject semantics 与 liability 模板骨架

这一章不再解释结构宿主修复稳态纠偏再纠偏执行最常怎样失真，而是把 Claude Code 式结构再纠偏改写压成一张 builder-facing 模板手册。

它主要回答五个问题：

1. 为什么结构再纠偏改写真正要救回的不是“系统重新恢复秩序感”，而是同一个 `one writable present`。
2. 怎样把假 authority、假 single-source、假 lineage、假 anti-zombie 与假 reopen liability 压回固定模板顺序。
3. 哪些现象应被直接升级为 shared reject semantics，而不是继续补 recorrection、archive 与责任文案。
4. 哪些内容只配做 packet field、rewrite hint 或 mirror gap note。
5. 怎样用苏格拉底式追问避免把纠偏写成“把系统再纠偏流程再做细一点”。

## 0. 第一性原理

结构再纠偏改写真正要救回的不是：

- 更像样的 recorrection 流程
- 更完整的 archive 清单
- 更谨慎的 reopen 提示

而是：

- 同一个 `current-truth surface + writer chokepoint + freshness gate + later reject path` 继续成立

因此更稳的 builder-facing 模板顺序不是：

- 先看 pointer 有没有恢复
- 先看 telemetry 有没有转绿
- 先看 archive prose 有没有解释通

而是：

1. `current-truth surface`
2. `writer chokepoint`
3. `freshness gate`
4. `shared reject semantics`
5. `later reject path`

## 1. 哪些内容只配做模板载体

下面这些内容仍可保留，但必须降回模板载体：

1. `recorrection card`
2. `authority seam block`
3. `lineage reproof block`
4. `anti-zombie evidence block`
5. `reopen liability ticket`

它们的职责不是定义结构世界，而是：

- 帮 builder 复写同一条结构真相链

## 2. 固定模板顺序

### 2.1 先冻结假 surface

先把 pointer、sidecar、telemetry、archive prose 与“标题恢复了”的秩序感一律降为 `forbidden proxy patterns`。

### 2.2 再恢复 `current-truth surface`

至少确认：

1. `authority_object_id`
2. `authoritative_path`
3. `current_write_path`
4. `writer_chokepoint`
5. `one_writable_present_witness`

### 2.3 再封口 `single-source` 与 `writeback seam`

至少确认：

1. `single_source_ref`
2. `writeback_primary_path`
3. `append_chain_ref`
4. `side_write_quarantined`
5. `metadata_bypass_blocked`

### 2.4 再恢复 `freshness gate`

至少确认：

1. `lineage_reproof_ref`
2. `generation_guard_attested`
3. `stale_finally_suppressed`
4. `anti_zombie_evidence_ref`
5. `duplicate_or_zombie_blocked`

### 2.5 最后才回到 `later reject path`

至少确认：

1. `mirror_gap_ref`
2. `reopen_boundary`
3. `later_reject_path`
4. `reopen_required_when`

## 3. shared reject semantics

builder-facing 模板最值得固定的 reject 语义是：

1. `authority_conflict`
2. `writeback_target_ambiguous`
3. `lineage_mismatch`
4. `duplicate_or_zombie`
5. `reopen_boundary_invalid`

向上仍要回到结构线稳定的 reject trio：

1. `layout-first drift`
2. `recovery-sovereignty leak`
3. `surface-gap blur`

## 4. 可直接套用的模板骨架

### 4.1 writer chokepoint block

1. `authority_object_id`
2. `authoritative_path`
3. `current_write_path`
4. `writer_chokepoint`
5. `one_writable_present_witness`

### 4.2 freshness gate block

1. `lineage_reproof_ref`
2. `generation_guard_attested`
3. `fresh_merge_contract`
4. `stale_finally_suppressed`
5. `anti_zombie_evidence_ref`

### 4.3 later reject path block

1. `mirror_gap_ref`
2. `reopen_boundary`
3. `return_authority_object`
4. `return_writeback_path`
5. `reopen_required_when`

## 5. 苏格拉底式检查清单

在你准备宣布“结构再纠偏模板已经成立”前，先问自己：

1. 我现在保护的是 `one writable present`，还是一次更像样的流程。
2. 我现在保住的是 writer chokepoint，还是只是入口感更稳定。
3. 我现在阻止的是 stale writer，还是只是暂时没再观察到它写回来。
4. 我现在保留的是 later reject path，还是一句“以后再试一次”。
5. 如果把所有卡片、ticket 与 prose 全部藏起来，模板是否仍成立。

## 6. 一句话总结

真正成熟的结构宿主修复稳态纠偏再纠偏执行模板，不是把 recorrection、archive 与责任流程写得更像制度，而是先 `current-truth surface -> writer chokepoint -> freshness gate -> shared reject semantics -> later reject path`，最后才允许 artifacts 出场。
