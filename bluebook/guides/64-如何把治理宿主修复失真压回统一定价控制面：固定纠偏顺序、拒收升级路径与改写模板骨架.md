# 如何把治理宿主修复失真压回统一定价控制面：固定纠偏顺序、拒收升级路径与改写模板骨架

这一章不再解释治理宿主修复演练最常怎样失真，而是把 Claude Code 式治理修复纠偏压成一张可执行的 builder-facing 手册。

它主要回答五个问题：

1. 为什么治理修复纠偏真正要救回的不是更顺手的 mode 补救，而是同一个统一定价控制面。
2. 怎样把 authority 假恢复、假窗口重置与免费重入压回固定纠偏顺序。
3. 哪些现象应被直接升级为硬拒收，而不是继续观察。
4. 怎样把升级卡、rollback 模板与 re-entry ticket 重新压回治理对象骨架。
5. 怎样用苏格拉底式追问避免把纠偏写成“把审批流再做清楚一点”。

## 1. 第一性原理

治理修复纠偏真正要救回的不是：

- 当前 mode 名字
- 一组审批记录
- 一块刷新后的 usage 面板

而是：

- 围绕同一个治理对象成立的统一定价控制面

所以更稳的纠偏目标不是：

- 先把页面调到“看起来正常”

而是：

1. 先把 authority source 从 mode 投影里救出来。
2. 先把 permission ledger 从审批完成感里救出来。
3. 先把 decision window 从 usage dashboard 里救出来。
4. 先把 continuation repricing 从默认继续里救出来。
5. 先把 rollback object 从 mode/file 动作里救出来。

这些纠偏之所以同时在恢复安全设计与省 token 设计，是因为：

- 安全保护的是“允许什么”。
- 省 token 保护的是“继续多久、扩张多少”。
- Claude Code 把这两件事都压在同一个治理对象上统一定价。

## 2. 固定纠偏顺序

### 2.1 先拆掉 mode 假恢复

第一步不是补弹窗，而是：

1. 把 `permission_mode` 降回 authority source 的投影。
2. 恢复 `source / effective / applied` 的统一权威视图。
3. 明确 internal mode 与 external mode 的边界。

### 2.2 再恢复 ledger 主权

第二步要救回：

1. `typed_decision`
2. `permission_ledger`
3. `pending_permission_requests`
4. `request_id + tool_use_id`

这一步不恢复，交接就仍然只剩“刚刚应该批过了吧”。

### 2.3 再恢复 decision window

第三步要修的是：

1. `decision_window`
2. `pending_action`
3. `context_usage_snapshot`
4. generation / late response 的时态边界

不要继续做的事：

1. 不要把 usage 百分比当治理对象。
2. 不要把压缩次数当窗口解释。
3. 不要把“看起来还能跑”当窗口结论。

### 2.4 再恢复 continuation repricing

第四步要把继续从默认行为拉回：

1. `continuation_gate`
2. `token_budget_result`
3. `completion_event`
4. `re_entry_condition`

这一步不恢复，时间与输出扩张就仍然是免费的。

### 2.5 最后恢复 rollback object

最后才修：

1. `rollback_object`
2. `baseline_reset_required`
3. `re_entry_ticket`
4. file rewind 与对象的映射

不要反过来：

1. 不要先修 mode 面板。
2. 不要先修 usage 图表。
3. 不要先修文件回退。

## 3. 硬拒收规则

出现下面情况时，应直接升级为硬拒收：

1. `permission_mode` 仍在替代 authority source。
2. `permission_ledger` stale，或 typed decision 只剩审批历史。
3. `decision_window` 仍只是仪表盘刷新，不进入 pending action 与 generation。
4. continuation 仍默认免费，只看“还能不能跑”。
5. compact 仍被当成降本技巧，而不是窗口重写动作。
6. rollback 仍围绕 mode / file 动作，而不是 `rollback_object`。
7. duplicate / orphan / cancel 语义仍被 later 补写说明取代。

## 5. 苏格拉底式检查清单

在你准备宣布“治理修复失真已经纠偏完成”前，先问自己：

1. 我救回的是治理对象，还是 mode 面板。
2. 我现在保护的是 authority source，还是审批完成感。
3. `Context Usage` 在我的系统里是 decision window，还是 usage dashboard。
4. continuation 在我的系统里是正式 gate，还是默认行为。
5. rollback 在我的系统里还是对象边界，还是 mode 与文件动作。

## 6. 一句话总结

真正成熟的治理修复纠偏，不是把 mode、弹窗和图表修得更漂亮，而是把 authority 假恢复、假窗口重置与免费重入重新压回同一个统一定价控制面。
