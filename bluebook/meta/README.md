# 目录元协议

`meta/` 不是第二根入口，也不是另一套 why 页。

它只回答三件事：

1. 同一类第一问题，是否又在多页里第一次被回答。
2. why、object、route-only 与 execution verdict 的发言权，是否又被写混。
3. later maintainer 当前该删词、降格、退回哪一层，而不是继续加厚哪一页。

更硬一点说，`meta/` 只处理目录契约失真，不处理三条母线本身。

## 什么时候进来

- 当你已经知道自己在问 Prompt、治理或当前真相保护，但争议已经变成“哪一页有资格先回答这个问题”。
- 当你发现某页同时在发 first answer、列 second-hop inventory、补 why-proof 与代发 execution verdict。
- 当你需要判断一次目录优化究竟减少了 same-question 多页首答，还是只是把目录解释写得更长。

## 这里不回答什么

- 不重新发 `README` 的三条首跳。
- 不替 `philosophy/` 重讲 why-proof。
- 不替 `architecture / security / risk / playbooks / api` 重发对象、机制、读回或执行 verdict。
- 不替 `navigation/` 提供 artifact gap 索引。

## 苏格拉底四问

1. 删掉目录名后，这段话还能只站成结构原则吗；如果不能，它只是 route note。
2. 这页现在是在保同一主语，还是在偷长第二前门。
3. 这页是否在替 sibling 目录代签。
4. 这次目录优化是否真的减少了 later consumer 第一次学错问题的概率。
