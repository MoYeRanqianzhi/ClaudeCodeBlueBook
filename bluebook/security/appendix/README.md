# 安全专题附录索引

## 这一子目录放什么

`security/` 主目录继续保留“主线章节”。  
这里的 `appendix/` 专门放：

- 源码证据索引
- 模块到结论的映射表
- 术语表、图表、速查卡这类压缩材料

这样做的目的不是增加文档数量，  
而是避免 `security/` 主目录继续线性膨胀，  
让主线更像论证链，附录更像检索层。

同时也继续和 `../source-notes/` 分工：

- `appendix/` 负责短表、矩阵、索引
- `source-notes/` 负责较长的源码剖面

这样可以避免“速查表越写越长，最后退化成半篇主线长文”。
