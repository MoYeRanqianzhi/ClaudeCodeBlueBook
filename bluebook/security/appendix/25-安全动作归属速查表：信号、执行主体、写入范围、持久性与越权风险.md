## 3. 最短判断公式

看到一条安全动作建议时，先问四句：

1. 这条动作真正的执行主体是谁
2. 这条动作写入的是会话层、命令层还是持久设置层
3. 这条动作本应持续多久
4. 如果由错误主体代做，最可能造成哪种越权

## 4. 最常见的四类归属错误

| 归属错误 | 会造成什么后果 |
| --- | --- |
| 把宿主当成授权主体 | 一次性审批被偷偷升级成长期规则 |
| 把系统自动恢复当成人工修复 | 用户会在错误时机做多余或有害操作 |
| 把管理员动作派发给普通用户 | 造成伪可修复体验和支持链混乱 |
| 把短期例外写进持久层 | 临时放宽变成结构性边界泄漏 |

## 5. 一条硬结论

对 Claude Code 这类安全控制面来说，  
最危险的不是动作字段不够多，  
而是：

`动作已经给对了，却被错误主体在错误层级上执行。`
