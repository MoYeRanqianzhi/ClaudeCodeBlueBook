## 模块到结论映射表

| 模块 | 主要职责 | 最适合提炼的结论 |
| --- | --- | --- |
| `src/utils/permissions/dangerousPatterns.ts` | 列出会提前赋予任意执行能力的 allow-rule 模式 | 危险不在命令名，而在规则是否把未来一整类高风险动作默认放行 |
| `src/utils/permissions/permissionSetup.ts` | 在 auto mode 入口识别危险或过宽的 Bash / PowerShell / Agent allow rule | classifier 之前先做规则层去风险化，防止仲裁层被 allow rule 架空 |
| `src/utils/permissions/permissions.ts` | 总权限仲裁链，整合 rule、tool check、mode、classifier、headless fallback | deny、content-specific ask 与 `safetyCheck` 比 bypass 和大多数 fast-path 更高 |
| `src/utils/permissions/pathValidation.ts` | 防止路径在校验与执行之间发生语义漂移 | 路径校验的本质不是字符串过滤，而是验证对象与执行对象一致 |
| `src/utils/permissions/shadowedRuleDetection.ts` | 检测不可达 permission rules | 系统治理的不只是坏规则，还有“写了也到不了”的假规则 |
| `src/utils/permissions/permissionExplainer.ts` | 为 ask/approval 场景生成结构化解释 | 解释层重要，但它属于可解释性，不属于授权执行链本身 |
| `src/tools/WebFetchTool/preapproved.ts` | 维护 WebFetch 的读取型预批准域名 | GET 读取白名单绝不等于网络写出或通用外联白名单 |
| `src/utils/mcpValidation.ts` | 控制 MCP 输出的 token 预算、截断与压缩 | 外部工具输出规模本身就是安全与上下文主权问题 |
| `src/services/mcp/config.ts` | 治理 MCP 连接定义、策略过滤、enterprise 独占与 plugin-only 锁定 | MCP 的核心风险不是“能不能连”，而是谁有资格定义连接 |
| `src/utils/managedEnv.ts` | 过滤 settings-sourced env，保护 SSH、host-managed provider 与宿主 spawn env | 真正高风险的是运行时主链被低信任来源悄悄改写 |
| `src/utils/settings/settings.ts` | 解析、合并、排序 settings 来源 | `policySettings` 采用 first-source-wins，表明主权优先级高于文件出现顺序 |
| `src/utils/settings/validation.ts` | 格式化校验错误，并局部过滤坏 permission rules | 单个坏规则不应毒化整份配置 |
| `src/utils/settings/pluginOnlyPolicy.ts` | 用 managed policy 锁定 customization surface | “谁有资格扩展系统”本身就是安全问题 |

## 四条最短证据链

### 1. 规则越权链

- `dangerousPatterns.ts`
- `permissionSetup.ts`
- `permissions.ts`
- `shadowedRuleDetection.ts`

最短结论：

`系统先识别会绕过仲裁的规则，再运行仲裁链，最后还检查规则是否真实可达。`

### 2. 路径语义链

- `pathValidation.ts`
- `permissions.ts`

最短结论：

`路径安全不是“路径像不像白名单里的字符串”，而是“验证对象与执行对象是否保持同一语义”。`

### 3. 外部入口链

- `WebFetchTool/preapproved.ts`
- `mcpValidation.ts`
- `services/mcp/config.ts`

最短结论：

`Claude Code 不把外部能力当成一个统一的插件口，而是把读取、连接定义、输出污染半径分别治理。`

### 4. 来源主权链

- `managedEnv.ts`
- `settings.ts`
- `validation.ts`
- `pluginOnlyPolicy.ts`

最短结论：

`在高风险链路上，Claude Code 先问“这值从哪来”，再问“这值是什么”。`
