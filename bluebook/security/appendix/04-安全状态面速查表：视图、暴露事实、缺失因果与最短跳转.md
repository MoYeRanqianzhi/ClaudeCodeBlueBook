# 安全状态面速查表：视图、暴露事实、缺失因果与最短跳转

## 1. 这一页服务于什么

这一页是 [21-安全状态面源码解剖：为什么系统已有很多零件，却还没有一张统一安全控制台](../21-%E5%AE%89%E5%85%A8%E7%8A%B6%E6%80%81%E9%9D%A2%E6%BA%90%E7%A0%81%E8%A7%A3%E5%89%96%EF%BC%9A%E4%B8%BA%E4%BB%80%E4%B9%88%E7%B3%BB%E7%BB%9F%E5%B7%B2%E6%9C%89%E5%BE%88%E5%A4%9A%E9%9B%B6%E4%BB%B6%EF%BC%8C%E5%8D%B4%E8%BF%98%E6%B2%A1%E6%9C%89%E4%B8%80%E5%BC%A0%E7%BB%9F%E4%B8%80%E5%AE%89%E5%85%A8%E6%8E%A7%E5%88%B6%E5%8F%B0.md) 的配套速查表。

它不重新展开长文，  
只回答一个问题：

`当前不同安全状态视图，各自到底暴露了什么事实，又缺了什么因果。`

## 3. 一句话压缩

把上表压成一句话：

`Claude Code 现有安全状态面不是没有全局信息，而是把全局信息拆成了多个局部最优视图。`

每个视图都对，  
但每个视图都只解释局部真相。

## 4. 该如何使用这张表

- 如果你先看到的是一个界面，就用这张表反推它当前到底在暴露哪一类安全事实
- 如果你先遇到的是一个问题，就用这张表决定先去哪一个视图，而不是盲目在所有设置里乱找
- 如果你在设计别的 Agent 平台，就用这张表识别：哪些状态应该做摘要，哪些状态应该做阻断，哪些状态必须带主权语义

## 5. 最终用途

这张表的最终作用，是把 `21` 里“系统已有很多零件，却没有统一控制台”的判断，  
压成一张能直接检索、直接引用、直接拿来做产品设计讨论的速查视图。
