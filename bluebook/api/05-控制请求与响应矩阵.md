# 控制请求与响应矩阵

本章把 `controlSchemas.ts` 里的 control protocol 做成可查阅矩阵。

## 1. 包装层

所有控制协议都包在三种 envelope 里：

- `control_request`
- `control_response`
- `control_cancel_request`

证据：

- `claude-code-source-code/src/entrypoints/sdk/controlSchemas.ts:552-619`
- `claude-code-source-code/src/cli/structuredIO.ts:469-531`

这说明控制协议是一等消息流，而不是 out-of-band 回调。

## 3. 基础控制面

### `initialize`

作用：

- 初始化 hooks
- 注入 sdk MCP servers
- 注入 JSON schema / systemPrompt / appendSystemPrompt
- 注入 agents、promptSuggestions、agentProgressSummaries

响应会返回：

- commands
- agents
- output styles
- models
- account
- fast mode state

证据：

- `claude-code-source-code/src/entrypoints/sdk/controlSchemas.ts:57-95`

### `interrupt`

作用：

- 中断当前 turn

证据：

- `claude-code-source-code/src/entrypoints/sdk/controlSchemas.ts:97-103`

### `set_permission_mode` / `set_model` / `set_max_thinking_tokens`

证据：

- `claude-code-source-code/src/entrypoints/sdk/controlSchemas.ts:124-155`

## 4. 权限与上下文控制面

### `can_use_tool`

这是最关键的控制请求之一。  
它会把 tool name、input、permission suggestions、blocked path、tool_use_id、agent_id 等发给宿主。

证据：

- `claude-code-source-code/src/entrypoints/sdk/controlSchemas.ts:106-122`

### `get_context_usage`

证据：

- `claude-code-source-code/src/entrypoints/sdk/controlSchemas.ts:175-220`

## 8. `StructuredIO` 赋予它们的运行语义

`StructuredIO.sendRequest(...)` 给 control protocol 加了四个重要语义：

1. pending request 管理
2. abort 时发 `control_cancel_request`
3. can_use_tool 的 sent/resolved 生命周期追踪
4. 晚到/重复 response 的去重保护

证据：

- `claude-code-source-code/src/cli/structuredIO.ts:149-158`
- `claude-code-source-code/src/cli/structuredIO.ts:160-162`
- `claude-code-source-code/src/cli/structuredIO.ts:469-531`

这意味着 control protocol 在实际运行中是一个可靠请求-响应层，而不是裸 JSON 消息。

## 9. 从第一性原理看，这个协议解决了什么

它解决的是一个本质问题：

如果 Claude Code CLI 是真正的 runtime，那么外部宿主该如何不是“旁观输出”，而是“参与控制”？

答案就是这套 control protocol：

- 宿主不只看消息，还能改 mode、接权限、改 settings、管 MCP、回滚文件、停止任务。

这也是 Claude Code 和普通 SDK 封装的根本差异。
