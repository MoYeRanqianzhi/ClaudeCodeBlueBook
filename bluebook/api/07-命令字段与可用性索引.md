# 命令字段与可用性索引

这一章是对 [内置命令域索引](06-%E5%86%85%E7%BD%AE%E5%91%BD%E4%BB%A4%E5%9F%9F%E7%B4%A2%E5%BC%95.md) 的继续下钻。  
前一章回答“命令面按什么功能域组织”，这一章回答“一个命令对象到底由哪些字段构成，以及这些字段如何决定它是否出现、如何执行、是否可被模型调用”。

## 1. 先说结论

Claude Code 的命令系统至少有五层过滤与装配：

1. 编译期裁剪：某些命令根本不会出现在 external build。
2. 静态主表：`COMMANDS()` 与 `INTERNAL_ONLY_COMMANDS` 定义硬编码入口。
3. 运行时装配：skills、plugins、workflow commands、dynamic skills 会继续注入。
4. 可用性过滤：`availability` 按 auth/provider 过滤。
5. 启用态过滤：`isEnabled()`、`isHidden()`、交互模式判断继续缩减最终可见集。

关键证据：

- `claude-code-source-code/src/types/command.ts:25-57`
- `claude-code-source-code/src/types/command.ts:154-216`
- `claude-code-source-code/src/commands.ts:224-346`
- `claude-code-source-code/src/commands.ts:417-517`

## 2. Command 类型不是一个平面结构

### 2.1 `prompt` 命令

`PromptCommand` 主要用于 skill、prompt 型 slash command 和部分 plugin/bundled 能力。  
它除了 `type/name/description` 外，还支持：

- `progressMessage`
- `contentLength`
- `argNames`
- `allowedTools`
- `model`
- `source`
- `hooks`
- `skillRoot`
- `context`
- `agent`
- `effort`
- `paths`

证据：

- `claude-code-source-code/src/types/command.ts:25-57`

这已经不是“命令描述对象”，而是一个可携带执行约束与调度信息的 capability descriptor。

### 2.2 `local` 命令

`LocalCommand` 更像本地 runtime 操作：

- `supportsNonInteractive`
- `load()`
- 返回值可能是 `text`、`compact`、`skip`

证据：

- `claude-code-source-code/src/types/command.ts:16-23`
- `claude-code-source-code/src/types/command.ts:59-78`

像 `/compact`、非交互 `/context`、`/rewind` 就属于这一层。

### 2.3 `local-jsx` 命令

`LocalJSXCommand` 适合本地 UI / 对话框 / picker / 设置页一类交互。  
它不是简单 return text，而是通过 `onDone(...)` 决定：

- 用户是否要看到系统消息
- 是否继续 query
- 是否追加 metaMessages
- 是否填入或直接提交下一条输入

证据：

- `claude-code-source-code/src/types/command.ts:80-152`

这说明 slash command 本身已经是 REPL 控制平面的组成部分。

## 3. 公共字段语义

`CommandBase` 把命令对象的公共控制面拆得很细：

- `availability`: 这个命令面向谁。
- `isEnabled()`: 当前环境下是否打开。
- `isHidden`: 是否从 typeahead / help 中隐藏。
- `aliases`: 别名。
- `argumentHint`: 参数提示。
- `whenToUse`: 主要给 skill/扩展命令提供使用场景说明。
- `disableModelInvocation`: 模型能不能主动调用。
- `userInvocable`: 用户能不能手敲 `/name` 触发。
- `loadedFrom`: 命令来源。
- `kind`: 例如 workflow。
- `immediate`: 是否绕过 stop point 队列立即执行。
- `isSensitive`: 参数是否应从会话历史中脱敏。
- `userFacingName()`: 展示名是否和内部名不同。

证据：

- `claude-code-source-code/src/types/command.ts:175-216`

这里最重要的思想是：Claude Code 把“可见性”“执行语义”“模型可调用性”“来源归属”分开建模，而不是塞进一个 `enabled`。

## 4. 命令列表如何被组装

### 4.1 `INTERNAL_ONLY_COMMANDS` 不是隐藏命令，而是 build/runtime 双边界的一部分

`INTERNAL_ONLY_COMMANDS` 明确标注“会被 external build 裁掉”，其中包括：

- `commit`
- `commitPushPr`
- `initVerifiers`
- `version`
- `teleport`
- `agentsPlatform`

以及其他 ant-only 项。

证据：

- `claude-code-source-code/src/commands.ts:224-254`

它们不是“用户碰巧看不到”，而是产品形态上就不属于公开构建。

### 4.2 `COMMANDS()` 只是内置主表，不是最终真相

`COMMANDS()` 里汇总了几十个静态内置命令，并按 feature / 用户类型条件拼接：

- `bridge`
- `voice`
- `workflowsCmd`
- `briefCommand`
- `assistantCommand`
- `INTERNAL_ONLY_COMMANDS`

证据：

- `claude-code-source-code/src/commands.ts:256-346`

### 4.3 `loadAllCommands()` 会继续合并多来源命令

真正的运行时命令集来自：

- bundled skills
- builtin plugin skills
- skill dir commands
- workflow commands
- plugin commands
- plugin skills
- `COMMANDS()` 内置主表

证据：

- `claude-code-source-code/src/commands.ts:353-398`
- `claude-code-source-code/src/commands.ts:445-469`

### 4.4 `getCommands()` 最后再应用 availability / enabled / dynamic skill 规则

`getCommands()` 做三件事：

1. 先执行 `meetsAvailabilityRequirement(_) && isCommandEnabled(_)`。
2. 再注入 dynamic skills。
3. 把 dynamic skills 插到 plugin skills 与 built-in commands 之间。

证据：

- `claude-code-source-code/src/commands.ts:476-517`

因此最终 slash command 列表是运行时计算结果，不是一个固定清单。

## 5. 可用性分层：至少有四类边界

### 5.1 provider / auth 边界：`availability`

当前 `CommandAvailability` 至少包括：

- `claude-ai`
- `console`

`meetsAvailabilityRequirement()` 会基于：

- `isClaudeAISubscriber()`
- 是否为 3P 服务
- 是否为第一方 Anthropic base URL

判断命令是否应出现。

证据：

- `claude-code-source-code/src/types/command.ts:154-173`
- `claude-code-source-code/src/commands.ts:408-443`

### 5.2 runtime 边界：`isEnabled()`

很多命令再往下还会受环境、平台、GrowthBook、交互模式影响。  
例如：

- `/compact` 受 `DISABLE_COMPACT` 控制
- `/fast` 受 `isFastModeEnabled()` 控制
- `/install-github-app` 受环境变量控制
- `/remote-control` 受 bridge gate 控制

证据：

- `claude-code-source-code/src/commands/compact/index.ts:4-13`
- `claude-code-source-code/src/commands/fast/index.ts:8-24`
- `claude-code-source-code/src/commands/install-github-app/index.ts:4-11`
- `claude-code-source-code/src/commands/bridge/index.ts:5-24`

### 5.3 可见但不展示：`isHidden`

有些命令即使存在，也会被从 UI 中隐藏。  
例如：

- `/remote-control` 在未启用时通过 getter 直接隐藏
- 非交互版 `/extra-usage` 在交互会话中隐藏

证据：

- `claude-code-source-code/src/commands/bridge/index.ts:18-23`
- `claude-code-source-code/src/commands/extra-usage/index.ts:21-30`

### 5.4 同名双实现：交互模式也是边界

同一个 slash name 可能对应两套实现：

- `/context` 有 `local-jsx` 交互版与 `local` 非交互版
- `/extra-usage` 也有交互版与非交互版

证据：

- `claude-code-source-code/src/commands/context/index.ts:4-24`
- `claude-code-source-code/src/commands/extra-usage/index.ts:13-30`

这意味着“命令名称”不是实现 identity；运行模式也是调度条件。

## 6. 字段级高价值例子

### 6.1 `/compact`: 最典型的 `local` 命令

它展示了 `local` 命令的几个关键字段组合：

- `type: 'local'`
- `supportsNonInteractive: true`
- `argumentHint`
- `isEnabled()` 受环境变量控制

证据：

- `claude-code-source-code/src/commands/compact/index.ts:4-13`

### 6.2 `/remote-control`: feature gate、alias、hidden、immediate 同时出现

`/remote-control` 具备：

- `aliases: ['rc']`
- `immediate: true`
- `get isHidden()`
- `isEnabled()` 依赖 `feature('BRIDGE_MODE')` 与 `isBridgeEnabled()`

证据：

- `claude-code-source-code/src/commands/bridge/index.ts:1-24`

这是最能体现“命令对象也是 rollout 控制对象”的例子之一。

### 6.3 `/fast`: availability 和 immediate 都可能是动态 getter

`/fast` 的：

- `description`
- `immediate`
- `isHidden`

都不是纯静态字面量，而是按运行状态动态计算。

证据：

- `claude-code-source-code/src/commands/fast/index.ts:8-24`

这说明命令元数据本身也是 runtime surface。

### 6.4 `/review` 与 `/ultrareview`: 同域不同执行平面

`/review` 是 `prompt` 命令，本地拼出 PR review prompt。  
`/ultrareview` 则是 `local-jsx` 命令，作为远程 bughunter path 的唯一入口。

证据：

- `claude-code-source-code/src/commands/review.ts:33-54`

这组例子说明：同一个“功能域”并不等于同一种命令类型。

### 6.5 `/chrome`、`/usage`、`/install-github-app`: provider-gated 命令

这几个命令都声明了 `availability`：

- `/chrome` 仅 `claude-ai`
- `/usage` 仅 `claude-ai`
- `/install-github-app` 同时支持 `claude-ai` 与 `console`

证据：

- `claude-code-source-code/src/commands/chrome/index.ts:4-10`
- `claude-code-source-code/src/commands/usage/index.ts:3-8`
- `claude-code-source-code/src/commands/install-github-app/index.ts:4-11`

因此 availability 不是理论字段，而是广泛使用的 provider/product 边界。

### 6.6 “迁到插件”的兼容命令也是正式命令类型

`createMovedToPluginCommand()` 生成的仍是 `prompt` 命令。  
它的职责不是执行原功能，而是把用户引导到 marketplace / plugin 路径。

证据：

- `claude-code-source-code/src/commands/createMovedToPluginCommand.ts:22-65`

这说明命令系统不仅承载“执行”，也承载产品迁移。

## 7. 技能与插件命令会把字段空间进一步拉大

### 7.1 skill frontmatter 会落到 Command 字段上

`parseSkillFrontmatterFields()` 与 `createSkillCommand()` 把 frontmatter 解析进命令对象：

- `allowedTools`
- `argumentHint`
- `argNames`
- `whenToUse`
- `version`
- `model`
- `disableModelInvocation`
- `userInvocable`
- `context`
- `agent`
- `effort`
- `paths`
- `hooks`
- `loadedFrom`

证据：

- `claude-code-source-code/src/skills/loadSkillsDir.ts:185-264`
- `claude-code-source-code/src/skills/loadSkillsDir.ts:267-400`

这意味着 `Command` 类型里最丰富的字段，很多并不主要由内置命令使用，而是为 skill/plugin/MCP 生态准备。

### 7.2 bundled skill 与 built-in plugin skill 会被转换成统一 Command

例如：

- `verify` 通过 `registerBundledSkill(...)` 成为 bundled skill
- built-in plugin skills 会在 `skillDefinitionToCommand()` 中被转成 `Command`

证据：

- `claude-code-source-code/src/skills/bundled/verify.ts:12-29`
- `claude-code-source-code/src/plugins/builtinPlugins.ts:132-158`

这里能直接看到 `loadedFrom: 'bundled'`、`allowedTools`、`whenToUse`、`disableModelInvocation`、`userInvocable`、`context`、`agent` 等字段如何进入统一命令面。

## 8. 当前可观察到的可用性矩阵

从公开源码可保守总结出五类命令：

1. 普遍可用命令：无 `availability`，也无额外 gate。
2. provider-gated 命令：靠 `availability` 区分 `claude-ai` / `console`。
3. runtime-gated 命令：靠 `isEnabled()` 绑定 feature/env/platform。
4. internal-only 命令：只在 ant 用户且特定 build 条件下进入主表。
5. 迁移/兼容命令：保留 slash surface，但真实功能已转移到 plugin 或远程产品面。

## 9. 当前边界

需要保守说明三点：

1. `kind: 'workflow'` 字段在类型中存在，但当前提取源码里没有直接看到 workflow 命令构造实现，因此不能过度断言其所有运行语义。
2. 某些动态命令来源依赖 plugin、MCP、skills 目录或 feature gate，当前仓库只能确认其装配接口，不等于默认对所有用户出现。
3. `Command` 字段空间很大，但内置命令实际使用的字段子集并不覆盖全部类型字段。
