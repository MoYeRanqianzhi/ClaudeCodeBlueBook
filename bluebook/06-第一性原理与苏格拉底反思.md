# 第一性原理与苏格拉底反思

这章只做总控自校，不做第二入口页。

它不再替 `README / philosophy / guides / architecture` 重发首答，只持续追问一件事：我们是不是又把第一问题写成投影词、把 why 写成路由、把对象权责写成作者体感。

先固定三条判断：

1. Prompt 线
   - 只有 later consumer 在 `verify / delegate / tool choice / resume / handoff` 时都不用再重述现场、重划边界、重接消息血缘、重判继续资格，当前世界才算真的被合法编译进模型。
2. 治理线
   - 治理成熟度不先看安全、成本、体验或统一预算器，先问最早 `unpaid expansion` 是什么，再核对 `repricing proof`、`lease checkpoint` 与 `cleanup`。
3. 源码质量线
   - 源码先进性不先看目录观感或文件大小，先问当前真相是否已被 `change-risk record / why-proof / landing card / local quality gate` 正式保护。

`continuity` 不是第四条母线，只是三条判断在时间轴上的联合验收。

这页的职责边界也只剩三条：

1. 只验三条母线是否仍闭合，不重发导航层与 owner chain。
2. 只验第一反证、缺口 artifact 与最小回退，不重做对象百科。
3. 源码锚点只作为各母线 `first reject` 的支撑，不再单列“锚点墙”。

一旦问题已经退化成 next-hop、页归属或 route dispute，本章就该退出；那不是 self-audit，而是 route-only 层的问题。

## 1. 最小自检法只保留四问

三条母线在这里不是三张并列表，只是同一组四问的不同答案：

1. 哪条 `law` 先失效了。
2. 第一反证信号是什么。
   - Prompt：`compact / delegate / resume / handoff` 之后还要重述现场，same-world test 直接失败。
   - 治理：说不清最早 `unpaid expansion`、谁 repriced、谁结算旧 authority，就先按非法续租处理。
   - 源码质量：说不清 `one writable present` 由谁保护，或 later maintainer 拿不到第一条 veto，质量判断直接降格。
3. 缺哪个 artifact / verdict。
   - Prompt 侧若还答不出 later consumer 凭什么不用重答世界定义、边界与工具资格，就说明 `compiled world verdict` 还没真正落地。
   - 治理缺 `repricing proof / lease checkpoint / cleanup`。
   - 源码质量缺 `change-risk record / why-proof / landing card / local quality gate`。
4. 最小回退动作是什么。
   - 降格 projection、冻结继续资格、补 artifact，或退出到 route-only。

其余自校问题都只是这四问的展开。继续压短时，也只是在追 `compile / price / preserve` 哪一环先失效。

## 2. 何时值得下钻到 runtime 对象层

只有当四问已经判到问题确实落在 runtime plane，而不是 route-only 或 artifact gap 时，才继续下钻。更稳的 drill-down 也只保留四桶，而不是再立一张“八类对称总表”。

### 2.1 场景编译与继续资格

- 先看 `query.ts`、`QueryEngine.ts`、`sessionMemoryCompact.ts`、`stopHooks.ts` 这类 turn/runtime/compact 对象。
- 第一反证不是“Prompt 看着不够强”，而是 later consumer 在 `verify / delegate / resume / handoff` 时又得补世界定义、补边界、补继续资格。

### 2.2 当前真相与回写

- 先看 `structuredIO.ts`、`remoteIO.ts`、writeback seam 一带。
- 第一反证不是“状态很多”，而是 display transcript、replay 或人类体感开始越权改写 current truth。

### 2.3 协作与隔离

- 先看 `tools/AgentTool`、worktree、remote、sidechain transcript 这类隔离对象。
- 第一反证不是“agent 数量不够多”，而是 child 开始自立主语、共享不该共享的现场，或合并后没人说得清谁在结算残局。

### 2.4 定价与扩张控制

- 先看 `tools.ts`、`commands.ts`、feature gate、prompt-cache break、permission/runtime policy 这类扩张控制面。
- 第一反证不是“token 更多了”或“安全更严了”，而是系统解释不出最早 `unpaid expansion`、`lease checkpoint` 和旧 authority `cleanup` 到底在哪完成。

如果某段下钻不能回收到这四桶之一，它大概率还只是 inventory，不是 self-audit 必须保留的对象。

## 3. 四种常见误判

1. 误把更强模型当成外层制度的替代品。
   - 反证：更强模型不会自动补齐权限边界、恢复链与 public/gated 边界。
2. 误把更多工具当成系统更强。
   - 反证：若工具没有统一权限、统一上下文与统一恢复，它只会让动作选择更乱。
3. 误把 Claude Code 看成传统 IDE 插件。
   - 反证：一旦问题涉及持久化、中断恢复、协作与扩展，讨论的就是 runtime，不是 widget。
4. 误把“解释得更顺”当成设计内涵。
   - 反证：如果删掉术语后就说不清谁在定义世界、谁在批准扩张、谁在否决陈旧写入，这段还在讲外观，不在讲制度内核。

## 4. 这类分析最常会在哪里失真

高频失真也只按四桶看，不再扩成百科全书。

### 4.1 第一反证信号丢失

- Prompt 被写成“文案更强”，却不给 later consumer 的冷启动失败信号。
- 治理被写成“拦得更严”，却不给最早 `unpaid expansion` 的定位。
- 源码质量被写成“结构更漂亮”，却不给 later maintainer 的第一条 veto。

苏格拉底追问：

- 如果这段明天失效，最先坏的是哪条 `law`，不是哪句赞美词。

### 4.2 Artifact / evidence 缺席

- Prompt 侧还答不出 later consumer 凭什么不用重答世界定义、边界与工具资格，却先夸协作流畅。
- 治理没有 `repricing proof / lease checkpoint / cleanup`，却先夸统一预算。
- 源码质量没有 `change-risk record`，却先下成熟结论。

苏格拉底追问：

- 我现在缺的到底是哪份正式制品，而不是哪段更会说的话。

### 4.3 边界混写

- 把 UI transcript 写成 protocol truth。
- 把 mode、弹窗、usage 条写成治理主语。
- 把 why-proof、landing card、route map 与对象判词写成同一层。

苏格拉底追问：

- 现在是谁在发言，谁只是在投影，谁只配当 evidence。

### 4.4 升级先坏点

- 还没给出第一反证，就先升级成全局公式。
- 还没给出最小回退，就先谈一次性重写。
- 并行深挖后只拼接更长摘要，没有保住同一问题主语。

苏格拉底追问：

- 第一条坏信号出现时，我知道该先降格哪层结论，还是只能等全局崩掉。

## 5. 持续审读最小协议

这页以后只按下面五步反复审读，不再维持十张判据卡：

1. 同一主语
   - 我现在回答的，仍是同一个第一问题吗，还是已经偷偷换成 projection、route note 或对象库存。
2. 第一反证
   - 我是否给出了最早失败信号，而不是只重复正解术语。
3. Artifact 所有权
   - 我是否说清楚缺哪份 artifact，以及它该由哪条母线补齐。
4. 最小回退
   - 我是否说清楚应先降格 projection、冻结继续资格、补记录，还是退出到 route-only。
5. 并行合并
   - 多 Agent 深挖后，合并保住的是同一问题主语，还是只是更长的摘要拼盘。

只要这五步里有一步答不上，本页就不该继续加厚，而应先回去删词、降格或补 artifact。

## 6. 最后一句

从第一性原理看，Claude Code 的值钱之处不在“替你写代码”，而在它先把世界准入、扩张定价与当前真相保护写成不可外包的 runtime 席位。
从苏格拉底式反思看，更稳的写法也只剩一句：少发第二前门，少发作者 inventory，多追第一反证、artifact 缺口与最小回退，让 later consumer 接手时不必重新发明同一个世界。
