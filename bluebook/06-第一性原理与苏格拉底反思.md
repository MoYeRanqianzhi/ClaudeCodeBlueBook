# 第一性原理与苏格拉底反思

这章只做总控自校，不做第二入口页。

它不再替 `README / philosophy / guides / architecture` 重发首答，只持续追问一件事：我们是不是又把第一问题写成投影词、把 why 写成路由、把对象权责写成作者体感。

先固定三条自校主语：

1. Prompt 线
   - 只验 same-world 是否仍闭合。
2. 治理线
   - 只验 `authority lease` 是否仍闭合。
3. 源码质量线
   - 只验 present-veto 是否仍闭合。

`continuity` 不是第四条母线，只是三条判断在时间轴上的联合验收。

这页的职责边界也只剩三条：

1. 只验三条母线是否仍闭合，不重发导航层与归属链。
2. 只验第一反证、缺口 artifact 与最小回退，不重做对象百科。
3. 源码锚点只作为各母线 `first reject` 的支撑，不再单列“锚点墙”。

一旦问题已经退化成 next-hop、页归属或 route dispute，本章就该退出；那不是 self-audit，而是 route-only 层的问题。

## 1. 最小自检法只保留四问

三条母线在这里不是三张并列表，只是同一组四问的不同答案：

1. 哪条 `law` 先失效了。
2. 第一反证信号是什么。
   - Prompt：`compact / delegate / resume / handoff` 之后还要重述现场，same-world test 直接失败。
   - 治理：说不清最早 `unpaid expansion` 是否已被 repriced、旧 authority 是否已 cleanup、当前继续该由谁结算，就先按非法续租处理。
   - 源码质量：说不清 `one writable present` 由谁保护，或 later maintainer 拿不到第一条 veto，质量判断直接降格。
3. 缺哪种正式承接 / verdict。
   - 先问这条母线缺的究竟是正式判断、正式记录，还是只是被投影词替代了。
4. 最小回退动作是什么。
   - 先判该把哪层 claim 降格成 `projection`、冻结哪类 `continuation`、撤回哪条 `maturity verdict`，而不是先补哪张检查表。

其余自校问题都只是这四问的展开。继续压短时，也只是在追 `compile / price / preserve` 哪一环先失效。再压一层，也只剩一句反问：我是不是又让 later consumer 重新获得了对世界、扩张或当前的重判资格；若是，这页就不该继续加厚，而该先判哪条母线已经失守。若连该降格哪层 claim、冻结哪类 `continuation`、撤回哪条 `maturity verdict` 都说不清，就说明这里还没完成自校，只是在催生下一张检查表。
更硬一点说，自校最该先抓的也不是“解释还顺不顺”，而是我是不是又把 `zero-delta re-adjudication` 写成了进步：same-world 已成立却还在重谈世界，repricing 已无新增增益却还在重复 ask，present-veto 已失守却还在夸目录；只要出现任一条，就先停在降格与回退，不继续扩写。

## 2. 何时退出到对象层

只有当四问已经判到问题确实落在对象层，而不是 route-only 或 artifact gap 时，才继续下钻；但对象层目录、runtime 桶位与源码簇不在本页列出。更稳的做法是：先在本页写清第一反证、artifact 缺口与最小回退，然后把 next-hop 与对象层统一交回 `navigation`。

## 2.1 目录结构的苏格拉底六问

如果正在优化蓝皮书目录，本页默认只反问六件事：

1. 删掉目录名后，这段话还能单独站成 why、object 或 verdict 吗；如果不能，它只是 route note。
2. 这页是否同时在回答 first answer 与 second-hop inventory；如果是，它就在偷长第二 frontdoor。
3. 这页是否在替别的目录代签：`philosophy` 不代签 route，`guides` 不代签 object truth，`architecture` 不代签 evidence ceiling，`playbooks` 不代签 why。
4. 这次目录优化是否真的减少了同题多页首答，而不只是把目录解释写得更长。
5. 并行多 Agent 合并后，保住的是同一问题主语，还是把不同目录的句子各抄一句。
6. 这条新增 why 若离开归属页就站不住，是否还被误抬成总控综合；如果是，缺的不是新 frontdoor，而是归属页自己的单线因果。

若问题已经从局部句子失真，升级成整个目录契约是否仍成立，统一回 `meta/README`；本页继续只保留自校问句，不代写目录总章。

## 3. 四种常见误判

1. 误把更强模型当成外层制度的替代品。
   - 反证：更强模型不会自动补齐权限边界、恢复链与 public/gated 边界。
2. 误把更多工具当成系统更强。
   - 反证：若工具没有统一权限、统一上下文与统一恢复，它只会让动作选择更乱。
3. 误把 Claude Code 看成传统 IDE 插件。
   - 反证：一旦问题涉及持久化、中断恢复、协作与扩展，讨论的就是 runtime，不是 widget。
4. 误把“解释得更顺”当成设计内涵。
   - 反证：如果删掉术语后就说不清谁在定义世界、谁在批准扩张、谁在否决陈旧写入，这段还在讲外观，不在讲制度内核。

## 4. 这类分析最常会在哪里失真

高频失真也只按四桶看，不再扩成百科全书。

### 4.1 第一反证信号丢失

- Prompt 被写成“文案更强”，却不给 later consumer 的冷启动失败信号。
- 治理被写成“拦得更严”，却不给最早 `unpaid expansion` 的定位。
- 源码质量被写成“结构更漂亮”，却不给 later maintainer 的第一条 veto。

苏格拉底追问：

- 如果这段明天失效，最先坏的是哪条 `law`，不是哪句赞美词。

### 4.2 正式承接缺席

- Prompt 侧还答不出 later consumer 凭什么不用重答世界定义、边界与工具资格，却先夸协作流畅。
- 治理没有 `repricing proof / lease checkpoint / cleanup`，却先夸统一预算。
- 源码质量还没把 current-truth claim 写成可反对记录，却先下成熟结论。

苏格拉底追问：

- 我现在缺的到底是哪份正式制品，而不是哪段更会说的话。

### 4.3 边界混写

- 把 UI transcript 写成 protocol truth。
- 把 mode、弹窗、usage 条写成治理主语。
- 把 why-proof、landing card、route map 与对象判词写成同一层。

苏格拉底追问：

- 现在是谁在发言，谁只是在投影，谁只配当 evidence；若一时分不清，就先按越级代签处理，把 `UI transcript`、`mode / usage` 面板与作者说明一律降回 `projection / evidence`。
- `weak readback / projection` 的降格也不是语气保守，而是治理纪律：凡不能补 `repricing proof / lease checkpoint / cleanup` 的 surface，都不配再说当前真相。

### 4.4 升级先坏点

- 还没给出第一反证，就先升级成全局公式。
- 还没给出最小回退，就先谈一次性重写。
- 并行深挖后只拼接更长摘要，没有保住同一问题主语。

苏格拉底追问：

- 第一条坏信号出现时，我知道该先降格哪层结论，还是只能等全局崩掉。

## 5. 持续审读最小协议

这页以后只按下面六步反复审读，不再维持十张判据卡：

1. 同一主语
   - 我现在回答的，仍是同一个第一问题吗，还是已经偷偷换成 projection、route note 或对象库存。
2. 第一反证
   - 我是否给出了最早失败信号，而不是只重复正解术语。
3. Artifact 所有权
   - 我是否说清楚缺哪份 artifact，以及它该由哪条母线补齐；再问这份 artifact 到底配签到哪一层，foundation/context 若越级代签执行侧裁决，本身就是缺口。
4. 最小回退
   - 我是否说清楚应先降格 projection、冻结继续资格、补记录，还是退出到 route-only。
5. 并行合并
   - 多 Agent 深挖后，合并保住的是同一问题主语，还是只是更长的摘要拼盘。
6. 重判资格
   - 我是不是又让 later consumer 重新获得了对世界、扩张或当前的重判资格，而不是继续消费已成立的正式判断。
   - `approval fatigue` 也在这里归档：若同类安全扩张在没有新增 `repricing / deny` 决策增益时还要人类反复批准，就说明价格正在被 later consumer 代签。
   - Prompt 自校若再压成一句，也只该追问：later consumer 是不是又得先枚举候选世界与候选动作，甚至重新证明哪些分支已被当前 `authority / boundary` 排除，才能决定下一步；若必须重新搜索现场、动作空间与排除理由，same-world 即使文面仍顺也已失效。

只要这六步里有一步答不上，本页就不该继续加厚，而应先回去删词、降格或补 artifact。

## 6. 最后一句

从第一性原理看，Claude Code 的值钱之处不在“替你写代码”，而在它先把世界准入、扩张定价与当前真相保护写成不可外包的 runtime 席位。
从苏格拉底式反思看，更稳的写法也只剩一句：少发第二前门，少发作者 inventory，多追第一反证、artifact 缺口与最小回退，让 later consumer 接手时不必重新发明同一个世界。
