## 根入口只做三件事

1. 先定题，不先找页。
   - 先判断自己究竟在问请求装配、扩张定价，还是当前真相保护。
2. 先给最小顺序，不先摊深链。
   - 先拿到上面的三条首跳，再决定自己缺的是 first reject signal、对象链，还是执行 verdict。
3. 先判是否要跨目录，最后才进入 route map。
   - 只有当问题已经变成“下一层证据该去哪里找”时，才转 [navigation/README.md](navigation/README.md)。
