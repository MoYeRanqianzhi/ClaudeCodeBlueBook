# 哲学专题

`philosophy/` 当前有 86 篇编号文档，范围 `01-86`。本目录解释 Claude Code 为什么要以 runtime plane、控制面、预算和恢复闭环的方式设计，而不是仅仅罗列功能。

## 目录分层

- `01-17`: 基础设计哲学、上下文经济学、安全观、状态优先和交互控制面。
- `18-32`: Prompt 契约、Token 经济、产品边界、可观测预算与单一真相入口。
- `33-47`: 可解释稳定性、控制面、远程失败、依赖图诚实性与可恢复真相。
- `48-67`: 协作语法、统一定价、未来维护者消费者、Prompt Constitution 与构建系统。
- `68-86`: Evidence Envelope、共享工件、拒收语义、持续验证，以及三条更高阶控制面的第一性原理收束。

## 适合什么时候读

- 想先知道这套系统为什么这样设计：从 `01-17`
- 想理解 Prompt / 安全 / Token 经济为何同构：从 `18-32`
- 想看治理、可观察性与单一权威的底层判断：从 `33-67`
- 想把这些判断回灌到升级证据、工件协议、控制面总图与构建方法：从 `68-86`
- 想抓 Prompt 魔力的高阶收束：`48 -> 52 -> 81 -> 84 -> architecture/82`
- 想抓“当前世界准入主权”这条治理母线：`38 -> 52 -> 58 -> 61 -> 79 -> 85 -> architecture/83`
- 想抓源码质量 / 可演化结构这条母线：`50 -> 59 -> 65 -> 80 -> 83 -> 86 -> architecture/84`
