# 哲学专题

`philosophy/` 解释 Claude Code 为什么要这样设计，而不是仅仅罗列功能：为什么 Prompt 要被做成协作语法，为什么治理要被写成统一定价，为什么源码质量首先服务当前真相保护。

还要先记一句：

- `84-87` 负责把 `09` 的三张控制面图继续压成第一性原理判断；它们现在分别更明确覆盖 `message lineage / governance pricing chain / current-truth surface / future maintainer rejectability`，不是另一套并列主线

## 目录分层

- `01-17`: 基础设计哲学、上下文经济学、安全观、状态优先和交互控制面。
- `18-32`: Prompt 契约、Token 经济、产品边界、可观测预算与单一真相入口。
- `33-47`: 可解释稳定性、控制面、远程失败、依赖图诚实性与可恢复真相。
- `48-67`: 协作语法、统一定价、未来维护者消费者、Prompt Constitution 与构建系统。
- `68-87`: Evidence Envelope、共享工件、拒收语义、持续验证，以及三条更高阶控制面的第一性原理收束与源码质量高阶判断。
