# Prompt 魔力来自约束叠加与状态反馈

Claude Code 的 prompt 之所以显得“有魔力”，不是因为写得更像咒语，而是因为它同时叠加了：

1. 角色约束
2. 工具边界
3. 缓存边界
4. 动态状态反馈
5. 协作语法

如果只抄文案，抄不走的是后四层。

从第一性原理看，真正强的 prompt 必须同时回答：

1. 你是谁。
2. 你能做什么。
3. 你不能做什么。
4. 你当前看到了什么最新状态。
5. 你和其他执行单元怎样分工。

Claude Code 之所以强，是因为这五个问题都不靠一段静态文本独自承担，而是由 runtime 持续供给。

所以 prompt 魔力的本体不是文本技巧，而是：

- 约束叠加
- 状态反馈

如果一个系统没有后者，再漂亮的 system prompt 也只能维持短暂表象。
