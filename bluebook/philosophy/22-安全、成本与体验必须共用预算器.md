# 安全、成本与体验必须共用预算器

很多系统把三件事拆开看：

1. 安全
2. 成本
3. 体验

Claude Code 更像是在说：

- 这三件事必须共用一个预算器

为什么？

因为它们都在约束“系统可以无序扩张到什么程度”。

- 安全限制动作空间
- token 经济限制上下文空间
- 体验限制认知噪音与恢复成本

这三者如果分开优化，最终常见结果是：

- 更安全，但卡得不可用
- 更省 token，但上下文断裂
- 更流畅，但边界失控

Claude Code 的做法更接近：

- 用统一 runtime 预算器持续校正

这解释了为什么它会同时重视：

- deny rule
- policy gate
- tool result budget
- compact / reactive compact
- state writeback

它们表面不同，本质上都在做一件事：

- 防止系统失去可治理性

真正成熟的 Agent 产品，不该在“安全 vs 成本 vs 体验”之间做口号式权衡，而该先把三者并入同一控制论框架。
