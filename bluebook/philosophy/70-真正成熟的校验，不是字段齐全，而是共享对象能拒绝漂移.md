# 真正成熟的校验，不是字段齐全，而是共享对象能拒绝漂移

成熟系统的校验，最常被误解成：

- 有没有把表单填满

但 Claude Code 式系统真正要保护的不是表单，而是：

- 不同角色在不同时间点，是否仍然围绕同一个共享对象继续判断

所以当蓝皮书走到 Artifact Validator 这一层时，真正该收束的判断是：

- 好校验不是让字段更整齐，而是让同一条 continuation truth 一旦开始裂开，系统会正式拒绝继续说谎

## 1. 第一性原理

如果一个系统已经有：

- shared header
- 最小样例
- drift casebook

却仍没有 validator / linter / reject rule，

那么它仍然只是：

- 会描述真相

而不是：

- 会捍卫真相

这意味着它依旧依赖人工自觉，依旧允许局部真相在外观完好时重新夺权。

## 2. Prompt 的魔力，为什么最后要落到 validator

Prompt 最容易被神化成：

- 文案更强

但更接近 Claude Code 真相的说法是：

- Prompt 的魔力来自同一条 `request compiler -> message lineage -> stable bytes -> continuation qualification` 链，能被宿主、CI、评审与交接持续验证

如果宿主卡退回原文 prompt、CI 附件只剩绿灯、评审卡只剩总结、交接包只剩摘要，

那么所谓的“魔力”就会立刻退回：

- 文案崇拜

因此真正成熟的 Prompt validator 不是在检查：

- 有几张卡

而是在检查：

- `request_compiler_ref`、`message_lineage_ref`、`stable_bytes_ledger_ref` 与 `continuation_qualification_ref` 是否仍在共同保护同一个继续对象

## 3. 安全与省 Token，为什么最后要共用 validator

安全最容易被退回成：

- 拦得够不够多

省 token 最容易被退回成：

- 花得够不够少

但更接近 Claude Code 真相的说法是：

- 两者都在判断当前这轮行动是否仍有决策增益

如果治理 artifact 退回状态色、计数、verdict、token 百分比与状态摘要，

那么系统就会重新滑回：

- 结果面治理

而不再是：

- externalized truth chain、decision window、continuation pricing、failure semantics、rollback object 与 next action 的治理

因此真正成熟的治理 validator 不是在检查：

- dashboard 丰不丰富

而是在检查：

- 当前真相是否已经先被外化
- `Context Usage` 是否仍只是 `decision window` 的诚实投影
- 没有 decision gain 时，系统是否正式拒绝继续烧 token、继续假放行、继续把回退对象留给后来者去猜
- 没有决策增益的检查，是否也被系统认定为免费扩张并拒绝进入主工作集

## 4. 源码先进性，为什么最后要共用 validator

源码先进性最容易被退回成：

- 目录漂亮

或：

- 作者解释得很好

但更接近 Claude Code 真相的说法是：

- 先进结构会把 current-truth surface、current-truth writeback、freshness gate、回退对象与反 zombie 证据提前编码进结构

如果结构 artifact 退回目录图、恢复成功率、结构夸奖与作者说明，

那么所谓“先进性”就会立刻退回：

- 审美与权威

因此真正成熟的结构 validator 不是在检查：

- 有没有结构图

而是在检查：

- `current_truth_surface_ref`、`current_truth_writeback_ref`、`freshness_gate_ref`、`anti_zombie_evidence_ref` 与 `rollback_object_ref` 是否仍在共同保护同一个当前真相对象

## 5. 为什么成熟校验必须是拒绝漂移，而不是接受填写

接受填写，只能回答：

- 这套制度存在

拒绝漂移，才能回答：

- 这套制度不会在最关键的时刻退回局部真相

两者的差别，就像：

- 看见一份合同

和：

- 看见一份合同在违约时仍然能触发约束

Claude Code 值得学的地方，不在于它有很多字段、很多模式、很多机制，

而在于：

- 它不断把“描述真相”的层，往“约束真相”的层继续推进

如果把成熟校验压成最短公式，只剩三句：

1. 世界不再同一，就拒绝继续。
2. 价格没有重新声明，就拒绝扩张。
3. 当前真相没有正式写回，就拒绝交接。

## 6. 苏格拉底式自反

在你准备宣布“我们已经有成熟校验了”前，先问自己：

1. 这套规则保护的是同一个继续对象，还是只保护表单完整度。
2. 如果删掉原作者解释，仅靠校验通过结果，后来者是否仍能继续、回退、交接。
3. 当前 lint 报的是格式噪音，还是 message lineage、decision window、freshness gate 与 rollback 正在分裂。
4. 校验通过后，宿主、CI、评审与交接是否仍在围绕同一对象说话。
5. 我是在奖励填写完成，还是在拒绝局部真相重新夺权。
6. 第一条 reject signal 是否已经被正式写进工件，而不是继续藏在作者解释里。
