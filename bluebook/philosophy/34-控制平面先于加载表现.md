# 控制平面先于加载表现

这一章回答四个问题：

1. 为什么 Claude Code 不把“正在加载”当成 UI 表象，而把它当成运行时真相。
2. 为什么同步 authority 比异步 batched loading state 更接近可靠系统。
3. 为什么竞态不是边缘 bug，而是本地交互控制面的第一性原理约束。
4. 这对 Agent runtime 和前台系统设计有什么可迁移启发。

## 1. 先说结论

Claude Code 在本地查询这一层传达出一条很强的设计原则：

- 控制平面先于加载表现

也就是说，系统首先要回答的不是：

- spinner 该不该转

而是：

- 现在到底是不是一个查询正在占有运行权

只有这个问题先被 authoritative state 回答，后面的：

- spinner
- queue
- submit gate
- cancel
- idle append

才不会各自活在不同真相里。

## 2. 为什么“正在加载”不能只由 UI 决定

如果“正在加载”只是 UI 表象，就会出现一个典型误区：

- 只要界面看起来没在转，就当系统空闲

Claude Code 明确不接受这一点。

因为真正危险的地方往往不是：

- 一个查询已经稳定运行

而是：

- 刚出队
- 刚进入异步链
- 还没真正到 onQuery
- 旧 finally 还没收尾

在这些时刻，UI 很容易还没来得及更新，但运行权已经必须被占住。

所以从第一性原理看，“正在加载”本质上不是视觉状态，而是：

- 当前运行权是否被占用

## 3. 为什么同步 authority 比 batched loading state 更可靠

React state 的问题不在于它不好，而在于它解决的是：

- 渲染一致性

不是：

- 运行时排他性

一旦你需要控制：

- 是否允许再次 submit
- 是否允许 queue dequeue
- 是否允许 cancel 抢回运行权
- 是否允许 stale finally 改写当前状态

你就已经进入了：

- control plane

这时如果还把 authoritative truth 托付给 batched UI state，系统会天然陷入：

- 不同消费者看到不同真相

Claude Code 的做法就是把这个问题反过来：

- 先有同步 authority
- UI 再订阅 authority

这条顺序非常重要。

## 4. 为什么竞态不是边缘 bug，而是第一性原理约束

很多系统会把竞态当成：

- 以后再补的边缘情况

Claude Code 在 `QueryGuard` 这里体现的态度完全不同：

- 竞态是设计起点

这从三个细节能看得最清楚：

1. 有 `dispatching`，说明作者在意异步空窗。
2. 有 generation，说明作者在意 stale finally。
3. 有 `forceEnd()`，说明作者在意 cancel 并不会自动让旧异步路径消失。

这背后的哲学其实很朴素：

- 只要系统存在异步链，就存在竞争未来

所以竞争条件不是异常，而是：

- 运行时的一等约束

## 5. 这对前台系统设计的启发

Claude Code 这里给出的启发，不只适用于终端 REPL。

任何 agent 前台只要同时有：

- submit
- queue
- cancel
- background event
- 异步 render

都应该先问：

- 我的 authoritative control plane 在哪里

而不是先问：

- loading state 存在哪个组件里

更可迁移的写法是：

1. 把运行权做成同步 authority。
2. 让 UI 只订阅这个 authority。
3. 让 submit / queue / cancel / finally 全部服从它。

## 6. 苏格拉底式追问

### 6.1 如果 spinner 停了，是否就等于系统空闲

不一定。
真正该问的是：运行权释放了吗。

### 6.2 如果多个路径都能改写“是否忙碌”，谁才是权威

如果回答不出来，系统就已经在半真相上运行。

### 6.3 如果 finally 可能晚到，为什么还把它当成无条件可信的清理动作

不应该。
所以需要 generation 去裁决谁是当前 owner。

### 6.4 为什么这比“把 loading 管好”更高一级

因为这里管的不是视觉表现，而是：

- 系统谁有权说自己正在执行

## 7. 一句话总结

Claude Code 的本地查询之所以稳，不是因为 loading 做得更细，而是因为它先建立了 authoritative control plane，再让加载表现去订阅它。
