# 单一权威优于单一全景：多消费者系统必须分层暴露真相

这一章回答四个问题：

1. 为什么“单一真相入口”不等于“所有消费者共享同一份全景表示”。
2. 为什么多消费者 agent runtime 必然要分 contract、主路径与消费子集。
3. Claude Code 为什么宁可分层暴露真相，也不愿追求一份看起来更整齐的全量表示。
4. 这对 agent 平台设计者有什么第一性原理启发。

## 1. 先说结论

Claude Code 在真相治理上的更深原则，可以压成一句话：

- 单一权威优于单一全景

也就是说，它追求的是：

- 权威来源单一

而不是：

- 所有消费者都看同一份完整表示

这并不是在制造多重真相。

它真正做的是：

1. 保住一份权威合同。
2. 再按对象和职责，把它编译成不同消费者能诚实使用的投影。

所以 Claude Code 反对的不是：

- single source of truth

它反对的是：

- single representation of truth

## 2. 为什么“单一全景”在多消费者系统里反而危险

一个多宿主、多前台、多恢复路径的 runtime，至少有几类完全不同的消费者：

- 模型请求面
- UI 渲染面
- 宿主控制面
- 搜索 / 检索面
- snapshot / resume 面

如果强行让它们共享同一份全量表示，马上会出现三类污染：

### 2.1 职责污染

模型需要的是：

- cache-stable
- schema-legal
- 可计费的 request surface

UI 需要的是：

- 用户可理解的 render truth

resume 需要的是：

- 当前状态快照

这三类目标根本不一样。

### 2.2 演化污染

只要所有消费者吃同一份全量表示，任何一层的内部调整都会牵动全链：

- prompt section 重排
- tool schema 变化
- metadata 增减
- event subtype 演化

系统会很快失去局部演化能力。

### 2.3 治理污染

一些对象本来就不该被所有人看见：

- internal control message
- managed-only inputs
- tool-only metadata
- pending action snapshot

把它们全塞进同一份全景，只会让权限、安全、token 和前台真相重新混成一团。

## 3. Claude Code 的真实选择：单一合同，多种投影

Claude Code 更成熟的地方，在于它反复做同一件事：

- 先建立单一合同
- 再生成多种投影

这条原则同时体现在三处。

### 3.1 prompt 不是一份共享长文本

它先有：

- prompt contract
- section 编译
- cache boundary

再落成真正的 request surface。

所以模型看到的 protocol transcript，不等于 UI 看到的 transcript。

### 3.2 宿主控制面不是内部消息全集镜像

它先有：

- 协议全集
- 权威控制主路径

再允许 bridge、direct connect、`RemoteSessionManager` 只接各自的子集。

所以声明能力不等于接入能力。

### 3.3 状态真相也不强求单一表示

它先维护：

- 事件时间线
- 当前状态快照

再让不同消费者各取所需。

所以“发生过什么”和“现在是什么状态”既同源，又不被强行写成一份文本。

## 4. 苏格拉底式追问

### 4.1 如果所有消费者都看同一份全量真相，系统真的会更简单吗

表面更简单，实际更脆。

因为所有职责都被绑进同一表示里了。

### 4.2 如果模型、UI、resume 的目标不同，为什么还要强迫它们共享同一种表示

没有必要。
共享权威合同已经够了。

### 4.3 “单一权威”是不是会天然导向“单一全景”

不会。

单一权威解决的是：

- 真相源头不能分裂

单一全景试图解决的是：

- 所有人都用同一个外观

这两件事不是一回事。

### 4.4 真正成熟的 runtime 更该追求哪一个

显然是：

- 单一权威

而不是：

- 单一全景

## 5. 对 Agent 设计者的启发

如果想学 Claude Code，最值得学的不是：

- 把所有状态、prompt、事件拼成一个“大而全”的统一对象

而是：

1. 先定义权威合同。
2. 再定义真正执行合同的主路径。
3. 最后按消费者职责生成诚实子集。
4. 永远不要把“共享权威”误写成“共享同一表示”。

## 6. 一句话总结

Claude Code 的真相治理之所以成熟，不在于它把一切压成单一全景，而在于它坚持单一权威，再允许多消费者按职责分层暴露真相。
