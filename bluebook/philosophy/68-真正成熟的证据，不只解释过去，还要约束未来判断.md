# 真正成熟的证据，不只解释过去，还要约束未来判断

这一章回答五个问题：

1. 为什么“解释过去发生了什么”还不够构成成熟证据。
2. 为什么成熟证据必须同时服务宿主、CI、评审与后来接手者。
3. 为什么对象、窗口、字节与回退边界，是证据真正该约束的四类东西。
4. 为什么没有这层约束，系统会重新退回多处半真相。
5. 这对 agent runtime 设计者意味着什么。

## 0. 代表性源码锚点

- `claude-code-source-code/src/utils/sessionState.ts:92-146`
- `claude-code-source-code/src/services/api/promptCacheBreakDetection.ts:243-360`
- `claude-code-source-code/src/query/tokenBudget.ts:1-82`
- `claude-code-source-code/src/services/api/sessionIngress.ts:57-170`
- `claude-code-source-code/src/utils/task/framework.ts:160-248`

这些锚点共同说明：

- Claude Code 的证据之所以更成熟，不是因为它记录得更多，而是因为它持续保存未来判断所需的对象、窗口、字节与回退边界。

## 1. 先说结论

真正成熟的证据，不是：

- 帮你复盘一遍过去

而是：

1. 让当前宿主知道现在该怎么做。
2. 让 CI 知道现在该不该放行。
3. 让评审者知道当前结论有没有站住。
4. 让后来者知道接下来该从哪里继续。
5. 让失败发生时，团队知道该回退哪个对象。

从第一性原理看，成熟证据真正做的是：

- 约束未来判断

## 2. 第一层：如果证据只解释过去，它就仍然是叙事

很多系统的“证据”其实只是：

- 一次复盘
- 一次汇报
- 一次事故总结

这些东西当然有价值，但它们还不够成熟。

因为成熟证据真正要回答的是：

- 现在基于这份材料，系统下一步该继续、暂停、升级对象还是回退

如果答不出这个问题，所谓证据就还停留在：

- 解释层

而没有进入：

- 约束层

## 3. 第二层：成熟证据必须同时服务不同消费者

证据只有被不同消费者共享，才会真正变成系统的一部分。

如果：

1. 宿主看一套状态
2. CI 看一套阈值
3. 评审看一套总结
4. 交接者看一套 transcript

那么系统就会重新长出：

- 四套局部真相

所以成熟证据并不是：

- 所有人都看同一页面

而是：

- 所有人都围绕同一套对象、窗口、字节与回退边界判断

## 4. 第三层：对象、窗口、字节与回退边界，是成熟证据的四个硬约束

更成熟的证据至少要同时约束：

1. 对象
   - 这次升级到底在改什么。
2. 窗口
   - 这次判断基于哪一段观测。
3. 字节
   - 这次真正发生变化的治理字节是什么。
4. 回退边界
   - 如果坏了，到底撤回哪个对象。

这四件事缺任何一件，证据都会很快退化。

没有对象，就会：

- 大家都在讲这次升级，却不是同一个对象

没有窗口，就会：

- 大家都在看同一组数据，却不知道结论对应哪段现场

没有字节，就会：

- 大家都在说变了，却说不清变了什么

没有回退边界，就会：

- 一到失败，只剩文件和 commit，而没有对象真相

## 5. 第四层：真正危险的不是证据不够多，而是证据不再约束未来

一个系统最危险的时刻，不是没有任何记录，而是：

- 记录很多，但未来判断仍然各说各话

因为这意味着系统拥有：

- 信息

却没有：

- 制度

Claude Code 更值得学的地方，不是它会记更多东西，而是它让：

1. 当前真相可恢复
2. 关键字节可解释
3. 判断窗口可复查
4. 回退边界可指认

这就是证据开始约束未来判断的时刻。

## 6. 第五层：Prompt、安全与源码先进性都会在这里重新会合

Prompt 线最终要约束的是：

- 编译后 request truth

安全与省 token 线最终要约束的是：

- 当前决策窗口与仲裁顺序

源码先进性最终要约束的是：

- 当前对象真相与回退对象边界

所以这三条线最终都不是在要求：

- 更多解释

而是在要求：

- 更强的未来判断约束

这正是成熟证据最深的统一点。

## 7. 苏格拉底式追问

### 7.1 为什么“解释清楚了”仍然不够

因为解释清楚只能帮助理解，而不能自动帮助：

- 继续
- 停止
- 放行
- 回退
- 接手

### 7.2 为什么成熟证据必须服务不同消费者

因为只有这样，它才不再属于某个作者，而开始属于系统本身。

### 7.3 为什么这会成为新的高级工程判断

因为一旦证据开始约束未来判断，系统就不再只是一次次成功运行，而是真正在长期演化中保持判断连续性。

## 8. 对 Agent 设计者的启发

如果想学 Claude Code，最该抄走的不是：

- 更多日志
- 更复杂报表
- 更漂亮复盘

而是：

1. 让证据总是先指向对象。
2. 让证据总是绑定观察窗口。
3. 让证据总是能点名关键字节。
4. 让证据总是包含回退边界。
5. 让这套证据能继续约束宿主、CI、评审与后来者的未来判断。
