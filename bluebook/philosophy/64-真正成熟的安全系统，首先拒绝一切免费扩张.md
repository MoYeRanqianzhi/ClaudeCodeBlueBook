# 真正成熟的安全系统，首先拒绝一切免费扩张

这一章回答五个问题：

1. 为什么安全与省 token 的更深共同点不是“都在定价”，而是“都在拒绝免费扩张”。
2. 为什么动作、能力、上下文与时间都必须被统一看成稀缺资源。
3. 为什么没有决策增益的检查，本身就是风险和浪费。
4. 为什么一个允许免费扩张的系统，即使看起来更自由，也会更脆弱。
5. 这对 agent runtime 设计者意味着什么。

- `claude-code-source-code/src/utils/permissions/permissionSetup.ts:235-1283`
- `claude-code-source-code/src/utils/permissions/permissions.ts:593-984`

- `claude-code-source-code/src/services/api/promptCacheBreakDetection.ts:224-520`

这些锚点共同说明：

- Claude Code 的安全与省 token 真正成熟的地方，在于它沿着同一条治理收费链，系统性拒绝未定价的动作、可见性、上下文、时间与残留。

## 1. 先说结论

真正成熟的安全系统，不是：

- 更会阻止动作

而是：

- 它不允许免费扩张获得合法入口

把这句话压回 canonical chain，就是：

- `governance key -> externalized truth chain -> typed ask -> decision window -> continuation pricing -> durable-transient cleanup`

也就是说：

1. `governance key`
   - 不让未被授权的边界变更免费发生。
2. `externalized truth chain`
   - 不让 mode、visibility、pending action、usage 的投影免费宣布自己是真相。
3. `typed ask`
   - 不让未定价动作混进执行主路径。
4. `decision window`
   - 不让没有决策增益的上下文继续免费占位。
5. `continuation pricing`
   - 不让时间与 token 免费续命。
6. `durable-transient cleanup`
   - 不让陈旧残留在 stop、compact、resume 后继续免费复活。

## 2. 动作与能力为什么不能免费扩张

如果任何动作都能在“先做了再说”的逻辑里进入系统，后面的审批和分类只是补救。

Claude Code 更成熟的地方在于，它先问：

- 这个动作配不配进入当前治理顺序

这首先对应：

- `typed ask`

而在动作真正进入 ask 之前，系统还会先通过：

- `governance key`
- `externalized truth chain`

决定当前到底谁配改边界、哪些能力配暴露给模型。

所以真正强的安全，不是动作都更难发生，而是：

- 未经定价的动作和能力根本不配进入同一条执行主路径

## 3. 上下文与时间为什么不能免费扩张

很多 token 浪费看起来来自回答更长，实则来自：

- 不该进入的事实进入了主 prompt
- 本该外置的结果继续占着上下文
- 本该升级对象的工作继续假装值得延长

所以真正成熟的省 token，不是把回答裁短，而是：

- 不给没有决策增益的字节长期霸占当前工作窗口

这首先对应：

- `decision window`

而当 decision gain 已经失效后，系统还必须继续问：

- 这项工作还配继续吗
- 该 `stop / continue / object upgrade / human-fallback / cleanup-before-resume` 哪一种

这对应：

- `continuation pricing`

所以“时间不能免费扩张”不是体验口号，而是：

- continuation 是否仍然付得起制度价格

## 4. 为什么残留也不能免费扩张

免费扩张最隐蔽的一层，常常不是动作或时间，而是残留对象继续偷偷活着：

- 旧 tool result 没被外置
- 旧上下文在 compact 后偷偷回流
- 旧状态在 resume 后假装仍有当前资格

Claude Code 更成熟的地方在于，它把 stop、compact、resume 看成：

- `durable-transient cleanup`

也就是说，系统不仅要决定这轮是否继续，还要决定：

- 什么该留下
- 什么必须退场
- 什么只能以外部可回钉对象存在

如果这一步失效，免费扩张就会以“恢复成功”的名义悄悄回来。

## 5. 这页真正要防的三种失真

这一页的治理 reject trio 应落成：

1. `free-expansion relapse`
   - 主 reject
   - 动作、可见性、上下文、时间、残留重新获得免费扩张资格。
2. `decision-window collapse`
   - `Context Usage` 只剩成本显示，不再决定当前是否继续。
3. `projection usurpation`
   - classifier、mode 条、token 条或 UI 投影开始冒充治理主权。

如果只说“统一预算器”，还不够。

因为预算器只告诉你：

- 都要被定价

却还没告诉你：

- 哪些扩张压根不该被允许发生

## 6. 苏格拉底式追问

### 6.1 为什么自由如果不定价，最终会反过来伤害行动

因为免费扩张看似给了更多空间，实际上是在制造：

- 更高的误判率
- 更低的解释力
- 更贵的继续成本

### 6.2 为什么安全与省 token 的深层共同点不是“保守”，而是“拒绝免费”

因为真正成熟的系统不是凡事都更紧，而是：

- 只允许付过制度价格的东西进入运行时真相

### 6.3 为什么自动化不能在无增益时继续免费存在

因为继续执行如果不再创造决策增益，就不再是效率，而是：

- 免费冒险
- 免费烧 token
- 免费扩大错误面

## 7. 对 Agent 设计者的启发

如果想学 Claude Code，最该抄走的不是：

- 再叠一层限制器

而是：

1. 先识别哪些边界变更在免费发生。
2. 先识别哪些能力暴露在免费扩张。
3. 先识别哪些字节在免费霸占 decision window。
4. 先识别哪些 continuation 在免费拖长时间。
5. 先识别哪些 stop / compact / resume 残留还在免费复活。

## 8. 一句话总结

真正成熟的安全系统，首先拒绝一切免费扩张；而 Claude Code 的成熟之处，就在于它把这件事沿 `governance key -> externalized truth chain -> typed ask -> decision window -> continuation pricing -> durable-transient cleanup` 一段段写硬了。
