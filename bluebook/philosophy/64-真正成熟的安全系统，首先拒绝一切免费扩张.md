# 真正成熟的安全系统，首先拒绝一切免费扩张

这一章回答五个问题：

1. 为什么安全与省 token 的更深共同点不是“都在定价”，而是“都在拒绝免费扩张”。
2. 为什么动作、能力、上下文与时间都必须被统一看成稀缺资源。
3. 为什么没有决策增益的检查，本身就是风险和浪费。
4. 为什么一个允许免费扩张的系统，即使看起来更自由，也会更脆弱。
5. 这对 agent runtime 设计者意味着什么。

## 0. 代表性源码锚点

- `claude-code-source-code/src/utils/permissions/permissionSetup.ts:235-1283`
- `claude-code-source-code/src/utils/permissions/permissions.ts:593-984`
- `claude-code-source-code/src/query/tokenBudget.ts:1-93`
- `claude-code-source-code/src/utils/toolResultStorage.ts:367-860`
- `claude-code-source-code/src/services/compact/autoCompact.ts:257-338`
- `claude-code-source-code/src/services/api/promptCacheBreakDetection.ts:224-520`

这些锚点共同说明：

- Claude Code 的安全与省 token 真正成熟的地方，在于它系统性地拒绝未定价的动作、能力、上下文与 continuation。

## 1. 先说结论

真正成熟的安全系统，不是：

- 更会阻止动作

而是：

1. 不给未定价动作合法入口。
2. 不给未定价能力免费暴露面。
3. 不给未定价上下文长期霸占主 prompt。
4. 不给未定价 continuation 无限续命。
5. 不给无决策增益的自动化继续消耗资源。

从第一性原理看，安全与省 token 的共同问题不是：

- 如何更省
- 如何更严

而是：

- 如何拒绝一切免费扩张

## 2. 第一层：动作不能免费扩张

如果任何动作都能在“先做了再说”的逻辑里进入系统，后面的审批和分类只是补救。

Claude Code 更成熟的地方在于，它先问：

- 这个动作配不配进入当前治理顺序

所以真正强的安全，不是动作都更难发生，而是：

- 未经定价的动作根本不配进入同一条执行主路径

## 3. 第二层：能力不能免费扩张

一个系统最容易自欺的地方是：

- 能力先全量暴露，后面再靠 prompt 和人工记忆去约束

Claude Code 的做法更像：

- 先缩小可见面
- 再让模型在已定价世界里行动

这说明它真正拒绝的是：

- 免费能力扩张

因为一旦能力免费暴露，成本、安全与解释能力会一起失控。

## 4. 第三层：上下文不能免费扩张

很多 token 浪费看起来来自回答更长，实则来自：

- 不该进入的事实进入了主 prompt
- 本该外置的结果继续占着上下文
- 本该被忘掉的历史继续免费存活

所以真正成熟的省 token，不是把回答裁短，而是：

- 不给不值钱的字节长期霸占上下文席位

这同样是一条安全原则，因为被免费放进 prompt 的世界，也是在免费改写模型的决策边界。

## 5. 第四层：时间不能免费扩张

continuation、auto mode、自动化链路如果没有停止条件，就会把：

- 本该中止的判断
- 本该交还给人的主权
- 本该升级对象的任务

都继续拖在同一条时间线上。

这是一种最隐蔽的免费扩张：

- 免费占用时间

而时间一旦被免费延长，token、风险与上下文污染都会一起上升。

## 6. 第五层：自动化不能在无增益时继续免费存在

真正成熟的系统不会把自动化视为：

- 一旦开了就该尽量别停

它会继续追问：

- 这条自动链路还在创造决策增益吗

如果答案是否定的，那么继续执行就不再是效率，而是：

- 免费冒险
- 免费烧 token
- 免费扩大错误面

所以成熟自动化的本质不是覆盖人工，而是：

- 在无增益处主动退场

## 7. 苏格拉底式追问

### 7.1 为什么只说“统一预算器”还不够

因为预算器只告诉你：

- 都要被定价

却还没告诉你：

- 哪些扩张压根不该被允许发生

### 7.2 为什么自由如果不定价，最终会反过来伤害行动

因为免费扩张看似给了更多空间，实际上是在制造：

- 更高的误判率
- 更低的解释力
- 更贵的继续成本

### 7.3 为什么安全与省 token 的深层共同点不是“保守”，而是“拒绝免费”

因为真正成熟的系统不是凡事都更紧，而是：

- 只允许付过制度价格的东西进入运行时真相

## 8. 对 Agent 设计者的启发

如果想学 Claude Code，最该抄走的不是：

- 再叠一层限制器

而是：

1. 先识别哪些动作在免费扩张。
2. 先识别哪些能力在免费暴露。
3. 先识别哪些字节在免费霸占上下文。
4. 先识别哪些 continuation 在免费拖长时间。
5. 让自动化在无增益时主动退场，而不是把“继续运行”误当成成熟。
