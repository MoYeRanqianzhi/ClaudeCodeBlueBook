# Prompt 前门速记：先定合法输入，再谈提示词优化

本页只保留 Prompt 前门的一屏速记与兼容跳转，不再和 `philosophy/84` 并列争首答；真正的 Prompt canonical source 仍回 `84`。

Prompt 最容易被误写成：

- 一段更会组织措辞的文案

但 Claude Code 更接近的真相是：

- Prompt 的效力来源，不在说服模型，而在先规定世界如何进入模型

## 1. 第一性原理

如果系统已经有：

- 很长的 system prompt
- 很多技巧性措辞

却还没有先规定：

- 谁能发言
- 什么能进入稳定前缀
- UI 历史怎样被编译成协议真相
- 删掉什么以后仍能合法继续

那么它依旧只是：

- 一段更长的输入

而不是：

- 一个真正成立的 Prompt Runtime

## 2. 为什么这比“提示词优化”更接近真相

Claude Code 的高级感来自：

1. 世界先被编译
2. 边界先被固定
3. 继续条件先被保护
4. 失稳原因还能被解释

所以 prompt 效力真正强的地方不是：

- 这段话更像文案技巧

而是：

- 模型从一开始就没有机会看见一个失序的世界

## 3. 为什么合法遗忘也属于 Prompt 效力

真正强的 Prompt 不只会规定“现在怎么看世界”，还会规定：

- 删掉什么以后，下一轮依旧知道该怎样继续

所以 compact、resume、handoff 不是 Prompt 之外的补丁，而是 Prompt 设计的一部分。

## 4. 苏格拉底式自反

在你准备宣布“我们已经把 Prompt 优化得很强”前，先问自己：

1. 我优化的是措辞，还是上下文准入顺序。
2. 模型消费的是稳定协议真相，还是显示层历史。
3. 旁路循环消费的是共享前缀，还是各自重建的半真相。
4. 我是否已经知道删掉什么以后系统还能合法继续。
