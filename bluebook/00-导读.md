# 导读

## 1. 研究对象

> Evidence mode
> - `mirror present`：当前 worktree 含 `claude-code-source-code/` 时，优先按本地镜像核对版本锚点、源码锚点与运行时对象。
> - `public-evidence only`：当前 worktree 不含该目录时，本页只承认蓝皮书正文、`docs/` 归档锚点与已公开材料，不把缺席镜像写成当前在场事实。

本蓝皮书研究的对象设计上对应 `claude-code-source-code/` 镜像；当本地镜像在场时，它通常以 `package.json`、源码树与 README 提供版本锚点与运行时证据。若当前 worktree 缺席该目录，先回 [../docs/README.md](../docs/README.md) 与 `docs/development/02-证据索引` 查看已归档锚点，不在入口层假装已经完成本地核证。

研究方法、版本锚点与证据索引单独归档；本页只保留主线前言，不再承担第二套导航与研究工作台职责。
根入口的默认首答顺序仍是 `问题分型 -> 工作对象 -> 控制面 -> 入口`；`00-导读` 只负责把这组顺序落成主线前言，不单独改判 deeper page 的 truth / verdict。

## 2. 先带着三条判断进入蓝皮书

如果只先带三句判断进入整本蓝皮书，也只先记自然语言版：

1. Prompt 强不在 secret text，而在同一个世界能否被持续编译、验证并合法继续。
2. 治理强不在 approval popup，而在 `pricing-right`、`truth-surface attestation` 与 `rollback ceiling` 是否先把扩张写成合法 contract。
3. 源码质量强不在目录观感，而在当前真相、恢复资产与旧写入者是否已被正式分权。

这里还应再多记一句：

- `continuity` 不是第四条进入主题；它只是 Prompt `Continuation`、治理 `continuation pricing` 与源码质量 `recovery non-sovereignty / anti-zombie` 在时间轴上的共同继续问题。

更硬的 canonical formula、最小顺序与 first reject signal 统一回 [README](README.md)；本页不再承担第二套入口职责。

## 3. 为什么要先画结构图

Claude Code 的强，不是单点 prompt 技巧，而是很多工程机制叠加出来的:

- 启动阶段极度在意 fast path。
- 核心 agent loop 足够小，但外围 harness 很厚。
- 工具、技能、插件、MCP 不是孤立系统，而是共享一套运行时装配逻辑。
- 多 Agent、远程、上下文压缩、权限系统都不是外挂，而是主路径的一部分。

如果先不把结构图画清，后面写“使用指南”和“设计哲学”会不可避免地漂浮。

## 4. 阅读后的第一判断

Claude Code 的本质不是“会写代码的聊天框”，而是一个把以下对象统一纳入一个 runtime 的系统:

- conversation transcript
- tool registry
- permission context
- slash commands and skills
- dynamic MCP connections
- background tasks and subagents
- compaction and resume
- remote and bridge transports

如果再往上压缩一层，这套 runtime 最难被抄走的，不是功能数量，而是三条制度判断：

1. 世界怎样进入模型
2. 扩张怎样进入当前世界
3. 过去与失控复杂度怎样不得写坏当前世界

所以更稳的源码阅读顺序不是：

- 先看哪个模块看起来更复杂

而是：

1. 先看谁在宣布真相
2. 再看谁只是消费真相
3. 再看哪些恢复资产与旧写入者可能重新篡位
4. 最后才看目录体感、文件大小与功能表覆盖率

如果你在入口层最先看到的是长文案、mode 条、token 百分比、长摘要或目录观感，而不是 `Authority -> Boundary -> Transcript -> Lineage -> Continuation -> Explainability`、`decision window` 或 `authority surface / temporal honesty`，那就说明你已经在更外层开始学错对象了。
如果一个入口判断还答不上“这次先在保护哪个工作对象、哪条控制面、哪个入口只是 route 而不是 signer”，就说明它还没有压回第一性原理。

一级阅读路径看 [README](README.md)，跨目录深链看 [navigation/README.md](navigation/README.md)，方法附录与证据归档另见 [../docs/README.md](../docs/README.md)。
如果你还分不清现在是在校正模仿对象、寻找第一条反证信号，还是要把高阶结论压回第一性原理，先回 `navigation/05 -> navigation/15 -> navigation/41`，再回来继续读根目录正文。
