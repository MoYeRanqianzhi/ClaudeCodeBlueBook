# 速查卡：错误语义、用户动作与支持路径

## 1. 用法

这一页不是详细解释，而是一页式速查卡。  
当你或读者遇到“像被封了”的情况时，先看这里，再决定回到哪一章深读。

## 2. 错误语义速查表

| 表现 | 更可能的真实含义 | 首先应该做什么 | 主要支持路径 |
| --- | --- | --- | --- |
| `401` | token 过期、身份连续性断裂、刷新失败 | 停止切换账号和 token，核对身份源与组织 | 登录/账号支持 |
| `403` | access denied、组织权限不足、scope 不足 | 判断是否是 `insufficient_scope` 或高安全链路拒绝 | 组织/授权支持 |
| `not enabled for your account` | entitlement 缺失、rollout 未开放 | 不要先频繁重登，先判断是否本就未开放 | 套餐/能力开放支持 |
| `disabled by your organization's policy` | 组织治理结果 | 不要尝试本地覆盖，联系管理员确认 | 管理员 / IT |
| `needs-auth` | 某个连接域需要重新认证 | 视为子系统认证问题，不要当成全局封号 | 该连接域支持 |
| `rate limit` / `out of extra usage` | 配额或额外用量耗尽 | 先看套餐、重置时间、overage 状态 | 计费/套餐支持 |
| risky action blocked | 本地执行安全阻断 | 看 permission / sandbox / trust | 本地配置与安全诊断 |
| remote env unavailable | 高安全远程能力域不成立 | 看 entitlement、组织、设备信任 | Remote Control / bridge 支持 |

## 3. 用户动作速查表

| 你现在看到的现象 | 不该做什么 | 更好的动作 |
| --- | --- | --- |
| 一类错误刚出现 | 立刻再试十次 | 先记录时间和错误语义 |
| 怀疑身份出问题 | 同时换账号、换 token、换 provider | 固定一个身份路径逐层排查 |
| 怀疑组织限制 | 改本地设置硬顶 | 去确认 policy 与组织要求 |
| Remote Control 出问题 | 把它当成普通 CLI 故障 | 单独按高安全远程链路处理 |
| 觉得自己被误伤 | 继续让变量变得更多 | 冻结设备/网络/账号/组织变量 |
| 想找支持 | 只说“不能用了” | 准备时间线、错误、套餐、组织、设备信息 |

## 4. 高波动网络环境用户速查表

| 风险点 | 为什么更容易放大误伤体感 | 合规降低风险的做法 |
| --- | --- | --- |
| 网络频繁波动 | 更容易触发 token/remote session 不连续 | 固定主力网络与主力设备 |
| 多身份并存 | 更容易制造身份源混乱 | 一个环境只保留一条主身份路径 |
| 组织与个人路径混用 | 更容易触发 entitlement/组织错配 | 工作与个人账号职责分离 |
| 高安全远程能力频繁试错 | 更容易制造 fatal / reconnect 噪声 | 先确认资格，再进入高安全链路 |
| 缺少证据保留 | 更难向支持解释 | 提前保留付款、时间线、错误截图/文本 |

## 5. 证据保全最小清单

最少要保留：

1. 错误发生时间
2. 错误原文
3. 当前账号类型
4. 当前组织或是否属于组织环境
5. 当前设备与网络环境是否发生切换
6. 是否涉及 remote-control / bridge / MCP / auto mode
7. 套餐和付款信息

## 6. 一句话总结

先分清语义，再做动作；先保护证据，再找支持。  
不要把所有失败都当成封号，也不要把所有问题都试成封号。
