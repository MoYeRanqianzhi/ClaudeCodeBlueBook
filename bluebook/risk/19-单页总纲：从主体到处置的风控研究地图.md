# 单页总纲：从主体到处置的风控研究地图

## 1. 这页是做什么的

这一页把整个 `risk/` 专题压缩成一张“研究地图”。

如果你只想带走最核心的框架，而不想逐章展开，这一页就是总纲。

## 3. 一条总逻辑

可以把整套系统浓缩成一句更短的话：

先识别你是谁，再判断你配不配，再判断组织允不允许，再决定动作能不能做，最后决定失败要怎么恢复或收口。

这就是 Claude Code 风控的最小核心逻辑。

## 4. 用户最重要的三条行动原则

1. 不要把所有错误都当成“被封号”。
2. 不要在身份、组织、设备、网络都在变化时继续试错。
3. 不要在没有证据包的情况下就开始申诉或找支持。

## 5. 构建者最重要的三条行动原则

1. 不要把资格、组织策略、动作阻断和真正处罚揉成一个状态。
2. 不要只做判定，不做解释和恢复。
3. 不要把失败风暴当成运维细节，它本身就是治理对象。

## 6. 中国用户最值得记住的一句话

在高波动跨境环境里，最能保护自己利益的不是寻找绕过系统的方法，而是让自己的身份、组织、设备、网络和证据始终保持稳定、可解释、可支持。

## 7. 这个专题最后想证明什么

Claude Code 的“封号/风控”并不是一个 ban switch，  
而是一套围绕主体、资格、组织、动作、高安全会话、失败控制和平台正义展开的分层治理架构。
