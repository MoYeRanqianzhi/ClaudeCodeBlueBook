# 风控总论：封禁不是单点开关

## 1. 先说结论

从 Claude Code 源码看，“风控”不是一个独立的 ban switch，而是同一损失函数在用户侧的五种显形：

1. 身份有效性：OAuth token、新鲜度、组织绑定、订阅类型。
2. 服务端策略：policy limits、remote managed settings、GrowthBook gates。
3. 本地安全执行：permission pipeline、auto mode classifier、sandbox、trust。
4. 高安全会话：Remote Control、bridge、trusted device、elevated auth。
5. 持续观测：telemetry、1P event logging、后台轮询、缓存刷新。

它们看起来像五类机制，实际上共同在回答一件事：

- 当前主体还能合法走多远
- 哪些动作与能力还配继续
- 继续下去会把什么风险、成本与解释负担转嫁给系统和用户

所以源码里更常见的不是“此账号已封禁”的显式判断，而是：

- 401/403 导致的鉴权失败或会话终止
- 404 导致的能力不可用或环境不可注册
- 组织策略把某项能力直接关掉
- managed settings 强制改写本地行为边界
- 本地执行面在高风险动作处先行阻断

## 2. 客户端到底在管理什么风险

如果只看客户端代码，Claude Code 主要在管理四种风险：

### 2.1 身份风险

系统必须确认：

- 你是不是有效登录用户
- 这个 token 是否过期
- 这个 token 是否属于正确组织
- 这个账号是否具备对应订阅或组织资格

## 3. 代码里为什么几乎看不到“封号判断”

因为 Claude Code 客户端并不是最终裁决中心。

更准确地说，它在参与一条统一定价链：

1. 资格先被定价
2. 能力再被定价
3. 连续性能否被续租再被定价
4. 错误发生后由谁承担恢复与支持成本也被继续定价

它做的事情更像：

1. 采集当前用户、组织、设备、订阅与会话上下文。
2. 拉取服务端给出的 gate、settings、restrictions。
3. 在本地执行链路上加安全前置阻断。
4. 遇到服务端拒绝时，把拒绝解释成清晰的用户体验与状态迁移。

例如：

- `policyLimits` 和 `remoteManagedSettings` 都是“服务端拉取 -> 本地缓存 -> 后台轮询 -> fail open”的模式。
- bridge 模式把 401/403 视为 fatal 或 auth_failed，而不是继续乐观重试。
- trusted device 同时有 CLI 侧 gate 和服务端 gate，明显是分阶段 rollout 的双侧控制。

## 4. 这套风控体系的真实结构

可以把 Claude Code 的风控结构理解成四层：

### 4.1 入口资格层

- auth source
- token freshness
- subscription type
- organization UUID
- account UUID

这层回答的是“你有没有资格进入某个能力域”。

### 4.2 策略下发层

- GrowthBook runtime gates
- remote managed settings
- policy limits

这层回答的是“你所在组织、账号、实验桶当前被允许什么”。

### 4.3 执行仲裁层

- permission rules
- safety checks
- auto mode classifier
- sandbox
- trust gating

这层回答的是“即便能力存在，这次动作能不能执行”。

### 4.4 会话保活与持续观测层

- telemetry attributes
- 1P event logging
- BigQuery / OTLP / tracing
- background polling
- heartbeat / reconnect / fatal exit

这层回答的是“系统如何持续观察并重新配置前面三层”。

## 5. 从源码角度，什么现象更接近“封号”

更贴近“封号/限制”的源码表现包括：

1. token 失效后无法刷新，功能持续 401。
2. 组织强制登录到某个 org，验证失败后 fail closed。
3. policy limit 把某项能力显式标成 `allowed: false`。
4. Remote Control 环境注册 404，说明该账号没有该能力域。
5. multi-session、remote environments、bridge 等能力提示“not enabled for your account yet”。
6. elevated session 缺少 trusted device token 时远程高安全路径无法成立。

这些都不是传统意义上的“账号全局封停”字样，但对用户来说会呈现为非常明确的“被限制”。

## 6. 一句话总结

Claude Code 的风控本质不是“本地有个封号名单”，而是“身份、组织、策略、执行和会话基础设施共同决定你还能走到哪一层”。
