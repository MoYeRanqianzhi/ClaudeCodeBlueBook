# 体验与入口

这一层不该再是“体验相关页列表”，更稳的前门只有三句：

1. `host -> session -> projection -> display`
   - 先分宿主、会话、投影和显示层，不要把体验词写成主权词。
2. `tuning != truth`
   - model、effort、cost、style、rate 这类入口主要在调节行为和消费投影，不在单独签当前世界。
3. `multi-host != same surface`
   - IDE、终端、Vim、语音、桌面、浏览器、remote-control 都在消费同一个 runtime 的不同子集，不是同一张面板换皮。

如果你还在把 UI 体感、状态词或平台入口当成 truth source，就先不要继续靠目录名碰运气。
