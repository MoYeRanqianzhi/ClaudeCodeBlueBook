# 主线使用

这一部分按一个真实用户的工作顺序来组织：

1. 你怎样进入 Claude Code。
2. 你怎样给它任务、附件和上下文。
3. 你怎样区分普通提示词、斜杠命令、技能和工具。
4. 你怎样让会话在长时间工作后仍然可继续、可压缩、可恢复。
5. 你怎样在真正投入工作前确认状态、预算和工作边界。

如果把这一组主线再压成最短动作链，也只剩：

1. 先进入正确入口
2. 再把任务送进同一个工作对象
3. 再分清命令面、技能面和普通提示词
4. 再保证恢复、压缩与记忆仍服务同一条连续性
5. 最后才确认当前状态、预算和工作边界还配继续

更稳一点看，这五页也分别在承接三组最小顺序：

- `01 / 02 / 04`
  共同回答 Prompt 线怎样维持同一个工作对象。
- `05`
  回答治理线现在该继续、停、降级还是清理后恢复。
- `01 / 05`
  共同回答当前工作目录、worktree、会话状态与边界是不是还在同一个现场。
