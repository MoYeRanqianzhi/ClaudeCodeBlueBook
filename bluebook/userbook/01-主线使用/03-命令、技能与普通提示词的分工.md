# 命令、技能与普通提示词的分工

## 先抓住三种命令类型

`src/types/command.ts` 定义了 Claude Code 的三个命令类型：

- `local`
- `local-jsx`
- `prompt`

这三个类型决定了它们和模型的关系。

## `local`

`local` 命令本地执行，通常直接返回文本、状态变更或本地副作用，不需要经过模型继续推理。

典型例子：

- `/clear`
- `/compact`
- `/vim`
- `/reload-plugins`

使用判断：

- 如果事情是“立刻调整本地会话状态”，通常优先落在 `local`。

## `local-jsx`

`local-jsx` 命令会在本地打开一个 UI。它们往往承担“管理面板”的角色。

典型例子：

- `/config`
- `/permissions`
- `/mcp`
- `/plugin`
- `/agents`
- `/help`
- `/status`

使用判断：

- 如果操作需要列表、分页、切换、确认框、表单或状态视图，通常是 `local-jsx`。

## `prompt`

`prompt` 命令本质上是“受控的 prompt 注入器”。它们通常把一段预定义技能内容、工作流模板或特殊指令送给模型。

典型例子：

- `/commit`
- `/review`
- `/init`
- `/init-verifiers`

使用判断：

- 如果目标是“触发一段现成的模型工作流”，通常是 `prompt`。

## 技能和命令的关系

`src/commands.ts`、`src/skills/loadSkillsDir.ts`、`src/skills/bundledSkills.ts` 说明 Claude Code 的命令空间不是只有内置命令，还会把下列来源合并进来：

- 内置命令。
- bundled skills。
- 内置插件暴露的 skills。
- 项目技能目录中的 skills。
- 插件命令与插件 skills。
- 工作流命令。
- 动态技能。

这件事有两个很重要的后果：

1. `/help` 和补全看到的命令面不是固定死表，而是可扩展命令空间。
2. “技能”在 Claude Code 里不是旁白文档，而是正式运行时对象。

## 技能还有两种执行上下文

`src/types/command.ts` 中 `PromptCommand` 还定义了 `context`：

- `inline`
- `fork`

这意味着技能不一定在当前上下文内直接展开。

### inline

技能内容被并入当前会话，适合轻量工作流。

### fork

技能转交给子代理运行，适合：

- 长任务。
- 有独立 token 预算需求的任务。
- 不想污染主上下文的任务。
- 需要 worktree 或后台运行的任务。

## 从用户视角的分工规则

### 用普通提示词

当你要描述一个尚未完全定型的任务，并愿意让模型自己规划读文件、改代码和测试路径时。

### 用命令

当你知道自己要做的是“系统操作”，不是“自然语言协商”时。

### 用技能

当你想复用一套稳定工作流，而不是每次重写要求时。

### 用插件或 MCP

当能力必须扩展到 Claude Code 自身以外的系统边界时。
