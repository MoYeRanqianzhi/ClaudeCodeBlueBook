# 进入系统：启动、会话与工作目录

## 第一事实：Claude Code 不是只有一个入口

`src/entrypoints/cli.tsx` 明确展示了多个快路径入口：

- 纯版本输出。
- 普通 CLI / REPL。
- `-p/--print`、结构化 I/O 与非交互输出。
- `remote-control` / `bridge`。
- `daemon`。
- 后台会话管理，如 `ps`、`logs`、`attach`、`kill`。
- 模板作业、环境运行器、自托管 runner。
- `--worktree --tmux` 的特殊快路径。

这意味着用户从一开始就不该把 Claude Code 只理解成“打开终端后聊天”。

如果你当前主要卡在：

- root flags
- root commands
- slash commands

到底先分哪一层，

先回 [../00-阅读路径.md](../00-%E9%98%85%E8%AF%BB%E8%B7%AF%E5%BE%84.md) 里的“路径 14：我想分清 root flags、root commands 和 slash commands”。

## 第二事实：会话是一级对象，不是文本历史

从 `src/commands/resume/index.ts`、`src/commands/session/index.ts`、`src/utils/sessionState.ts`、`src/Task.ts` 能看出，系统把会话、任务、后台任务输出、恢复点都当成可寻址对象来管理。

对用户的直接含义：

- 你可以恢复旧会话，而不是从零重开。
- 你可以把当前工作迁移到别的前端，如桌面端、移动端、远端会话 viewer，或把本机暴露给远程控制端。
- 你可以在同一大任务下保留多个分叉、标签、检查点和回滚点。

## 第三事实：工作目录不是静态背景

Claude Code 对工作目录非常敏感：

- `src/commands/add-dir/index.ts` 允许把新目录加入工作集。
- `src/tools/EnterWorktreeTool/EnterWorktreeTool.ts` 允许在会话中途创建并切入隔离 worktree。
- `src/tools/ExitWorktreeTool/ExitWorktreeTool.ts` 允许退出 worktree 返回原目录。
- `src/utils/cwd.js`、`src/utils/Shell.js`、`src/utils/worktree.js` 把当前目录、shell 执行目录、隔离目录明确区分开。

这说明 Claude Code 的一个核心用法不是“站在一个仓库里一直改”，而是：

- 在主仓库里探索。
- 在需要把更多目录正式纳入当前工作面时，先用 `/add-dir` 扩边界。
- 在需要隔离的时候切进 worktree。
- 在子代理或批量执行场景下，让不同工作单元各自有独立目录面。

这里最容易混淆的两件事其实不同：

- `/add-dir` 是扩张当前工作面。
- `EnterWorktree` 是隔离实现面。

## 用户应怎样选择入口

### 交互式 REPL

适合大多数开发工作。你在这里得到完整的命令面、上下文可视化、权限审批和工具编排。

### 非交互模式

适合一次性任务、自动化调用和外部流水线。源码里多处把非交互路径单独处理，说明它不是附带模式，而是正式入口。

这里最容易误判的是：

- `claude doctor`
- `claude -p doctor`

看起来都像在“运行 doctor”，

但前者是 root command，

后者已经先被 `-p/--print` 改写成默认 action 路径。

### 远端会话与桥接入口

这里至少有两种不同目标：

- 让另一台机器提供执行环境，此时更接近 remote session / teleport。
- 让当前终端变成宿主，被 web、mobile 或 Claude app 接入，此时才是 `remote-control` / bridge。

更稳的用法是先分清自己要“连到哪台机器”，再选入口，而不是把所有远程相关能力都当成同一种模式。

### worktree 入口

适合需要隔离改动、并行实现、批量代理执行、风险控制的场景。它不是“git 技巧”，而是 Claude Code 运行时的一部分。

如果你还不需要新代码面，只是要让另一片目录正式进入当前工作面，应先用 `/add-dir`，而不是一上来就切 worktree。

## 最低摩擦的进入姿势

从源码推回来的最佳主线通常是：

1. 在主仓库启动交互式会话。
2. 用普通提示词或 `/help`、`/status`、`/context` 快速摸清状态。
3. 任务变复杂时转入计划模式或 worktree。
4. 任务足够长时依赖 `/resume`、`/compact`、`/memory` 保持连续性。
