# 内置技能索引

以下基于 `src/skills/bundled/*` 与 `src/skills/bundled/index.ts`。

边界提醒：

- 这一页只回答“当前源码里有哪些 bundled skills”。
- 它是库存页，不承担能力曝光或发现机制前门。
- 它不等于 `/skills` 菜单总表，也不等于 Claude 当前整套 skill universe。
- 如果你想分清 file-based / plugin / bundled / MCP / dynamic skills 的差异，请结合 `02-技能来源、可见性与触发索引.md` 阅读。

## 常驻 bundled skills

| 技能 | 作用 |
| --- | --- |
| `verify` | 变更后的验证工作流 |
| `update-config` | 改 settings 配置、权限、hooks、环境变量 |
| `keybindings-help` | 帮你改快捷键配置 |
| `remember` | 审查与晋升记忆 |
| `simplify` | 审查并简化改动 |
| `skillify` | 把当前工作方式沉淀成技能 |
| `debug` | 调试导向技能 |
| `batch` | 大规模并行 worktree 代理执行 |
| `lorem-ipsum` | 长上下文测试用填充文本 |
| `claude-in-chrome` | 控制 Chrome 与网页交互 |
| `claude-api` | 构建 Claude API / SDK 应用 |

## 为什么 bundled skills 重要

因为它们说明 Claude Code 的“最佳实践”并不只写在文档里，而是已经被编译进运行时：

- 有正式名字。
- 有正式描述。
- 有工具白名单。
- 有可选模型与执行上下文。
- 有文件提取与 hook 接入能力。

这使 bundled skills 成为理解 Claude Code 官方工作流偏好的最佳切口之一。
