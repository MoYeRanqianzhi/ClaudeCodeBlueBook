# 命令类型、执行语义与可见性索引

这页只做速查，不替代长文专题。长文解释请结合：

- `05-控制面深挖/11-命令对象、执行语义与可见性：为什么 slash command 不是同一种按钮.md`
- `03-参考索引/02-能力边界/02-Feature Gate 与可见性索引.md`

如果这一组索引里你只先看一页，就先看这页。

先记这一页的最短公式：

- `command object -> execution semantics -> visibility/caller gate -> host mode`

把整组再往上压一层，可记成：

- `plane -> object -> semantics -> gate`

这里也要先压住一个常见误读：`/compact / /resume / /memory` 在命令类型层不是第四类命令对象；它们只是同一工作对象在时间轴上被压缩、恢复与续接的 continuation actions。

翻成速查语言就是：

- 先分 `prompt / local / local-jsx`
- 再分立即、本地、排队、fork、headless、remote、bridge
- 最后才分谁能看见、谁能调用、当前宿主会不会放行

更稳一点说，这一页也必须继承 `问题分型 -> 工作对象 -> 控制面 -> 入口` 的 first-answer order；如果对象和控制面还没先站住，命令类型表就会重新退回“按 slash 名称猜世界”。

## 1. 三类命令对象

| 类型 | 是否进入模型 | 典型输出 | 关键字段 | 典型例子 | 最易误判 |
| --- | --- | --- | --- | --- | --- |
| `prompt` | 会 | prompt 消息、meta 消息、附加工具权限 | `allowedTools` `model` `effort` `hooks` `context` `disableNonInteractive` | `/review` `/commit` `/statusline` | 误写成本地按钮 |
| `local` | 不会 | `text` `compact` `skip` | `supportsNonInteractive` `isSensitive` | `/cost` `/clear` `/compact` `/release-notes` | 误写成轻量 prompt |
| `local-jsx` | 不会 | 本地 Ink UI | `immediate` | `/config` `/status` `/doctor` `/mcp` | 误写成“有界面的 local” |

## 2. 执行语义字段

| 字段 | 主要作用对象 | 回答的问题 | 不等于什么 | 例子 |
| --- | --- | --- | --- | --- |
| `immediate` | 主要见于 `local-jsx` | 当前回合忙碌时，能否直接在本地宿主执行 | root command、headless-safe、bridge-safe | `/status` `/mcp` `/plugin` `/rename` |
| `context: 'fork'` | `prompt` | 是否在 forked sub-agent 中执行 | 第四种命令类型 | 运行时已支持，当前可见 builtin 注册并不普遍声明 |
| `disableNonInteractive` | `prompt` | headless 下是否从命令集合里去掉 | `supportsNonInteractive` 的反义词 | `/statusline` |
| `supportsNonInteractive` | `local` | headless 下是否保留 | remote-safe、bridge-safe | `/compact` `/cost` headless `/context` |
| `skipSlashCommands` | 输入处理链 | 当前输入是否先禁用 slash / exit-word / immediate 快路径解析 | bridge-safe | bridge inbound 输入默认开启 |
| `queryGuard` | 调度层 | 当前 turn 是否占用执行权、后续输入该排队还是继续执行 | 命令字段 | slash / bash 队列的串行闸门 |
| `allowedTools` | `prompt` | 这条 prompt/skill 额外授权哪些工具 | 所有工具都能自由用 | `/statusline` |
| `hooks` | `prompt` | skill 调用时是否注册 hooks | 普通会话设置 | skill frontmatter / bundled skill |
| `isSensitive` | `local` | transcript 是否对参数打码 | 隐藏命令 | 当前公开主线源码里几乎无显式案例 |

## 3. 可见性与调用者边界字段

| 字段 | 回答的问题 | 更接近哪类边界 | 典型例子 |
| --- | --- | --- | --- |
| `availability` | 谁能看见 | 静态资格 / 账户与 provider | `/desktop` `/usage` `/fast` |
| `isEnabled()` | 现在能不能用 | 动态运行时 / 平台 / policy / env | `/feedback` `ultrareview` `/chrome` |
| `isHidden` | 默认是否出现在 help/typeahead | 展示层可见性；知道准确名字时通常仍可直呼 | `/rate-limit-options` `/heapdump`，非交互镜像命令 |
| `userInvocable` | 用户能否直接 `/skill-name` | 人类调用权限 | `keybindings-help` 为 `false` |
| `disableModelInvocation` | 模型能否自动通过 SkillTool 调用 | 模型调用权限 | `debug` |
| `loadedFrom` | 命令从哪一类载入链来 | 来源元数据 / provenance，不直接改变执行 ABI | `skills` `bundled` `plugin` `mcp` |
| `source` | 内容源属于 builtin / plugin / bundled / mcp / setting source | 来源元数据 | builtin slash command 与 skill dir skill |
| `userFacingName()` | 对用户显示什么名字 | 展示层别名 | moved-to-plugin 类命令 |
| `kind` | UI 上是否打工作流等标签 | 展示标记；当前公开快照里更偏消费侧字段 | `workflow` |

## 4. 三套宿主过滤

| 过滤层 | 过滤集合 / 规则 | 核心问题 | 不能替代谁 |
| --- | --- | --- | --- |
| Headless / 非交互 | `prompt && !disableNonInteractive`；`local && supportsNonInteractive` | 脚本或无人值守时能跑哪些命令 | remote-safe / bridge-safe |
| Remote mode | `REMOTE_SAFE_COMMANDS` | `--remote` 下哪些本地控制面仍合理存在 | bridge-safe / headless-safe |
| Bridge inbound | `prompt` 默认允许，`local` 需进 `BRIDGE_SAFE_COMMANDS`，`local-jsx` 默认禁止 | 手机 / Web 远程输入能否触发 | headless-safe / remote-safe |

## 5. 三个高价值命令集合

| 集合 | 含义 | 应怎么写 |
| --- | --- | --- |
| `INTERNAL_ONLY_COMMANDS` | ant/internal 构建才会拼进主命令集合的一批命令 | 写成内部或灰度，不写入普通用户稳定主线 |
| `REMOTE_SAFE_COMMANDS` | remote mode 下保留的本地命令 | 写成 remote 宿主例外，不写成普遍规则 |
| `BRIDGE_SAFE_COMMANDS` | bridge/mobile/web 输入可执行的 `local` 命令 allowlist | 写成远程输入例外；还要继续检查 `getCommands()` 是否已把它过滤掉 |

## 6. 最容易混淆的对照

| 容易混淆 | 真正差异 |
| --- | --- |
| `availability` vs `isEnabled()` | 前者是静态资格，后者是动态放行 |
| `isHidden` vs 内部命令 | 隐藏不等于内部；内部也不一定只靠隐藏实现 |
| `supportsNonInteractive` vs bridge-safe | 非交互脚本可用，不等于手机/Web 远程可用 |
| `prompt` vs `prompt + fork` | 后者只是前者的一种执行上下文 |
| `userInvocable: false` vs `disableModelInvocation: true` | 前者禁用户直打，后者禁模型自动调用 |
| `local-jsx` vs `immediate local-jsx` | 后者只是允许在忙碌时先本地执行 |
| 同名命令 vs 同一对象 | `/context` 这类命令可在交互和 headless 下切换实现类型 |
| allowlist vs 当前可达 | 进入 remote/bridge allowlist 也可能继续被 `availability` / `isEnabled()` 过滤 |
