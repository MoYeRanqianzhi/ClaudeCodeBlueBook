# 设置、状态、预算与调参入口索引

这页只回答一个问题：

这些经常被一起提到的入口，到底属于哪种运行时对象。

如果你真正只关心的是：

- `/status`
- `/doctor`
- `/usage`

这三件事怎样组成“运行时自检”控制面，

改读 [../../05-控制面深挖/06-Status、Doctor、Usage：运行时自检、额度与诊断.md](../../05-%E6%8E%A7%E5%88%B6%E9%9D%A2%E6%B7%B1%E6%8C%96/06-Status%E3%80%81Doctor%E3%80%81Usage%EF%BC%9A%E8%BF%90%E8%A1%8C%E6%97%B6%E8%87%AA%E6%A3%80%E3%80%81%E9%A2%9D%E5%BA%A6%E4%B8%8E%E8%AF%8A%E6%96%AD.md)。

如果你还想把 `/config`、`/cost`、`/stats`、`/model`、`/effort`、`/statusline` 一起按对象层拆开，再继续留在本页，并接着看 [../../05-控制面深挖/10-设置面板、诊断屏与运营命令：会内控制面的三层分工.md](../../05-%E6%8E%A7%E5%88%B6%E9%9D%A2%E6%B7%B1%E6%8C%96/10-%E8%AE%BE%E7%BD%AE%E9%9D%A2%E6%9D%BF%E3%80%81%E8%AF%8A%E6%96%AD%E5%B1%8F%E4%B8%8E%E8%BF%90%E8%90%A5%E5%91%BD%E4%BB%A4%EF%BC%9A%E4%BC%9A%E5%86%85%E6%8E%A7%E5%88%B6%E9%9D%A2%E7%9A%84%E4%B8%89%E5%B1%82%E5%88%86%E5%B7%A5.md)。

## 四种对象

- `Settings tab`：挂在同一个 Settings 面板里
- `独立 screen`：不是 tab，而是单独 screen
- `session 调参`：直接改当前会话行为参数
- `预算/限流分流`：决定还能不能继续工作

## 入口对照

| 入口 | 对象类型 | 真实对象 | 关键边界 |
| --- | --- | --- | --- |
| `/status` | Settings tab | 当前宿主状态投影 | 没有真正的 root `claude status`；也不是 `auth status` |
| `/config` | Settings tab | 当前配置面 | 与 `/status`、`/usage` 共享同一容器 |
| `/usage` | Settings tab | 额度窗口与 extra usage 状态 | `Current session` 实际是 `five_hour` 配额窗，不是当前 `sessionId` |
| `/doctor` | 独立 screen | 环境可信度诊断 | 不在 Settings tabs 里；与 `claude doctor` 同屏异宿主 |
| `/stats` | 独立 screen | 长期使用统计与活动画像 | 主要基于本地 transcript / stats cache，不是远端 billing 真相 |
| `/cost` | local 命令 | 当前会话成本 / 当前计费语义 | 本地当前会话账本；对 subscriber 与非 subscriber 语义不同 |
| `/model` | session 调参 | 当前会话模型与相关能力面 | 会联动 fast mode 与 extra usage 提示 |
| `/effort` | session 调参 | 当前会话推理投入度 | 还会受 `CLAUDE_CODE_EFFORT_LEVEL` 覆盖；不等于 thinking |
| `/fast` | session 调参 | 当前会话吞吐策略 | 受可用性、冷却与模型兼容限制 |
| `/theme` | 显示层 | 全局主题 | 不改宿主真相或预算 |
| `/color` | 显示层 | 当前 session 的 prompt bar 身份色 | 只改显示层 |
| `/statusline` | prompt 工作流 | 状态线配置流程 | 不是查询命令，而是受控 subagent 工作流 |
| `/extra-usage` | 预算分流 | 继续工作时的 extra usage 配置 | 可能进入重新登录路径 |
| `/rate-limit-options` | 隐藏预算分流 | hit rate limit 时的产品分流 | 隐藏且 subscriber 条件化 |

## 最容易混写的几组

### `/status`、`/config`、`/usage`

这三条不是三个系统，而是同一个 `Settings` 面板的不同 tab。

### `/usage`、`/cost`、`/stats`

这三条不能合写成“费用入口”：

- `/usage`：额度窗口
- `/cost`：当前会话成本 / 当前计费语义
- `/stats`：长期统计画像

### `/model`、`/effort`、`/fast`

这三条都在调参，但不是同一维度：

- `/model`：能力面 / 成本面
- `/effort`：推理投入度
- `/fast`：吞吐与兼容策略

### `/statusline`

不要把它写成“显示当前状态”。

它做的是“配置状态线如何显示状态”。

## 邻接关系

- `claude auth status` 与 `/status` 不成对
- `/doctor` 与 `claude doctor` 成对，但对象分别是会内 screen 与会外诊断命令
- `/usage` 与 `/extra-usage` 紧邻，但前者是观测，后者是预算动作

## 用户策略

1. 想看当前宿主真相，用 `/status`。
2. 想改配置面，用 `/config`。
3. 想看额度窗口，用 `/usage`。
4. 想看环境可信度，用 `/doctor`。
5. 想看当前会话成本，用 `/cost`；想看长期画像，用 `/stats`。
6. 想调当前会话行为，再决定 `/model`、`/effort`、`/fast`。
7. 想把状态长期外显，用 `/statusline`，但把它当配置工作流而不是查询命令。
