# 命令索引

以下按功能分组，基于 `src/commands.ts` 与各命令定义整理。

边界提醒：

- 本页主要列 slash commands。
- `claude ...` 这一层的 root flags、root commands 与 fast-path 启动模式，请结合 `03-CLI 旗标与根命令索引.md` 阅读。
- 名字相近但层级不同的 root commands 与 slash commands，请结合 `04-根命令与斜杠命令对照索引.md` 阅读。
- `Settings` tabs、独立 screen、调参命令与预算分流，请结合 `05-设置、状态、预算与调参入口索引.md` 阅读。
- `prompt` / `local` / `local-jsx` 的对象差异，以及 `immediate`、headless、bridge、可见性字段的分工，请结合 `06-命令类型、执行语义与可见性索引.md` 阅读。
- `claude -p/--print`、`--bare`、`--sdk-url` 这类自动化相关入口，再结合 `04-专题深潜/13-非交互、后台会话与自动化专题.md` 一起读。

## 会话与导航

- `/help`：帮助与命令列表。
- `/tag`：给会话打标签，ant-only / 内部。
- `/exit`：退出 REPL。

## 上下文与连续性

- `/context`：查看上下文使用图。
- `/files`：列出当前上下文里的文件，ant-only / 内部。
- `/memory`：编辑记忆文件。
- `/compact`：压缩历史保留摘要。
- `/resume`：把同一工作对象重新接回可继续的现场。
- `/plan`：进入计划模式或查看当前计划。
- `/tasks`：查看与管理后台任务。

## 会话运营

- `/copy`：复制当前 Claude 输出或会话片段。
- `/rename`：重命名当前会话。
- `/export`：导出会话为文本。
- `/branch`：从当前点分叉会话。
- `/rewind`：回退代码或对话到较早状态。
- `/clear`：切出新的空工作面并生成新的 session id。

## 状态与运行时配置

- `/status`：打开 `Settings` 的 `Status` tab，查看当前宿主状态投影。
- `/config`：配置面板。
- `/permissions`：allow/deny 规则。
- `/model`：切换模型。
- `/fast`：切换快模式相关设置。
- `/effort`：设置 effort。
- `/theme`：主题。
- `/color`：会话提示条颜色。
- `/output-style`：输出风格，已趋向由 `/config` 接管。
- `/privacy-settings`：隐私设置。
- `/rate-limit-options`：限额命中后的隐藏分流入口。
- `/extra-usage`：额度不足时的额外使用策略。
- `/sandbox`：沙箱相关切换。
- `/passes`：额度/资格相关入口。

## 工具链与扩展管理

- `/mcp`：管理 MCP 服务器。
- `/plugin`：管理插件与市场。
- `/reload-plugins`：激活插件变更。
- `/skills`：列出可用技能。
- `/agents`：管理 agent 配置。
- `/hooks`：查看 hook 配置。
- `/add-dir`：加入新的工作目录并扩展工作面。
- `/install-github-app`：安装 GitHub Actions / GitHub App 相关支持，`claude-ai/console`。
- `/install-slack-app`：安装 Slack app，`claude-ai`。

## 代码工作流

- `/diff`：查看未提交改动与每轮 diff。
- `/review`：审查 PR。
- `/ultrareview`：远程深审路径，条件公开。
- `/commit`：生成 commit。
- `/commit-push-pr`：commit、push、开 PR。
- `/init`：初始化项目使用面。
- `/statusline`：设置或初始化 Claude Code 状态线。

## 设备与前端

- `/ide`：IDE 集成。
- `/desktop`：继续到 Claude Desktop，公开入口，但受平台 / 账户可见性约束。
- `/mobile`：移动端下载入口。
- `/chrome`：Claude in Chrome 设置，条件公开 / Beta。
- `/voice`：语音模式开关，条件公开。
- `/keybindings`：编辑按键绑定。
- `/vim`：Vim 模式切换。
- `/terminal-setup`：终端设置。

## 账户、观测与统计

- `/login`：登录 Anthropic 账户。
- `/logout`：退出登录。
- `/cost`：当前会话成本。
- `/usage`：查看额度与使用量。
- `/stats`：使用统计。
- `/insights`：生成会话分析报告。
- `/doctor`：安装、配置与环境诊断。
- `/release-notes`：版本说明，优先拉远程，失败退缓存。
- `/upgrade`：升级套餐或能力层级。
- `/feedback`：提交反馈，受 provider / policy / privacy level 约束。

## 远程与桥接

- `/remote-control`：把当前终端接成远程控制宿主，条件公开；默认帮助不显示。
- `/remote-env`：设置默认远程环境，条件公开，受 `claude.ai` 订阅与 `allow_remote_sessions` policy 约束。
- `/web-setup`：网页端相关设置，条件公开，受 `CCR_REMOTE_SETUP` 门控。
- `/session`：查看 remote mode 下的会话 URL / QR，viewer 侧接续，remote mode only。

## 杂项与非主线入口

- `/btw`：不打断主线的侧问题。
- `/stickers`：订贴纸。
- `/think-back`：年终回顾。
- `/thinkback-play`：播放动画。
- `/advisor`：顾问模型相关入口，条件公开 / 高级控制面。
- `/ultraplan`：网页端高级计划草拟，条件公开，受 `ULTRAPLAN` 门控。

## 内部命令与影子接近面

`INTERNAL_ONLY_COMMANDS` 中还注册了多条非普通用户内部命令，例如：

- `backfill-sessions`
- `break-cache`
- `bughunter`
- `ctx_viz`
- `good-claude`
- `issue`
- `mock-limits`
- `bridge-kick`
- `init-verifiers`
- `version`
- `summary`
- `teleport`（当前 slash 命令本体仍属内部 stub；普通用户更可能接触到的是隐藏 CLI flag `--teleport` 这一影子接近面）
- `env`
- `oauth-refresh`
- `debug-tool-call`
- `autofix-pr`

这些更适合视作内部诊断、实验或 Anthropic 自用能力，而不是稳定公开命令面；若只剩公开接近面，应另按影子能力处理。
