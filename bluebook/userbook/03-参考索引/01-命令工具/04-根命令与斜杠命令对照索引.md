# 根命令与斜杠命令对照索引

这页只回答一个问题：

哪些入口名字相近，但其实不在同一层。

请把它和下面两页配套读：

- `01-命令索引.md`：slash commands
- `03-CLI 旗标与根命令索引.md`：root flags、root commands、fast-path

如果把这页压成最短前门，只剩一条判断：

- `lifecycle plane -> session plane -> runtime gate`

再往 Prompt/runtime 这一层压，它其实只是在提醒一件事：

- root plane 先装世界，session plane 再在当前世界里继续
- 找不到对偶项时，直接承认 root-only 或 session-only，不要硬凑“CLI 版 / UI 版”

也就是：

- root 入口更偏会外生命周期、配置源、安装态与 inventory。
- slash 入口更偏当前 session 的状态、权限上下文、工具池与即时运行态。
- 名字相近时，先分 plane，再谈配对。

## 先记判断原则

- `claude ...` root command：更偏会外生命周期、配置源、安装态、inventory、诊断
- `/...` slash command：更偏当前会话状态、当前权限上下文、当前工具池、当前 session refresh

如果一个入口只存在于其中一层，不要硬配对。

## 进入本页前的 first reject signal

如果你现在问的是下面这些问题，也不该只停在这页：

- 这条 slash 命令到底是 `prompt` 还是本地动作。
- 这条入口当前为什么可见但不可用。
- 这条设置/状态/预算入口到底落在哪种运行时对象上。

这些都应继续回：

## 对照表

| 主题 | 会外 root 入口 | 会内 slash 入口 | 关键差异 |
| --- | --- | --- | --- |
| 认证登录 | `claude auth login` | `/login` | root 管认证生命周期；slash 会在当前 session 内刷新 auth state、policy、GrowthBook、authVersion |
| 认证退出 | `claude auth logout` | `/logout` | root 是会外注销；slash 会清当前身份现场并退出当前会话 |
| 认证状态 | `claude auth status` | 无真正等价项 | root 看认证状态；`/status` 看的是整个 Claude Code 会话状态，不是 auth-only |
| MCP 管理 | `claude mcp add/list/remove/get/...` | `/mcp` | root 管配置源和 server registration；slash 管当前 session 里的 server/connectivity 与即时开关 |
| MCP server | `claude mcp serve` | 无 | root 是让 Claude Code 充当 MCP server，不是 `/mcp` 管理页的另一种写法 |
| 插件生命周期 | `claude plugin install/list/update/...` | `/plugin` | root 管插件包和 marketplace 生命周期；slash 打开当前会话的插件设置面 |
| 插件生效 | 无 | `/reload-plugins` | 它解决的是“当前 session 吃到没有”，不是安装本身 |
| agent inventory | `claude agents` | `/agents` | root 列清单和来源覆盖；slash 打开当前会话的 agent 配置面 |
| 环境诊断 | `claude doctor` | `/doctor` | root 是会外诊断命令；slash 是当前会话里的诊断 screen |
| 宿主状态 | 无 | `/status` | 这是 session 状态投影，不是 root `auth status` 或 `doctor` 的别名 |
| token setup | `setup-token` | 无 | root-only，解决长生命周期 token setup |

## 最容易误判的几组

### `claude auth status` 与 `/status`

不要因为都有 `status` 就把它们写成同一类命令。

- `claude auth status`：认证状态
- `/status`：Claude Code 当前会话的宿主状态

### `claude mcp serve` 与 `/mcp`

不要因为都有 `mcp` 就把它们写成“一条是 CLI，一条是 UI”。

- `claude mcp serve`：server 侧入口
- `/mcp`：当前会话里的管理页

### `claude plugin install` 与 `/reload-plugins`

不要因为它们都和插件有关，就把它们写成同一阶段。

- `claude plugin install`：装包
- `/reload-plugins`：让当前 session 吃到变更

### `claude agents` 与 `/agents`

不要把两者都写成“agent 配置”。

- `claude agents`：inventory / resolved list
- `/agents`：当前 session 的管理面

## root-only

以下入口应直接写成 root-only，不要硬找 slash 同类项：

- `setup-token`
- `claude auth status`
- `claude mcp serve`
- `claude install`
- `claude update`

## slash-only 或 slash-first

以下入口应写成 session-first，不要硬找 root 同类项：

- `/status`
- `/reload-plugins`
- `/skills`
- `/hooks`

## 用户策略

更稳的速查方式是：

1. 看到同名词先问：这是在会外控制台，还是会内面板。
2. 需要持久化配置、安装、列清单、改来源时，优先找 root command。
3. 需要刷新当前会话、看当前状态、做即时开关时，优先找 slash command。
4. 找不到成对入口时，不要默认只是文档漏写，很多时候它本来就不成对。

更短的一句提醒是：

- 同名不等于同层，会外 plane 和会内 plane 先分开。
