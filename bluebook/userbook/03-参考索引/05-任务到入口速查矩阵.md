## 先分清你在改哪种对象

- `setup-token` 准备的是长期凭据；`/login` 重建的是当前会话身份现场。两者都和身份有关，但不是同一种动作。
- `/desktop` 解决的是当前工作对象 handoff；`/session` 提供的是 remote-mode viewer 接续地址；`/remote-control` 打开的是 host 侧宿主能力。三者相关，但不是同一种 continuation。
- `/plugin` 管插件包本体；`/reload-plugins` 让当前 session 吃到插件变更；`/skills`、`/hooks`、`/agents`、`/mcp` 看的是当前暴露面或外部 server 管理。不要把安装、刷新和可见能力写成同一步。

## 这张矩阵的用法

### 看主线

如果目标属于稳定列，就优先走稳定面，不要先上灰度面。

### 看边界

如果某项功能落在“条件公开但已成形”或“灰度更重”，就应在文档、团队约定和自动化中单独标注。

### 看替代路径

Claude Code 很多能力并不是“有或没有”，而是“有推荐路径和退化路径”。

但“次选入口”只有在仍然处理同一种对象时，才适合被当作 fallback；如果已经换了对象，就应该回到专题层先做主语分诊。

## 用户层结论

Claude Code 的真正强项不是单个命令有多炫，而是：

- 几乎每种工作目标都存在一个更低协调成本的正式入口。
- 当正式入口不可用时，通常还能退化到 prompt、shell、手工工作流。
- “补上下文”也不是单一步骤，而是至少拆成事实载体、事实面核对和工作面扩张三种动作。
- “自动化”也不是单一步骤，而是至少拆成一次性结果、当前会话内后台任务、CLI 后台会话和协议宿主四种对象。
- “交付”也不是单一步骤，而是至少拆成本地审查、远程深审、commit 收口、PR 协作升级、正式导出和证据化反馈六种对象。
- “终端效率”也不是单一步骤，而是至少拆成外观层、输入模式/键位、持续观测层和输入历史检索四种对象。
- “开工准备”也不是单一步骤，而是至少拆成仓库初始化、宿主安装/诊断/终端治理，以及外部平台接入三种对象。
- “账户面”也不是单一步骤，而是至少拆成身份现场、消费者产品隐私、资格显隐、升级后重新鉴权和限流分流五种对象。
