# 连续性与记忆专题

## 用户目标

让一次开发工作不是“聊完即散”，而是能够：

- 在长任务中不中断地继续。
- 在离开后恢复现场。
- 在上下文变长后压缩继续。
- 把稳定偏好沉到记忆层。

如果把这页继续压成工作对象入口，也只该剩三问：

1. 我现在缺的是继续当前工作、恢复旧工作现场，还是抽出长期稳定约束。
2. 我现在丢的是 working set、worksite，还是 stable rule。
3. 我现在要做的是 lawful forgetting、reload，还是回滚文件写入。

如果这三问还没分清，就不要先按命令名碰运气。

## 正确入口

### 上下文观察

先看 `/context` 和 `/files`。

这是为了确认“当前共同工作现场”到底包含什么，而不是靠感觉判断是否该继续喂信息。

### 记忆管理

用 `/memory` 处理长期有效的偏好、规则和项目事实。

源码里的 `claudemd.ts` 说明记忆层不是单文件，而是分层加载：

- managed memory
- user memory
- project memory
- local memory

而且支持 `@include`。

`memory.tsx` 还显示一个很实际的使用事实：

- `/memory` 不只是打开固定文件，而是先列出 memory file，再必要时创建文件并调用外部编辑器。
- 它会根据 `$VISUAL` / `$EDITOR` 决定编辑器来源，并把这一层行为显式反馈给用户。

所以 `/memory` 更像“记忆对象管理入口”，而不只是某个路径快捷方式。
更稳的读法还应再多记一句：`/memory` 只配沉长期稳定约束，不配替当前 working set 或 transient authority 签字。

### 压缩

用 `/compact`。

重要的不是“减少字数”，而是 Claude Code 会先做：

- compact boundary 处理。
- session memory compaction 尝试。
- microcompact。
- 再进入传统或 reactive compact 路径。

也就是说，它在尽量保留继续行动所需的结构，而不是随便生成一段摘要。
所以 `/compact` 不是单纯压字数，而是在执行 `lawful forgetting -> stable bytes externalize -> reload law`。

### 恢复

用 `/resume`。

`resume.tsx` 说明恢复逻辑考虑了：

- 同仓库工作树。
- 跨项目限制。
- 自定义标题匹配。
- session id 直达。
- lite log 与 full log。

所以 `/resume` 本质上是一个“恢复工作对象”的 UI，不只是聊天历史搜索器。
更硬一点说，它恢复的是仍具继续资格的 worksite，不是无条件接管旧标题、旧摘要或旧路径。

### 命名与导出

`/rename`、`/export` 虽然不直接改变主循环，但它们承担的是“让工作对象可识别、可回收、可外发”的责任。

对长任务来说，这些动作不是整理癖，而是在降低未来恢复与交接的认知成本。

需要额外纠偏的是：

- `/tag` 在当前源码里是 ant-only / 内部能力，不应继续写成普通用户稳定主线。

## 不该误用的邻近能力

### 不要把记忆当聊天备忘录

记忆适合长期稳定约束，不适合每回合临时任务状态。

### 不要把 `/compact` 当作清屏

它是连续性工具，不是 UI 整理工具。

### 不要把 `/resume` 当跨项目万能跳板

源码明确区分同目录、同仓库 worktree、不同项目三类恢复语义。

### 不要把 `/export` 当成连续性替代品

导出适合交付和外发，不适合替代系统内部的 compact / resume / memory 机制。

### 不要把故事连贯当成继续资格

如果 compact 后你只能说出“摘要还看得懂”，却说不出 `当前工作对象 / 下一步 guard / 必要资产 / stable rule`，就还不配继续。

## 稳定面与灰度面

### 稳定面

- `/context`
- `/files`
- `/memory`
- `/compact`
- `/resume`
- `/rename`
- `/export`

### 条件或内部面

- `/tag`
- 自动 session memory。
- 某些 reactive compact 或 cached microcompact 分支。
- 远程环境中的恢复补写逻辑。

## 对用户的策略结论

最稳的连续性工作流通常是：

1. 用 `/context` 看现场。
2. 用 `/memory` 沉淀长期规则。
3. 用 `/compact` 处理长任务。
4. 用 `/resume` 返回任务，而不是新开会话重讲。

如果继续把这条工作流压得更短，也只该剩一句：

- 先判 `working set / worksite / stable rule` 哪一层失真，再决定该用 `/compact`、`/resume`、`/memory` 还是 file checkpointing。
