# 字典与反查导航：标签字典、源码锚点与记录样例

这一篇不新增新的制度判断，而是回答一个更务实的问题：

- 当团队已经有索引层后，怎样最快查到标签定义、源码锚点和完整填表示例。

它主要回答四个问题：

1. 为什么索引层之后还需要参考层。
2. 为什么标签字典、反查表和样例库应一起出现。
3. 怎样从标签直接反查到章节与源码，而不再靠全文搜索。
4. 怎样用苏格拉底式追问避免把参考层做成新的文档堆。

## 1. 标签字典入口

如果问题是：

- 这个事故到底该叫 `boundary drift` 还是 `path parity split`。
- `stable-bytes drift` 和 `stop-logic failure` 的边界到底在哪里。

先读：

1. `casebooks/05`

它解决的是：

- 标签是什么意思
- 不该怎么误用

## 2. 反查入口

如果问题是：

- 知道标签了，但不知道该回到哪篇样本、哪份手册、哪条源码。

先读：

1. `casebooks/06`

它解决的是：

- `tag -> sample -> playbook -> chapter -> source anchor`

## 3. 样例入口

如果问题是：

- 模板已经有了，但还是不知道实际应怎样填写。

先读：

1. `playbooks/05`

它解决的是：

- 同一记录 ABI 在 Prompt、治理、结构三类对象上的完整示例

## 4. 一句话用法

如果：

- `12` 回答“怎样建立索引”

那么：

- `13` 回答“怎样直接用索引”

## 5. 苏格拉底式自检

在你准备继续扩案例或手册前，先问自己：

1. 团队现在缺的是新正文，还是缺可直接使用的定义与反查。
2. 一个新标签如果没有字典，团队真的会稳定使用它吗。
3. 一个新样本如果不能反查到源码，它真的能支持复盘吗。
4. 一个模板如果没有完整样例，团队真的会填吗。
