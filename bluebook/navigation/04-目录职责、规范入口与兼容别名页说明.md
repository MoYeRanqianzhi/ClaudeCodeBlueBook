# 目录职责与入口协议

这一篇只回答三个入口问题：

1. 蓝皮书的规范入口是什么。
2. 为什么根目录存在若干“别名页/兼容入口页”。
3. 读者在主线、专题与研究记录之间该如何切换，而不把目录结构读乱。

## 1. 规范入口

当前蓝皮书的规范主线入口是：

1. `bluebook/README.md`
2. `bluebook/00-导读.md`
3. `bluebook/01-源码结构地图.md`
4. `bluebook/02-使用指南.md`
5. `bluebook/03-设计哲学.md`
6. `bluebook/04-公开能力与隐藏能力.md`
7. `bluebook/05-功能全景与API支持.md`
8. `bluebook/06-第一性原理与苏格拉底反思.md`
9. `bluebook/07-运行时契约、知识层与生态边界.md`
10. `bluebook/08-能力全集、公开度与成熟度矩阵.md`
11. `bluebook/09-三张控制面总图：世界进入模型、扩张定价与防过去写坏现在.md`

也就是说：

- `README.md` 是顶层入口
- `00-09` 是规范主线

如果只想从一套最短路径进入，不需要猜，也不需要碰运气。

## 2. 兼容别名页

当前根目录还有三类“兼容入口页”：

- `00-总览.md`
- `00-蓝皮书总览.md`
- `01-源码总地图.md`

这些页面保留的原因不是目录继续失控，而是：

- 兼容旧链接
- 兼容不同命名习惯下的入口搜索

但从目录治理角度，它们不再是规范主线。

更准确地说：

- 规范入口负责当前阅读路径
- 兼容别名页负责历史链接不失效

## 3. 各目录职责

### 3.1 `bluebook/`

根目录只承载：

- 主线结论
- 总判断标准
- 最短阅读路径

### 3.2 `navigation/`

导航目录只承载：

- 阅读地图
- 问题检索
- 目录治理与入口协议

### 3.3 `architecture/`

承载：

- 运行时链路
- 状态机
- 装配链
- 恢复链
- 工程模式

### 3.4 `api/`

承载：

- 字段
- schema
- 控制协议
- 可接入性

### 3.5 `philosophy/`

承载：

- 第一性原理
- 设计内涵
- 演化方法

### 3.6 `guides/`

承载：

- 使用方法
- 接入策略
- 面向用户与宿主的实践路径

### 3.7 `playbooks/`

承载：

- 回归手册
- 事故复盘
- 演化演练
- 团队运行仪式

### 3.8 `casebooks/`

承载：

- 失败样本库
- 事故原型
- 结构反模式样本
- 具体案例复盘

它不承载整改待办或轮次级“改写路径”。

### 3.9 `security/`

承载：

- 安全控制面
- 权限与沙箱边界
- 恢复语义
- 工程化安全验证

### 3.10 `risk/`

承载：

- 风控
- 治理
- 账号/设备/策略

### 3.11 `docs/`

承载：

- 研究过程
- 证据索引
- 变更记录
- 目录治理记录

它不承载蓝皮书正文。

### 3.12 正文与研究记录分界

蓝皮书正文可以承载：

- 本篇回答什么、不再回答什么
- 概念边界、对象链与消费者顺序
- 第一性原理解释
- 苏格拉底式自检
- 稳定阅读顺序与跨目录跳转

蓝皮书正文不应承载：

- 环境路径、版本标记与提交细节
- 轮次进度、候选优先级与待办排程
- 编号推进日志与临时过程记事

这些内容都应进入：

- `docs/README.md`
- `docs/development/long-term-memory.md`
- `docs/development/research-log.md`
- `docs/changelog.md`

### 3.13 证据梯度分工

如果一个问题需要继续下钻，更稳的切换顺序不是“按主题找文件”，而是先问它属于哪一层证据职责：

1. 先验判断与不可约原则
   - 去 `philosophy/`
2. 机制对象、chokepoint 与当前真相保护
   - 去 `architecture/`
3. contract、schema 与 host-facing truth
   - 去 `api/`
4. 模板、审读清单与矩阵母版
   - 去 `guides/`
5. 准入、收口、回归与拒收 verdict
   - 去 `playbooks/`
6. distortion、伪证与误判样本
   - 去 `casebooks/`
7. judgment map 与跨目录反查
   - 去 `navigation/`
8. 研究过程、长时记忆与变更记录
   - 去 `docs/`

这条证据梯度一旦被打乱，目录结构就会重新混成：

- 总论页塞矩阵
- 导航页写正文
- 手册页写哲学
- 案例页写修复
- 记忆页抢正文

## 4. 目录切换协议

如果读者当前的问题是：

- “先理解 Claude Code 是什么”

就留在：

- `README.md + 00-09`

如果问题变成：

- “机制怎么实现”

就切到：

- `architecture/`

如果问题变成：

- “接口支不支持、字段是什么”

就切到：

- `api/`

如果问题变成：

- “为什么这样设计”

就切到：

- `philosophy/`

如果问题变成：

- “我应该怎么用”

就切到：

- `guides/`

如果问题变成：

- “我应该怎么运营、回归、复盘和演练”

就切到：

- `playbooks/`

如果问题变成：

- “安全控制面、权限边界、恢复语义怎样收口”

就切到：

- `security/`

如果问题变成：

- “真实失败长什么样、制度边界如何暴露”

就切到：

- `casebooks/`

如果问题变成：

- “我怎样把这些样本继续编目、标记并回流到演练体系”

就切到：

- `casebooks/04` 与 `playbooks/04`

如果问题变成：

- “风险、误伤、账号/设备约束与区域入口怎么理解”

就切到：

- `risk/`

如果问题变成：

- “研究过程、证据或变更记录该放哪里”

就切到：

- `docs/`

## 5. 一句话总结

蓝皮书当前的目录治理原则是：`README + 00-09` 是规范主线，`navigation/` 是检索层，`docs/` 是研究记录层，旧命名页只作为兼容别名保留，不再承担规范入口职责。
