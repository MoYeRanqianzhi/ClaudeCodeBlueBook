# 目录争议与兼容入口路由

这一页不再重写“蓝皮书的规范入口是什么”。规范入口、目录法、发言权与 first-answer order 的稳定 owner，统一回：

- `bluebook/README.md`
  - public frontdoor 与最小进入顺序
- `bluebook/09-三张控制面总图：世界进入模型、扩张定价与防过去写坏现在.md`
  - 宪法总图与三条母线
- `docs/development/00-研究方法.md`
  - 目录法、speaking rights、appeal chain 与入口升级规则

本页只处理两类问题：

1. 某个入口是不是只是 compat alias / route-only page。
2. 当你已经在争 page placement 或 route dispute 时，下一跳该去哪一层。

## 1. 兼容别名页

当前根目录仍保留若干 compat alias：

- `00-总览.md`
- `00-蓝皮书总览.md`
- `01-源码总地图.md`

它们存在的理由只有两个：

- 兼容旧链接
- 兼容不同命名习惯下的搜索

更稳一点说：

- 规范入口负责 first answer
- compat alias 只负责旧入口不断链

如果某个 compat alias 开始重新解释主语、最小顺序或首答来源/申诉链，应优先视为越位，而不是“补充说明”。

## 2. route dispute 怎么判

如果争议是：

- 这页有没有权第一次回答
  - 回 `bluebook/README.md + docs/development/00-研究方法.md`
- 这页是不是只是 next-hop / appeal path / compat alias
  - 留在本页
- 这次缺的是 evidence ceiling，而不是目录法
  - 回 `guides/102`
- 这次缺的是正文对象、why、contract 或 verdict
  - 先按下方类型分流；源码质量例外仍先经 `guides/102`

如果你已经知道对象，只是在争“该落哪一层”，也只按下面最短 next-hop 走：

- why：`philosophy/README.md`
- object / choke point / writeback seam：`architecture/README.md`
- contract / schema / host-facing truth：`api/README.md`
- audit ladder / template / candidate demotion：`guides/README.md`
- 源码质量例外：先经 `guides/102` 判 evidence mode / ladder，再分流到 `philosophy/87` 或 `architecture/README`
- recovery / reject / drill：`playbooks/README.md`
- distortion / anti-pattern：`casebooks/README.md`
- security / risk / reopen：`security/README.md`、`risk/README.md`
- process / memory / changelog：仓库根 `docs/`

## 3. 正文与记忆如何切换

- 需要稳定结论、对象链、why 或跨目录二跳时，回蓝皮书正文。
- 需要批次库存、候选顺序、持久化记忆或变更记录时，回仓库根 `docs/`。
- 如果争议已经上升到“谁拥有 speaking rights”，不要在这里继续争，直接回 `docs/development/00-研究方法.md`。

## 4. 一句话总结

本页只负责处理 compat alias 与 route dispute，不再重写规范入口；一切稳定首答来源/申诉链继续只认 `bluebook/README + 09 + docs/development/00`。
