# 目录职责与入口协议

这一篇只回答三个入口问题：

1. 蓝皮书的规范入口是什么。
2. 为什么根目录存在若干“别名页/兼容入口页”。
3. 读者在主线、专题与研究记录之间该如何切换，而不把目录结构读乱。

## 1. 规范入口

当前蓝皮书的规范主线入口是：

1. `bluebook/README.md`
2. `bluebook/00-导读.md`
3. `bluebook/01-源码结构地图.md`
4. `bluebook/02-使用指南.md`
5. `bluebook/03-设计哲学.md`
6. `bluebook/04-公开能力与隐藏能力.md`
7. `bluebook/05-功能全景与API支持.md`
8. `bluebook/06-第一性原理与苏格拉底反思.md`
9. `bluebook/07-运行时契约、知识层与生态边界.md`
10. `bluebook/08-能力全集、公开度与成熟度矩阵.md`
11. `bluebook/09-三张控制面总图：世界进入模型、扩张定价与防过去写坏现在.md`

也就是说：

- `README.md` 是顶层入口
- `00-09` 是规范主线

更稳一点的高阶入口协议还应补一句：

- `09` 负责宪法总图
- `05` 负责模仿对象校正
- `15` 负责失稳前追问
- `41` 负责第一性原理收束
- 如果问题已经变成“这一层到底有没有权第一次回答”，本页只负责把你路由到对应的 `README / guides / navigation` 节点，而不是再去各目录 README 拼目录政治。

如果只想从一套最短路径进入，不需要猜，也不需要碰运气。

## 2. 兼容别名页

当前根目录还有三类“兼容入口页”：

- `00-总览.md`
- `00-蓝皮书总览.md`
- `01-源码总地图.md`

这些页面保留的原因不是目录继续失控，而是：

- 兼容旧链接
- 兼容不同命名习惯下的入口搜索

但从目录治理角度，它们不再是规范主线。

更准确地说：

- 规范入口负责当前阅读路径
- 兼容别名页负责历史链接不失效

## 3. 常见问题先跳哪层

这一节不再替各目录发正式职责，只回答“这个问题先去哪里看”。

如果你要找：

1. 主线结论与最短进入规则
   - 先回 `bluebook/README.md` 与 `00-09`
2. 高阶前门校正
   - 先回 `navigation/05`、`navigation/15`、`navigation/41`
3. 第一性原理、设计内涵与演化方法
   - 先回 `philosophy/README.md`
4. 机制对象、chokepoint 与当前真相保护
   - 先回 `architecture/README.md`
5. contract、schema 与 host-facing truth
   - 先回 `api/README.md`
6. 模板、接入策略与降格规则
   - 先回 `guides/README.md`；若问题已明确落到 canonical ladder / candidate demotion，再进 `guides/102`
7. 回归、收口、拒收与演练
   - 先回 `playbooks/README.md`
8. distortion、伪证与误判样本
   - 先回 `casebooks/README.md`
9. 安全、风控与用户侧恢复语义
   - 先回 `security/README.md` 或 `risk/README.md`
10. 研究过程、长时记忆与变更记录
   - 先回仓库根 `docs/`

如果问题已经变成 `speaking rights / appeal chain / atlas projection / route dispute`，本页也只负责把你送到对应目录的 `README`、`guides/102`、`navigation/35` 或仓库根 `docs/`，不再替它们预先指定谁拥有目录宪法正文。

### 3.1 正文与记忆如何切换

- 需要稳定结论、对象链、第一性解释与跨目录二跳时，回蓝皮书正文。
- 需要轮次推进、候选优先级、长期记忆与变更记录时，回仓库根 `docs/`。
- 本页只提示先去哪一层，不再代签“正文该写什么、记忆该写什么”的全局宪法。

## 4. 目录切换协议

如果读者当前的问题是：

- “先理解 Claude Code 是什么”

就留在：

- `README.md + 00-09`

如果问题变成：

- “我是不是已经抄错对象”

就切到：

- `navigation/05`

如果问题变成：

- “第一条反证信号是什么、first reject path 在哪”

就切到：

- `navigation/15`

如果问题变成：

- “高阶结论是否已经压回第一性原理”

就切到：

- `navigation/41`

如果问题变成：

- “机制怎么实现”

就切到：

- `architecture/`

如果问题变成：

- “接口支不支持、字段是什么”

就切到：

- `api/`

如果问题变成：

- “为什么这样设计”

就切到：

- `philosophy/`

如果问题变成：

- “我应该怎么用”

就切到：

- `guides/`

如果问题变成：

- “我应该怎么运营、回归、复盘和演练”

就切到：

- `playbooks/`

如果问题变成：

- “安全控制面、权限边界、恢复语义怎样收口”

就切到：

- `security/`

如果问题变成：

- “真实失败长什么样、制度边界如何暴露”

就切到：

- `casebooks/`

如果问题变成：

- “我怎样把这些样本继续编目、标记并回流到演练体系”

就切到：

- `casebooks/04` 与 `playbooks/04`

如果问题变成：

- “风险、误伤、账号/设备约束与区域入口怎么理解”

就切到：

- `risk/`

如果问题变成：

- “研究过程、证据或变更记录该放哪里”

就切到：

- `docs/`

## 5. 一句话总结

蓝皮书当前的目录治理原则是：`README + 00-09` 是规范主线，`navigation/` 是检索层，`docs/` 是研究记录层，旧命名页只作为兼容别名保留，不再承担规范入口职责。
