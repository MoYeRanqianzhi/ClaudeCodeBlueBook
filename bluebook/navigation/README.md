# 导航专题

本目录不是新的正文平面，而是蓝皮书的导航层。

它回答的不是“Claude Code 的某个机制怎么实现”，而是：

1. 如果从第一性原理进入，这套文档应该怎么读。
2. 如果从某个具体问题进入，应该去哪一组章节查。
3. 如何把“功能全集”“协议支持”“治理边界”“设计哲学”放回同一张地图。

建议先读：

1. [第一性原理阅读地图](01-%E7%AC%AC%E4%B8%80%E6%80%A7%E5%8E%9F%E7%90%86%E9%98%85%E8%AF%BB%E5%9C%B0%E5%9B%BE.md)
2. [能力、API 与治理检索图](02-%E8%83%BD%E5%8A%9B%E3%80%81API%E4%B8%8E%E6%B2%BB%E7%90%86%E6%A3%80%E7%B4%A2%E5%9B%BE.md)
3. [深度专题导航：Prompt、预算、对象、底盘与治理](03-%E6%B7%B1%E5%BA%A6%E4%B8%93%E9%A2%98%E5%AF%BC%E8%88%AA%EF%BC%9APrompt%E3%80%81%E9%A2%84%E7%AE%97%E3%80%81%E5%AF%B9%E8%B1%A1%E3%80%81%E5%BA%95%E7%9B%98%E4%B8%8E%E6%B2%BB%E7%90%86.md)

和其他目录的关系：

- `bluebook/` 根目录：正式主线结论。
- `navigation/`：阅读地图与检索入口。
- `api/`：接口、字段、控制协议与可用性。
- `architecture/`：状态机、装配链、恢复链、工程分层。
- `philosophy/`：设计内涵、第一性原理与演化方法。
- `guides/`：面向使用者和接入者的实践路径。
- `risk/`：风控、治理、账号/设备/策略专题。
