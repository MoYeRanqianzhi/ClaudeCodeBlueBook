# 导航专题

本目录不是新的正文平面，而是蓝皮书的导航层。

它回答的不是“Claude Code 的某个机制怎么实现”，而是：

1. 如果从第一性原理进入，这套文档应该怎么读。
2. 如果从某个具体问题进入，应该去哪一组章节查。
3. 如何把“功能全集”“协议支持”“治理边界”“设计哲学”放回同一张地图。

和其他目录的关系：

- `bluebook/` 根目录：正式主线结论。
- `navigation/`：阅读地图与检索入口。
- `api/`：接口、字段、控制协议与可用性。
- `architecture/`：状态机、装配链、恢复链、工程分层。
- `philosophy/`：设计内涵、第一性原理与演化方法。
- `guides/`：面向使用者和接入者的实践路径。
- `risk/`：风控、治理、账号/设备/策略专题。
